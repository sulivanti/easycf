## Detalhamento

> **Contexto:** Este amendment deriva de [FR-000-M05](../fr/FR-000-M05.md), que por sua vez deriva de [UX-000-M04](../ux/UX-000-M04.md).
> O formulário de edição de roles (07-role-form-spec) inclui toggle de status, exigindo que o PUT aceite `status`.

### Alteração no OpenAPI v1.yaml

**Schema afetado:** `UpdateRoleRequest`

**Antes:**
```yaml
UpdateRoleRequest:
  type: object
  required: [scopes]
  properties:
    name: { type: string }
    description: { type: string, nullable: true }
    scopes:
      type: array
      minItems: 1
      items:
        type: string
```

**Depois:**
```yaml
UpdateRoleRequest:
  type: object
  required: [scopes]
  properties:
    name: { type: string }
    description: { type: string, nullable: true }
    scopes:
      type: array
      minItems: 1
      items:
        type: string
    status:
      type: string
      enum: [ACTIVE, INACTIVE]
      description: "Status do role. Se omitido, mantém o valor atual."
```

- Campo `status` é **opcional** (não está em `required`)
- Enum restrito a `ACTIVE` | `INACTIVE` (alinhado com CHECK constraint `roles_status_check` em DATA-000)

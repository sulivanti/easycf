## 2. UX-PARAM-001 — Configurador de Enquadradores

### 2.1 Jornada Principal (Happy Path)

```
1. Analista acessa /parametrizacao/enquadradores
2. GET /admin/framers carrega lista de enquadradores
3. GET /admin/target-objects carrega objetos-alvo
4. GET /admin/incidence-rules carrega regras de incidência
5. Tabela de enquadradores renderiza com filtro por tipo e status
6. Analista cria/edita enquadradores via drawer lateral
7. Painel de objetos-alvo exibe campos expandíveis
8. Matriz de incidência renderiza grid enquadradores × objetos
9. Analista clica em célula vazia para criar regra (ou ocupada para editar)
10. Analista simula motor no drawer de regra (dry-run)
```

## 5. Resolução de Conflitos Visuais na Matriz de Incidência

| Situação da célula | Ação do clique | Drawer abre como |
|---|---|---|
| **Vazia** | Criar nova regra | Modo criação (framer_id + target_object_id pré-preenchidos) |
| **Ocupada (ACTIVE)** | Editar regra existente | Modo edição (vigência, rotinas vinculadas, simular) |
| **Ocupada (INACTIVE)** | Visualizar regra inativa | Modo visualização (somente leitura, botão "Reativar") |

> **UNIQUE constraint** impede que o mesmo par (framer_id, target_object_id) tenha mais de uma regra — por isso célula ocupada nunca permite "Criar" (BR-003).

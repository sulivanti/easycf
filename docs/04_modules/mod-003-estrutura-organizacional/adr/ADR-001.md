# ADR-001 — N5 como Tenant Existente (MOD-000-F07), Não como Org Unit

## Contexto

O MOD-003 define uma hierarquia organizacional de 5 níveis (N1–N5). O nível N5 (Entidade Jurídica/Estabelecimento) corresponde conceitualmente ao tenant já existente no MOD-000-F07. A decisão é se o N5 deve ser modelado como uma nova `org_unit` de nível 5 ou se deve reutilizar a entidade `tenant` existente via tabela de vinculação.

## Decisão

**N5 NÃO é modelado como org_unit.** N5 é o tenant existente do MOD-000-F07, vinculado a nós N4 via tabela `org_unit_tenant_links`. O MOD-003 gerencia apenas N1–N4 na tabela `org_units`.

## Alternativas Consideradas

1. **Criar org_unit nível 5:** Duplicaria a entidade tenant, gerando inconsistência de dados e dois pontos de manutenção. Violaria o anti-pattern Foundation (DOC-FND-000).
2. **Usar herança (STI/polimorfismo):** Complexidade desnecessária para um vínculo simples N4→tenant.
3. **Vínculo via FK direto em org_units:** Misturaria responsabilidades — org_units teria campo `tenant_id` com semântica diferente do RLS.

## Consequências

- **Positivas:** Fonte única de verdade para tenants (MOD-000-F07); sem duplicação; tabela de link permite N4→múltiplos tenants.
- **Negativas:** Tree query precisa de JOIN extra para exibir N5; a árvore visual tem um nível "virtual" que não é org_unit.
- **Riscos:** Se MOD-000-F07 mudar schema de tenants, o JOIN pode precisar de ajuste.

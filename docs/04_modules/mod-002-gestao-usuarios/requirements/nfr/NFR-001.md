# NFR-001 — Requisitos Não-Funcionais (Gestão de Usuários)

> **Nota:** MOD-002 é UX-First (frontend). NFRs de backend (latência de API, disponibilidade de banco, DR) são responsabilidade do MOD-000-F05. Este documento foca nos NFRs da camada de apresentação.

---

## 1. SLOs (Service Level Objectives) — Frontend

| Métrica | Alvo | Medição |
|---|---|---|
| **Time to Interactive (TTI)** | ≤ 2s em conexão 4G | Lighthouse CI |
| **First Contentful Paint (FCP)** | ≤ 1.5s | Lighthouse CI |
| **Largest Contentful Paint (LCP)** | ≤ 2.5s | Lighthouse CI |
| **Cumulative Layout Shift (CLS)** | ≤ 0.1 | Lighthouse CI |
| **Debounce de busca** | 400ms (FR-001) | Timer client-side |
| **Redirect após sucesso** | 1.5s (FR-002) | Timer client-side |
| **Cooldown de reenvio** | 60s exato (BR-004) | Countdown client-side |

## 2. Topologia

- **Tipo:** Sync (frontend → API via HTTP REST)
- **Comunicação:** Todas as chamadas são síncronas (request/response). Não há WebSocket, SSE ou polling no MOD-002 v1.
- **Dependência:** Única — MOD-000 Foundation API. Sem dependências laterais.

## 3. Performance de UI

| Aspecto | Requisito |
|---|---|
| **Skeleton** | Exibido em ≤ 100ms após mount do componente |
| **Paginação** | Append (nunca re-render da lista inteira). Cursor-based sem offset. |
| **Filtros** | Refetch com overlay leve, sem re-mount da página |
| **Formulário** | Reset completo ao trocar modo (≤ 1 frame) |
| **Indicador de senha** | Client-side, sem chamada de API (0ms de latência de rede) |
| **Idempotency-Key** | UUID gerado no mount (síncrono, sem async) |
| **Botão submit** | Disable imediato após click (≤ 1 frame) para prevenir double-click |

## 4. Acessibilidade (a11y)

| Requisito | Padrão | Referência |
|---|---|---|
| Labels em todos os campos | MUST | WCAG 2.1 — 1.3.1 |
| Erros associados via `aria-describedby` | MUST | WCAG 2.1 — 1.3.1 |
| Foco no primeiro campo com erro | MUST | WCAG 2.1 — 3.3.1 |
| Trap de foco em modais | MUST | WCAG 2.1 — 2.4.3 |
| `aria-disabled` + `aria-busy` em botões loading | MUST | WCAG 2.1 — 4.1.2 |
| Contraste mínimo AA (4.5:1) | MUST | WCAG 2.1 — 1.4.3 |
| Navegação por teclado (tab, enter, esc) | MUST | WCAG 2.1 — 2.1.1 |
| Textos descritivos em badges de status | SHOULD | WCAG 2.1 — 1.1.1 |

## 5. Bundle e Code Splitting

| Requisito | Alvo |

| Tree-shaking | Apenas imports utilizados do design system |

## 6. Observabilidade (Client-Side)

| Aspecto | Requisito |
|---|---|
| **Correlation ID** | `X-Correlation-ID` propagado em todas as chamadas HTTP (DOC-ESC-001 §3.3) |
| **Telemetria UX** | `ux.action.requested`, `ux.action.succeeded`, `ux.action.failed` (DOC-DEV-001 §4.5 UX-010) |
| **Error tracking** | Erros não-handled capturados e reportados com screen_id + correlation_id |
| **Métricas de performance** | Web Vitals (LCP, FID, CLS) coletados e enviados ao backend de observabilidade |
| **Sem PII em logs** | NUNCA logar e-mail, nome ou dados sensíveis em console ou telemetria (DOC-ESC-001 §3.2) |

| Timeout HTTP | 15s (recomendado) | Herda config global do HTTP client (INT-001) |

## 10. Resiliencia de UI

| Aspecto | Requisito |
|---|---|
| **Erro de rede** | Toast generico "Sem conexao com o servidor." + correlationId (se disponivel) |
| **Timeout** | Apos 15s sem resposta: cancelar request, toast de timeout, botao habilitado |
| **Formulario preservado** | Erro 500 NAO limpa o formulario — admin pode corrigir e reenviar (FR-002) |
| **Retry manual** | Botao "Tentar novamente" em estados de erro de listagem (FR-001). Sem retry automatico. |
| **Cooldown sobrevive a erro** | Se cooldown esta ativo e ocorre erro em outra acao, cooldown NAO e resetado |
| **Navegacao durante loading** | Se admin navega durante request pendente: request e cancelado (AbortController) |

## 11. Seguranca de UI

| Aspecto | Requisito |
|---|---|
| **XSS** | Todos os dados da API renderizados via React (auto-escaped). Nunca `dangerouslySetInnerHTML`. |
| **Token storage** | Bearer Token gerido pelo HTTP client global. Nunca em localStorage acessivel a JS. |
| **PII em DevTools** | Console logs NAO devem conter e-mail ou dados sensiveis (DOC-ESC-001 §3.2) |
| **Scope enforcement** | Guards de rota no frontend SAO UX, NAO substituem enforcement do backend (DOC-FND-000 §2.3) |
| **CSP** | Nao utilizar `eval()`, inline scripts ou `unsafe-inline`. CSP gerido pelo Application Shell. |

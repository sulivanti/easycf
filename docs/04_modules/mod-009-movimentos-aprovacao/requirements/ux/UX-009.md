### State Machine — Card de Movimento no Inbox

```
[LOADING] ──load──► [IDLE]
  [IDLE] ──click_approve──► [CONFIRMING_APPROVE]
  [IDLE] ──click_reject──► [CONFIRMING_REJECT]
  [IDLE] ──click_cancel──► [CONFIRMING_CANCEL]
  [IDLE] ──click_override──► [OVERRIDE_MODAL]

  [CONFIRMING_APPROVE] ──confirm──► [SUBMITTING]
  [CONFIRMING_APPROVE] ──cancel──► [IDLE]
  [CONFIRMING_REJECT] ──confirm──► [SUBMITTING]
  [CONFIRMING_REJECT] ──cancel──► [IDLE]
  [CONFIRMING_CANCEL] ──confirm──► [SUBMITTING]
  [CONFIRMING_CANCEL] ──cancel──► [IDLE]
  [OVERRIDE_MODAL] ──submit──► [SUBMITTING]
  [OVERRIDE_MODAL] ──close──► [IDLE]

  [SUBMITTING] ──success──► [SUCCESS_REMOVING]
  [SUBMITTING] ──error_422──► [ERROR_INLINE]
  [SUBMITTING] ──error_5xx──► [ERROR_TOAST]

  [SUCCESS_REMOVING] ──animation_done──► [REMOVED]
  [ERROR_INLINE] ──dismiss──► [IDLE]
  [ERROR_TOAST] ──dismiss──► [IDLE]
```

- **LOADING:** Skeleton card, nenhuma ação disponível.
- **IDLE:** Card completo com botões de ação conforme condições de visibilidade.
- **CONFIRMING_*:** Parecer obrigatório (min 10 chars). Botão "Confirmar" habilitado após validação.
- **OVERRIDE_MODAL:** Modal com campo justificativa (min 20 chars) + checkbox dupla confirmação.
- **SUBMITTING:** Spinner no botão, card disabled, botões de ação desabilitados.
- **SUCCESS_REMOVING:** Toast de confirmação + card fade-out (300ms) + remoção da lista.
- **ERROR_INLINE:** Mensagem de erro inline abaixo do campo (422: segregação, parecer curto, etc.).
- **ERROR_TOAST:** Toast de erro genérico (5xx) com botão "Tentar novamente".

### Happy Path

1. Aprovador acessa `/aprovacoes` → inbox carrega movimentos pendentes filtrados por alçada
2. Clica "Ver detalhes" (`act-aprov-view-detail`) → painel lateral com Resumo, Operação, Cadeia de Aprovação, Histórico
3. Preenche parecer (min 10 chars) → clica "Aprovar" (`act-aprov-approve`) ou "Reprovar" (`act-aprov-reject`)
4. Toast de confirmação → card removido do inbox com fade-out

### Alternativas/erros

- **Segregação:** Botões "Aprovar/Reprovar" desabilitados se caller=solicitante E allow_self_approve=false. Tooltip: "Voce nao pode aprovar o proprio movimento."
- **Timeout proximo:** Badge vermelho "Expira em Xh Ym" no card. Countdown atualiza a cada 60s.
- **Override:** Modal com justificativa min 20 chars + checkbox de confirmacao dupla. Botao "Confirmar Override" so habilita apos ambos preenchidos.
- **Conflito de concorrencia:** Se outro aprovador decidiu durante visualizacao, retorna 409 Conflict → toast "Este movimento ja foi decidido." → card removido.

| Codigo HTTP | Componente | Comportamento |
|---|---|---|

| 403 | Painel inteiro | Empty state "Acesso negado" com mensagem contextual |
| 409 | Card individual | Toast "Movimento ja decidido" → card removido |
| 422 (segregacao) | Card / Form | Erro inline: "Voce nao pode aprovar o proprio movimento." |
| 422 (parecer curto) | ApproveRejectForm | Erro inline abaixo do campo: "Parecer deve ter no minimo 10 caracteres." |

| 422 (estado invalido) | Card | Toast: "Acao nao permitida no estado atual do movimento." |
| 429 | Global | Toast: "Limite de requisicoes excedido. Aguarde alguns segundos." |

### State Machine — Drawer de Regra

```
[CLOSED] ──click_new──► [CREATING]
[CLOSED] ──click_edit──► [EDITING]

  [CREATING] ──fill_fields──► [CREATING_DIRTY]
  [CREATING_DIRTY] ──submit──► [SAVING]
  [CREATING_DIRTY] ──close──► [CONFIRM_DISCARD]

  [EDITING] ──change_field──► [EDITING_DIRTY]
  [EDITING_DIRTY] ──submit──► [SAVING]
  [EDITING_DIRTY] ──close──► [CONFIRM_DISCARD]

  [SAVING] ──success──► [SAVED_REDIRECT_LEVELS]
  [SAVING] ──error_422──► [ERROR_INLINE]
  [SAVING] ──error_5xx──► [ERROR_TOAST]

  [SAVED_REDIRECT_LEVELS] ──auto──► [LEVEL_EDITOR]
  [ERROR_INLINE] ──fix──► [CREATING_DIRTY] | [EDITING_DIRTY]
  [ERROR_TOAST] ──dismiss──► [CREATING_DIRTY] | [EDITING_DIRTY]
  [CONFIRM_DISCARD] ──confirm──► [CLOSED]
  [CONFIRM_DISCARD] ──cancel──► [CREATING_DIRTY] | [EDITING_DIRTY]
```

- **CREATING / EDITING:** Drawer aberto com formulario. Campo `codigo` editavel apenas em CREATING (imutavel apos criacao).
- **SAVING:** Spinner no botao, campos desabilitados.
- **SAVED_REDIRECT_LEVELS:** Toast sucesso → abre automaticamente painel de alcadas da regra recem-criada.
- **CONFIRM_DISCARD:** Dialog "Descartar alteracoes nao salvas?" com botoes "Descartar" e "Continuar editando".

### Happy Path

1. Admin acessa `/aprovacoes/configurar` → tabela de regras de controle
2. Clica "Nova regra" (`act-config-create-rule`) → drawer com campos: codigo, object_type, operation_type, origin_types, priority, valid_from
3. Ativa toggle "Por valor" (`act-config-toggle-value`) → campos value_field e threshold aparecem
4. Cria regra → abre automaticamente painel "Alcadas"
5. Adiciona niveis de alcada (`act-config-add-level`) → preview visual: [Nivel 1 (24h)] → [Nivel 2 (48h)] → EXECUTAR
6. Clica "Simular motor" (`act-config-simulate`) → resultado inline: "CONTROLADO" ou "LIVRE"

### Alternativas/erros

- **Codigo duplicado:** 422 inline: "Ja existe regra com este codigo."
- **Campo codigo imutavel:** Em modo edicao, campo `codigo` exibido como texto (nao editavel) com tooltip "Codigo e imutavel apos criacao."
- **Vigencia expirada:** Badge ambar "Vigencia expirada" (ValidityBadge) na tabela. Regras expiradas podem ser editadas (atualizar valid_from).
- **Nenhuma alcada:** Alerta inline: "Regra sem alcadas — movimentos controlados ficarao bloqueados indefinidamente."
- **Limite de niveis:** Maximo 5 niveis por regra (NFR-009). Botao "Adicionar nivel" desabilitado apos 5 com tooltip "Maximo de 5 niveis atingido."

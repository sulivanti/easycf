# Emenda: FR-001-C02

- **Documento base:** [FR-001](../../requirements/fr/FR-001.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-25
- **owner:** ECF Core Team
- **Motivação:** `createOrgUnitEvent()` extrai `tenantId` de `payload['tenant_id']` que não existe nos payloads de CRUD (create, update, delete, restore). Resultado: `undefined ?? '' = ''` — string vazia em coluna `uuid NOT NULL` causa crash no INSERT. Use cases link/unlink passam `tenant_id` no payload, mas após o fix da factory, precisam passar `tenantId` como parâmetro explícito.
- **rastreia_para:** spec-fix-domain-events-tenant-id v2.0, DATA-000-C02, ADR-003, DATA-003

---

## Detalhamento

### 1. Problema observado

Ao criar, editar, deletar ou restaurar uma unidade organizacional, a API retorna erro 500:

```
Failed query: insert into "domain_events" (...) values (default, $1, ...)
params: ,org_unit,...  ← $1 (tenant_id) vazio
```

### 2. Causa-raiz

`createOrgUnitEvent()` em `org-unit-events.ts` (~linha 78):

```typescript
// ANTES (bugado):
tenantId: (params.payload['tenant_id'] as string) ?? '',
```

- Payloads de CRUD não contêm `tenant_id` → `undefined ?? '' = ''`
- Payloads de `tenant_linked`/`tenant_unlinked` contêm `tenant_id` → funciona por acidente

### 3. Decisão arquitetural

Org-units é **cross-tenant** por design (ADR-003). Domain events cross-tenant usam `SYSTEM_TENANT_ID` (`00000000-0000-0000-0000-000000000000`) como sentinel — mesmo padrão do Foundation para eventos pré-auth (DATA-000-C02).

### 4. Correção — Step 1: Fix factory `createOrgUnitEvent()`

**Arquivo:** `apps/api/src/modules/org-units/domain/events/org-unit-events.ts`

1. Importar `SYSTEM_TENANT_ID`:
```typescript
import { SYSTEM_TENANT_ID } from '@/modules/foundation/domain/events/foundation-events.js';
```

2. Adicionar parâmetro opcional na interface de params:
```typescript
tenantId?: string;  // opcional — SYSTEM_TENANT_ID como fallback (cross-tenant)
```

3. Substituir extração do payload:
```typescript
// ANTES:
tenantId: (params.payload['tenant_id'] as string) ?? '',

// DEPOIS:
tenantId: params.tenantId || SYSTEM_TENANT_ID,
```

### 5. Correção — Step 2: Fix use cases link/unlink

Após o fix da factory, `payload['tenant_id']` não é mais lido. Os use cases de link/unlink devem passar `tenantId` explicitamente.

**Arquivo:** `apps/api/src/modules/org-units/application/use-cases/link-tenant.use-case.ts`

```typescript
// No createOrgUnitEvent(), adicionar:
createOrgUnitEvent({
  tenantId: input.tenantId,  // ← ADICIONAR (UUID do tenant sendo vinculado)
  // ... demais params mantidos
})
```

**Arquivo:** `apps/api/src/modules/org-units/application/use-cases/unlink-tenant.use-case.ts`

```typescript
// Mesmo padrão:
createOrgUnitEvent({
  tenantId: input.tenantId,  // ← ADICIONAR (UUID do tenant sendo desvinculado)
  // ... demais params mantidos
})
```

### 6. Critérios de aceitação

- **AC-001:** `POST /api/v1/org-units` (criar unidade) → domain event `org.unit_created` persistido com `tenant_id = '00000000-0000-0000-0000-000000000000'`.
- **AC-002:** `PATCH /api/v1/org-units/:id` (editar), `DELETE /api/v1/org-units/:id` (deletar), `PATCH /api/v1/org-units/:id/restore` (restaurar) → domain events persistidos com `tenant_id = SYSTEM_TENANT_ID`.
- **AC-003:** `POST /api/v1/org-units/:id/tenants` (link tenant) → domain event `org.tenant_linked` persistido com `tenant_id` = UUID do tenant vinculado.
- **AC-004:** `DELETE /api/v1/org-units/:id/tenants/:tenantId` (unlink tenant) → domain event `org.tenant_unlinked` persistido com `tenant_id` = UUID do tenant desvinculado.
- **AC-005:** `pnpm -F @ecf/api build` compila sem erros.

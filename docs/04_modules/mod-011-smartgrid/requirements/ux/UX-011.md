## UX-SGR-001 — Grade de Inclusão em Massa

- **Screen ID:** UX-SGR-001
- **Entidade(s) principal(is):** `{entity_type}` do módulo destino (dinâmico)
- **Contexto:** Grade editável com inclusão em massa de novos registros
- **Features:** US-MOD-011-F02, US-MOD-011-F05

### Jornada (Happy Path)

1. Usuário navega para `/{modulo}/{rotina}/inclusao-em-massa`
2. Tela exibe skeleton enquanto carrega configuração da Operação (action: `load_operation_config`)
3. Motor retorna `visible_fields`, `required_fields`, `defaults`, `domain_restrictions`
4. Grade renderiza colunas conforme `visible_fields`, com asterisco em `required_fields` e defaults pré-preenchidos
5. Usuário adiciona linhas (action: `add_line`) e preenche valores
6. Usuário aciona "Validar Tudo" — motor chamado 1 linha por vez (action: `validate_line`)
7. Cada linha recebe status visual: ✅ válida, ❌ bloqueante, ⚠️ alerta
8. Com 100% ✅, botão "Salvar" é habilitado (BR-001)
9. Usuário aciona "Salvar" (action: `save_batch`) — registros criados no módulo destino
10. Feedback de sucesso com contagem

### Alternativas e Cenários de Erro

| Cenário | Comportamento | Referência |
|---|---|---|
| Motor retorna erro no mount (403/5xx) | Toast com mensagem de erro + `correlation_id`. Grade não carrega. | FR-002 |
| Linha com `blocking_validations` não vazio | Ícone ❌ com mensagem do motor. Save desabilitado. | BR-001, FR-003 |
| Linha com `validations` sem blocking | Ícone ⚠️. Save permitido (alerta informativo). | FR-003 |
| Nenhuma linha validada (status neutro) | Save desabilitado até validação. | BR-001 |
| Limite de linhas atingido | Botão "Adicionar" desabilitado. Mensagem explicativa. | FR-005, NFR-002 |
| Import JSON com Operação diferente | Aviso ao usuário. Grade populada, status neutro. | FR-004 |
| Import JSON acima do limite de linhas | Rejeição com mensagem explicativa. | FR-005, SEC-011 §7 |
| Import JSON com schema inválido | Rejeição com mensagem explicativa. | SEC-011 §7 |
| Fechar com dados não salvos | Modal de confirmação: "Exportar e sair" / "Sair sem exportar" / "Cancelar" | FR-004 |
| Erro ao salvar lote (parcial ou total) | Toast com contagem de sucesso/falha + `correlation_id`. | FR-009 |

### Estados da Tela

| Estado | Visual | Condição |
|---|---|---|
| Loading | Skeleton (grade completa) | Mount — carregando config do motor |
| Empty | Grade com 0 linhas + botão "Adicionar primeira linha" | Config carregada, nenhuma linha |
| Filling | Grade com N linhas, status neutro (sem ícone) | Linhas adicionadas, não validadas |
| Validating | Spinner por linha sendo validada | "Validar Tudo" em andamento |
| Validated (mixed) | Mix de ✅ ❌ ⚠️ por linha. Save desabilitado se ❌ existir. | Validação completa com erros |
| Validated (all OK) | Todas ✅. Save habilitado. | 100% válidas |
| Saving | Button loading no "Salvar" | Save em andamento |
| Error (mount) | Toast + grade vazia | Falha ao carregar config do motor |

## UX-SGR-002 — Formulário de Alteração de Registro

- **Screen ID:** UX-SGR-002
- **Entidade(s) principal(is):** `{entity_type}` do módulo destino (dinâmico)
- **Contexto:** Formulário de edição de registro existente, com validação por status via motor
- **Features:** US-MOD-011-F03

### Jornada (Happy Path)

1. Usuário navega para `/{modulo}/{rotina}/{id}/alterar`
2. Tela exibe skeleton enquanto avalia se registro pode ser editado (action: `evaluate_record_on_open`)
3. Motor avalia com `current_record_state` (F01 amendment)
4. Se sem `blocking_validations`: formulário renderiza com campos editáveis (visible_fields não bloqueados) e campos readonly (com cadeado)
5. Usuário altera campos editáveis
6. Usuário aciona "Validar" (action: `validate_changes`)
7. Se sem `blocking_validations`: botão "Salvar" habilitado
8. Usuário aciona "Salvar" (action: `save_changes`)
9. Registro alterado no módulo destino. Domain event com `changes[]` emitido.
10. Feedback de sucesso + redirecionamento para lista

### Alternativas e Cenários de Erro

| Cenário | Comportamento | Referência |
|---|---|---|
| Registro com status bloqueante | Componente BlockedRecordMessage exibido. Formulário NÃO abre. Botão "Voltar". | FR-006, BR-004 |
| Motor retorna erro (403/5xx) | Toast com mensagem de erro + `correlation_id`. Formulário não carrega. | FR-006 |
| Validação retorna `blocking_validations` | Mensagens inline por campo. Save desabilitado. | FR-006 |
| Validação retorna `validations` sem blocking | Mensagens de alerta inline. Save habilitado. | FR-006 |
| Erro ao salvar (409/422/5xx) | Mensagem inline + `correlation_id`. | FR-006 |
| Campo bloqueado por condição do motor | Campo renderizado como readonly com ícone de cadeado (não oculto). | BR-005, FR-006 |
| Cancelar e voltar | Navegação para `/{modulo}/{rotina}`. Sem modal de confirmação (formulário simples). | — |

### Estados da Tela

| Estado | Visual | Condição |
|---|---|---|
| Loading | Skeleton (formulário completo) | Avaliando registro via motor |
| Blocked | BlockedRecordMessage + botão "Voltar" | Motor retornou `blocking_validations` no open |
| Editing | Formulário com campos editáveis e readonly | Registro liberado para edição |
| Validating | Button loading no "Validar" | Validação em andamento |
| Validated (errors) | Mensagens inline por campo. Save desabilitado. | Validação com blocking |
| Validated (OK) | Save habilitado. | Validação sem blocking |
| Saving | Button loading no "Salvar" | Save em andamento |
| Error (mount) | Toast + formulário vazio | Falha ao avaliar registro |

## UX-SGR-003 — Grade de Exclusão em Massa

- **Screen ID:** UX-SGR-003
- **Entidade(s) principal(is):** `{entity_type}` do módulo destino (dinâmico)
- **Contexto:** Grade de validação pré-exclusão e confirmação de exclusão lógica em massa
- **Features:** US-MOD-011-F04

### Jornada (Happy Path)

1. Usuário seleciona registros na lista do módulo destino e navega para `/{modulo}/{rotina}/exclusao-em-massa`
2. Tela exibe lista dos registros selecionados (componente: SelectionList)
3. Usuário aciona "Verificar" — motor chamado por registro com `current_record_state` (action: `validate_for_delete`)
4. Registros classificados: liberados (sem blocking) em verde, bloqueados (com motivo) em vermelho
5. Componente DeleteConfirmationPanel exibe resumo: "{X} liberados para exclusão, {Y} bloqueados"
6. Usuário aciona "Confirmar exclusão ({X} registros)" (action: `confirm_delete`)
7. Exclusão lógica executada (`deleted_at` + `status=INACTIVE`) nos registros liberados
8. Componente DeleteResultFeedback exibe resultado por registro: ✅ excluído / ❌ falhou
9. Toast com contagem final

### Alternativas e Cenários de Erro

| Cenário | Comportamento | Referência |
|---|---|---|
| Todos os registros bloqueados | DeleteConfirmationPanel exibe todos em vermelho. Botão "Confirmar exclusão" desabilitado. | FR-007 |
| Cancelar antes da confirmação | Navegação para lista. Nenhum registro alterado. | FR-007 |
| Erro na verificação de 1 registro | Registro marcado como "erro" (ícone ❌) com mensagem. Demais continuam. | FR-007 |
| Erro ao excluir (parcial) | DeleteResultFeedback exibe mix de ✅ e ❌. Toast com contagem. | FR-007 |
| Motor retorna erro (403/5xx) | Toast com mensagem de erro + `correlation_id`. | FR-007 |
| Registro já excluído (409) | Registro marcado como "já excluído" no feedback. | FR-007 |

### Estados da Tela

| Estado | Visual | Condição |
|---|---|---|
| Loaded | SelectionList com registros selecionados + botão "Verificar" | Tela aberta, registros carregados |
| Verifying | Spinner global ou por registro | Validação em andamento |
| Verified | DeleteConfirmationPanel com liberados (verde) e bloqueados (vermelho) | Verificação completa |
| Confirming | Button loading no "Confirmar exclusão" | Exclusão em andamento |
| Completed | DeleteResultFeedback com resultado por registro | Exclusão finalizada |
| Error (verify) | Toast + registros não classificados | Falha na verificação |

## Telemetria Comum (todas as telas)

Todas as telas usam o pacote `@easycf/ui-telemetry` com os defaults definidos nos Screen Manifests:

| Configuração | Valor |
|---|---|
| `package` | `@easycf/ui-telemetry` |
| `include_tenant_id` | `true` |
| `propagate_correlation_id` | `true` |
| `capture_duration_ms` | `true` |

Cada ação emite um UIActionEnvelope (DOC-ARC-003) com campos obrigatórios: `screen_id`, `action`, `entity_type`, `tenant_id`, `correlation_id`, `status`, `duration_ms`.

## Acessibilidade (a11y)

- **MUST:** Grade (UX-SGR-001) acessível via teclado: Tab para navegar entre cells, Enter para editar, Escape para cancelar.
- **MUST:** Ícones de status (✅ ❌ ⚠️) DEVEM ter `aria-label` descritivo (ex: "Linha válida", "Linha com erro bloqueante", "Linha com alerta").
- **MUST:** Modal de confirmação (CloseConfirmationModal, DeleteConfirmationPanel) acessível: focus trap, Escape para fechar.
- **MUST:** Mensagens de erro inline associadas ao campo via `aria-describedby`.
- **Referência:** NFR-010

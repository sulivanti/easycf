### Alteração 3: Novo FR — Route Files e Wiring (20 rotas)

**Novo requisito funcional:**

O codegen do MOD-001 DEVE gerar route files para todas as rotas referenciadas na sidebar-config e shortcut-config. Três padrões de route file:

| Tipo | Quando usar | Exemplo |
|------|------------|---------|
| **A — Zero-prop** | Componente sem props de contexto | `_auth.profile.tsx` → import direto `ProfilePage` |
| **B — Wrapper** | Componente precisa de `userScopes` ou callbacks de navegação | `_auth.usuarios.tsx` → wrapper com `useAuthMe()` |
| **C — Params** | Rota com parâmetros de URL (`$id`, searchParams) | `_auth.cases.$id.tsx` → `useParams()` + wrapper |

**Regras (CA-07):**
- Route files NUNCA contêm JSX inline de página
- Route files SEMPRE importam Page Component do módulo correspondente
- Route files de módulos não gerados usam `ComingSoonPage` (CA-09)

**Done funcional:**
- 20 route files criados conforme tabela em UX-001-M02 §4
- `routeTree.ts` atualizado com ~24 rotas registradas como children de `authRoute`
- Cada route file importa componente real ou ComingSoonPage

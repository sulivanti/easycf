## Detalhamento

### 1. Dropdown de Ações — 4 Variantes por Status

A Jornada 1 de UX-001 (seção "Ações") deve incluir as seguintes ações contextuais, conforme layout 05-users-list-spec.md §7:

#### ATIVO
```
├── Editar              (primária, azul, ícone Pencil)        → navega para /usuarios/:id
├── Resetar senha       (normal, ícone KeyRound)              → ConfirmationModal
├── ── separador ──
├── Desativar           (perigosa, vermelha, ícone UserMinus)  → ConfirmationModal
└── Bloquear            (perigosa, vermelha, ícone ShieldBan)  → ConfirmationModal
```

#### INATIVO
```
├── Reativar            (primária, verde #1E7A42 sobre #E8F8EF, ícone UserCheck) → PATCH status
├── Editar              (normal, ícone Pencil)                → navega para /usuarios/:id
├── ── separador ──
└── Bloquear            (perigosa, vermelha, ícone ShieldBan)  → ConfirmationModal
```

#### BLOQUEADO
```
├── Desbloquear         (primária, verde #1E7A42 sobre #E8F8EF, ícone ShieldCheck) → PATCH status
├── Editar              (normal, ícone Pencil)                → navega para /usuarios/:id
├── ── separador ──
└── Desativar           (perigosa, vermelha, ícone UserMinus)  → ConfirmationModal
```

#### PENDENTE
```
├── Reenviar convite    (primária, azul, ícone Send)           → POST + CooldownButton 60s
├── Editar              (normal, ícone Pencil)                → navega para /usuarios/:id
├── ── separador ──
├── Cancelar convite    (perigosa, vermelha, ícone MailX)      → ConfirmationModal
└── Bloquear            (perigosa, vermelha, ícone ShieldBan)  → ConfirmationModal
```

### 2. Regras UX do Dropdown

1. **"Editar" existe em TODOS os estados** — sempre navega para o formulário de edição
2. **Ação primária** (destaque com fundo azul claro `#E3F2FD` ou verde claro `#E8F8EF`) é a mais relevante para aquele status
3. **Ações destrutivas** ficam abaixo do separador, em vermelho `#C0392B`
4. **ConfirmationModal obrigatório** para: Desativar, Bloquear, Cancelar convite
5. **Dropdown fecha** ao clicar fora ou ao selecionar uma ação
6. **O botão "•••" deve ser renderizado para TODOS os status** — nunca oculto

### 3. Tipografia do Dropdown (conforme layout)

```
Ação primária          600  13px  #2E86C1 (ou #1E7A42 para verde)
Ação normal            400  13px  #555555

```

## Impacto nos Pilares

- **Pilares afetados:** FR, INT, DATA, SEC
- **Ação requerida:**
  - **FR-001 (C):** Adicionar ações contextuais completas (Editar, Resetar senha, Bloquear, Reativar, Desbloquear, Cancelar convite) ao done criteria. Atualmente FR-001 só lista "Desativar" e "Reenviar convite".

  - **SEC-001 (M):** Definir scopes necessários para as novas ações (ex: `users:user:write` para bloquear/desbloquear/reativar/resetar senha, `users:user:delete` para cancelar convite).

## FR-000 — Requisitos Funcionais (Foundation)

- **Descrição:** Conjunto completo de capacidades do módulo Foundation (MOD-000), cobrindo autenticação, sessões, MFA, SSO, recuperação de senha, gestão de usuários, RBAC, filiais multi-tenant, telemetria de UI e infraestrutura de CI.
- **Prioridade:** Must
- **Perfis impactados:** Administrador, Usuário Padrão, Arquitetura, CI/CD Pipeline
- **Done funcional:** Pipeline CI verde, OpenAPI versionado sincronizado, cobertura de testes mínima por feature.
- **Efeito colateral?** [x] Sim
- **Dependências (IDs):** BR-000, DATA-000, INT-000, SEC-000, UX-000, NFR-000

---

### FR-000-F01 — Autenticação Nativa com E-mail e Senha

- **Descrição:** Login com `email` + `password`, criando `UserSession` em banco. Suporte a `remember_me` (TTL 12h ou 30d). Cookies `httpOnly` (`accessToken`, `refreshToken`). Kill-Switch via `DELETE /auth/sessions`. Renovação via `POST /auth/refresh`.
- **Prioridade:** Must
- **Endpoints:** `POST /auth/login`, `POST /auth/logout`, `GET /auth/sessions`, `DELETE /auth/sessions/:id`, `DELETE /auth/sessions`, `POST /auth/refresh`
- **Efeito colateral?** [x] Sim — `Idempotency-Key` MUST em `POST /auth/login`.

### FR-000-F02 — MFA / TOTP (RFC 6238)

- **Descrição:** Autenticação multifator via TOTP (Google Authenticator, Authy). Habilitação/desabilitação do MFA. Geração de `temp_token` com TTL 5min quando `mfa_secret` configurado. Verificação via `POST /auth/mfa/verify`.
- **Prioridade:** Must
- **Endpoints:** `POST /auth/mfa/setup`, `POST /auth/mfa/verify`, `DELETE /auth/mfa`

### FR-000-F03 — SSO OAuth2 (Google + Microsoft)

- **Descrição:** Login federado via OAuth2 (Google e Microsoft/Azure AD). Callback processa `code`, obtém token do provedor, localiza ou cria usuário local, cria `UserSession` e emite JWT. State CSRF obrigatório.
- **Prioridade:** Should
- **Endpoints:** `GET /auth/sso/google`, `GET /auth/sso/google/callback`, `GET /auth/sso/microsoft`, `GET /auth/sso/microsoft/callback`

### FR-000-F04 — Recuperação de Senha

- **Descrição:** Fluxo `forgot-password` (token UUID, TTL 1h, rate limit 3/15min) e `reset-password`. Token single-use (invalidado após uso). E-mail enviado via integração de transacional.
- **Prioridade:** Must
- **Endpoints:** `POST /auth/forgot-password`, `POST /auth/reset-password`
- **Efeito colateral?** [x] Sim — `Idempotency-Key` MUST em `POST /auth/forgot-password`.

### FR-000-F05 — Gestão de Usuários

- **Descrição:** CRUD de usuários (`POST`, `GET`, `PATCH`, `DELETE /users`). Soft-delete com `deleted_at`. Auto-registro público com `invite_token`. Ativação de conta via e-mail.
- **Prioridade:** Must
- **Endpoints:** `POST /users`, `GET /users`, `GET /users/:id`, `PATCH /users/:id`, `DELETE /users/:id`, `POST /users/:id/activate`, `POST /users/:id/block`

### FR-000-F06 — Roles / RBAC por Escopos

- **Descrição:** CRUD de Roles com arrays de `scopes` (`modulo:recurso:acao`). Validação semântica de scopes contra Catálogo de Permissões (F12). Cache Redis com TTL 5min por `(userId, tenantId)`. Herança de escopos do pai. DENY explícito prevalece.
- **Prioridade:** Must
- **Endpoints:** `POST /roles`, `GET /roles`, `GET /roles/:id`, `PATCH /roles/:id`, `DELETE /roles/:id`

### FR-000-F07 — Filiais Multi-Tenant

- **Descrição:** CRUD de filiais (`branches`). Hierarquia PAI → FILHA. Bloqueio de filial propaga para usuários. Soft-delete. Isolamento via `tenant_id` em todas as entidades.
- **Prioridade:** Must
- **Endpoints:** `POST /branches`, `GET /branches`, `GET /branches/:id`, `PATCH /branches/:id`, `DELETE /branches/:id`, `POST /branches/:id/block`

### FR-000-F08 — Perfil do Usuário Autenticado

- **Descrição:** Endpoint `GET /auth/me` retorna dados do usuário autenticado (perfil, filiais vinculadas, scopes). `PATCH /auth/me` permite edição de nome, avatar e preferências UX.
- **Prioridade:** Must
- **Endpoints:** `GET /auth/me`, `PATCH /auth/me`

### FR-000-F09 — Vinculação Usuário-Filial com Role

- **Descrição:** CRUD em `tenant_users` — vínculo entre usuário, filial e role. Múltiplos vínculos por usuário (escopos combinados). Remoção de vínculo = 204.
- **Prioridade:** Must
- **Endpoints:** `POST /branches/:id/users`, `GET /branches/:id/users`, `DELETE /branches/:id/users/:userId`

### FR-000-F10 — Alteração de Senha Autenticada

- **Descrição:** `POST /auth/change-password` com validação de senha atual, gate `force_pwd_reset` (bloqueia outras rotas até troca), revogação de todas as outras sessões ativas após a troca.
- **Prioridade:** Must
- **Endpoints:** `POST /auth/change-password`
- **Efeito colateral?** [x] Sim.

### FR-000-F11 — Endpoint GET /info

- **Descrição:** Endpoint público `GET /info` retornando versão da API, ambiente, timestamp e status de health. Não exige autenticação.
- **Prioridade:** Must
- **Endpoints:** `GET /info`

### FR-000-F12 — Catálogo de Permissões

- **Descrição:** CRUD de `permissions` (escopos pré-definidos). Validação referencial: Roles só podem atribuir escopos existentes no catálogo. Seed inicial obrigatório com escopos padrão do sistema.
- **Prioridade:** Must
- **Endpoints:** `GET /permissions`, `POST /permissions`, `GET /permissions/:id`, `PATCH /permissions/:id`, `DELETE /permissions/:id`

### FR-000-F13 — Utilitário de Telemetria UI (UIActionEnvelope)

- **Descrição:** Pacote `@easycf/ui-telemetry` exportando `UIActionEnvelope` conforme DOC-ARC-003. Campos obrigatórios: `screen_id`, `action`, `operation_id`, `correlation_id`, `status`, `duration_ms`. Emissão de eventos `ux.action.requested`, `ux.action.succeeded`, `ux.action.failed`.
- **Prioridade:** Must

### FR-000-F14 — Middlewares de Correlação E2E

- **Descrição:** Middleware `correlationIdMiddleware` que injeta/propaga `X-Correlation-ID` em cada request/response da API. Exportado em `@easycf/core-api`. Obrigatório em todos os módulos (registrado no bootstrap do `AppModule`).
- **Prioridade:** Must

### FR-000-F15 — Motor de Gates de Pipeline CI

- **Descrição:** Script `validate-manifests.js` que valida todos os arquivos YAML em `docs/05_manifests/screens/` contra o JSON Schema v1. Integrado em `npm run validate:manifests` e gate de CI. Falha com exit code != 0 se houver erro.
- **Prioridade:** Must

### FR-000-F16 — Módulo de Storage e Upload Centralizado

- **Descrição:** Geração de presigned URLs para upload direto no S3/GCS. Endpoint `POST /storage/upload-url` retorna `{ url, key, expires_at }`. Confirmação de upload via `POST /storage/confirm`. Nunca recebe arquivo binário diretamente na API.
- **Prioridade:** Must
- **Endpoints:** `POST /storage/upload-url`, `POST /storage/confirm`, `DELETE /storage/:key`
- **Efeito colateral?** [x] Sim — `Idempotency-Key` MUST em `POST /storage/upload-url`.

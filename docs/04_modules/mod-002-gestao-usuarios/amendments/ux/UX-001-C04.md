## Detalhamento

### Jornada 4 — Edição de Usuário (UX-USR-004)

#### Integração de Rota

A rota `/usuarios/:id` (`_auth.usuarios.$userId.tsx`) DEVE importar `UserEditPage` de `@modules/users/pages/UserEditPage` (MOD-002). O wrapper deve usar `useAuthMe()` para obter scopes e passar `userScopes`, `userId`, `onNavigateToList`.

#### Happy path

1. Admin clica "Editar" no dropdown da listagem → navega para /usuarios/:id
2. Dados carregam com skeleton → formulário preenchido
3. Campos editáveis: Nome completo, Perfil (select). E-mail readonly.
4. Se scope `users:user:write`: toggle de Status visível (ACTIVE/BLOCKED/INACTIVE)
5. Admin edita campos e clica "Salvar Alterações"
6. PATCH /api/v1/users/:id → botão isLoading → 200 → toast "Usuário atualizado com sucesso." → redirect /usuarios

#### Alternativas/erros

- Sem scope `users:user:read` → redirect /usuarios + toast "Sem permissão"
- GET 404 → redirect /usuarios + toast "Usuário não encontrado."
- PATCH 409 (e-mail duplicado, se editável no futuro) → inline no campo
- PATCH 422 → inline por campo
- PATCH 500 → toast + correlationId, formulário preservado

#### Estados

| Estado | Componente | Comportamento |
|---|---|---|
| **Loading** | Skeleton | Campos com skeleton durante GET /users/:id |
| **Error (load)** | Toast + redirect | 404 → redirect para /usuarios |
| **Error (submit)** | Toast ou inline | Inline para 422, toast para 500 |
| **Submit loading** | isLoading | Botão "Salvar" com spinner, disabled |

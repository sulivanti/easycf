# Emenda: UX-001-C02

- **Motivacao:** A rota `/usuarios/form` (`_auth.usuarios.form.tsx`) importa `UserFormPage` do modulo foundation (MOD-000) em vez do modulo users (MOD-002). A implementacao foundation e um CRUD generico sem modos de criacao (invite/password), sem idempotency-key, sem password strength meter, sem inline field errors RFC 9457 e sem protecao PII/LGPD nos toasts — nao atende UX-001 Jornada 2 (UX-USR-002).

## Detalhamento

### Problema

O route wrapper `apps/web/src/routes/_auth.usuarios.form.tsx` atualmente:

```typescript
import { UserFormPage } from '@modules/foundation/pages/users/UserFormPage';
// ...
<UserFormPage
  userId={id}
  onSuccess={() => navigate({ to: '/usuarios' })}
  onCancel={() => navigate({ to: '/usuarios' })}
/>
```

Deveria usar a implementacao completa do MOD-002:

```typescript
import { UserFormPage } from '@modules/users/pages/UserFormPage';
```

### Divergencias da Foundation vs MOD-002

| Aspecto | Foundation (atual) | MOD-002 (correto) | UX-001 |
|---------|-------------------|-------------------|---------|
| Modos de criacao | Formulario unico | Invite / Password (mutuamente exclusivos) | 2 modos (BR-003) |
| Password strength | Nenhum | Meter com 3 niveis (weak/medium/strong) | Strength meter |
| Idempotency-Key | Nenhum | Header gerado on mount (BR-005) | Idempotencia |
| Field errors | Toast generico | Inline por campo via RFC 9457 (BR-006) | Inline errors |
| PII em toasts | Email pode aparecer | Email nunca exposto (BR-002/LGPD) | PII-safe |
| Role selection | Nenhum | Select com roles da API MOD-000-F06 | Selecao de perfil |
| Redirect pos-sucesso | Manual via onSuccess | Automatico apos 1.5s | Auto-redirect |

### Correcao Requerida

**Arquivo:** `apps/web/src/routes/_auth.usuarios.form.tsx`

**De:**
```typescript
import { createRoute, useNavigate } from '@tanstack/react-router';
import { Route as authRoute } from './_auth';
import { UserFormPage } from '@modules/foundation/pages/users/UserFormPage';

export const Route = createRoute({
  path: '/usuarios/form',
  getParentRoute: () => authRoute,
  validateSearch: (search: Record<string, unknown>): { id?: string } => ({
    id: search.id as string | undefined,
  }),
  component: UserFormWrapper,
});

function UserFormWrapper() {
  const { id } = Route.useSearch();
  const navigate = useNavigate();

  return (
    <UserFormPage
      userId={id}
      onSuccess={() => navigate({ to: '/usuarios' })}
      onCancel={() => navigate({ to: '/usuarios' })}
    />
  );
}
```

**Para:**
```typescript
import { createRoute, useNavigate } from '@tanstack/react-router';
import { Route as authRoute } from './_auth';
import { UserFormPage } from '@modules/users/pages/UserFormPage';

export const Route = createRoute({
  path: '/usuarios/form',
  getParentRoute: () => authRoute,
  component: UserFormWrapper,
});

function UserFormWrapper() {
  const navigate = useNavigate();

  return (
    <UserFormPage
      onNavigateToList={() => navigate({ to: '/usuarios' })}
    />
  );
}
```

**Nota:** O MOD-002 `UserFormPage` nao aceita `userId` (e apenas formulario de criacao — edicao sera MOD-002-F04 futuro). O `validateSearch` pode ser removido pois nao e mais necessario. O redirect pos-sucesso e gerenciado internamente pelo componente (1.5s delay).

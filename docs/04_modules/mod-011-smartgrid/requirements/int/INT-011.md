## INT-001 — MOD-007 (Parametrização Contextual) — Motor de Avaliação

- **Direção:** Outbound (SmartGrid → MOD-007)
- **Criticidade:** CRITICAL (sem motor, grade não funciona)
- **Modo:** API REST (síncrono)
- **Auth:** Bearer JWT + scope `param:engine:evaluate`
- **Endpoint:** `POST /api/v1/routine-engine/evaluate`
- **operation_id:** `routine_engine_evaluate`
- **Frequência:** 1 chamada por linha por validação (BR-002)

### INT-001.1 — Contrato Request — Modo Inclusão (SEM current_record_state)

Usado na grade de inclusão em massa (UX-SGR-001) para carregar configuração da Operação e validar linhas novas.

```json
{
  "object_type": "string",
  "object_id": null,
  "context": [
    {
      "framer_id": "uuid"
    }
  ],
  "stage_id": "uuid | null",
  "current_record_state": null
}
```

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `object_type` | string | Sim | Tipo do objeto sendo avaliado (ex: `"compra_servico"`) |
| `object_id` | uuid / null | Não | `null` para novos registros (inclusão) |
| `context` | array | Sim | Array com pelo menos 1 framer (Operação) |
| `context[].framer_id` | uuid | Sim | ID do `context_framer` tipo OPERACAO |
| `stage_id` | uuid / null | Não | ID do estágio do processo (se aplicável) |
| `current_record_state` | object / null | Não | `null` para inclusão — motor opera em modo v1 |

### INT-001.2 — Contrato Request — Modo Alteração/Exclusão (COM current_record_state) — Amendment F01

Usado no formulário de alteração (UX-SGR-002) e na grade de exclusão em massa (UX-SGR-003).

```json
{
  "object_type": "string",
  "object_id": "uuid",
  "context": [
    {
      "framer_id": "uuid"
    }
  ],
  "stage_id": "uuid | null",
  "current_record_state": {
    "status": "string",
    "tipo": "string"
  }
}
```

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `object_type` | string | Sim | Tipo do objeto sendo avaliado |
| `object_id` | uuid | Sim | ID do registro existente |
| `context` | array | Sim | Array com framer(s) |
| `context[].framer_id` | uuid | Sim | ID do `context_framer` tipo OPERACAO |
| `stage_id` | uuid / null | Não | ID do estágio do processo |
| `current_record_state` | object | Sim (F01) | Estado atual do registro — campos avaliados por `condition_expr` |

> **Amendment F01:** Quando `current_record_state` está presente, o motor avalia `condition_expr` dos `routine_items` contra os campos fornecidos. Campos ausentes no `current_record_state` fazem a condição ser avaliada como `false` (degradação suave). Cache Redis é **BYPASSADO** quando `current_record_state` está presente (dado dinâmico não cacheia).

### INT-001.3 — Contrato Response (Comum a ambos os modos)

```json
{
  "visible_fields": ["field1", "field2"],
  "hidden_fields": ["field3"],
  "required_fields": ["field1"],
  "optional_fields": ["field2"],
  "defaults": {
    "field1": "value"
  },
  "domain_restrictions": {
    "field2": ["opt1", "opt2"]
  },
  "validations": [
    {
      "field": "field1",
      "rule": "regex_or_expression",
      "error": "Mensagem de validação"
    }
  ],
  "blocking_validations": [
    {
      "field": "field1",
      "rule": "regex_or_expression",
      "error": "Mensagem de bloqueio"
    }
  ],
  "applied_routines": [
    {
      "routine_id": "uuid",
      "routine_name": "Nome da rotina aplicada"
    }
  ]
}
```

| Campo do Response | Tipo | Uso no SmartGrid |
|---|---|---|
| `visible_fields` | `string[]` | Colunas da grade (F02) / campos visíveis no formulário (F03) |
| `hidden_fields` | `string[]` | Campos ocultos no formulário (F03) — não renderizados |
| `required_fields` | `string[]` | Campos com asterisco obrigatório |
| `optional_fields` | `string[]` | Campos sem marcação especial |
| `defaults` | `Record<string, any>` | Valores pré-preenchidos em novas linhas |
| `domain_restrictions` | `Record<string, string[]>` | Opções para dropdowns/selects |
| `validations` | `ValidationError[]` | Alertas não bloqueantes → ⚠️ |
| `blocking_validations` | `ValidationError[]` | Erros bloqueantes → ❌ |
| `applied_routines` | `RoutineRef[]` | Informativo — quais rotinas foram aplicadas |

### INT-001.4 — Mapeamento Response → Status Visual (PEND-SGR-01 — RESOLVIDA)

| Condição | Status Visual | Ícone | Impacto |
|---|---|---|---|
| `blocking_validations.length === 0` E `validations.length === 0` | Válida | ✅ | Save habilitado (contribui) |
| `blocking_validations.length > 0` | Bloqueante | ❌ | Save desabilitado (BR-001) |
| `blocking_validations.length === 0` E `validations.length > 0` | Alerta | ⚠️ | Save permitido |
| Sem avaliação realizada | Neutro | — | Save desabilitado |

## INT-003 — Módulo Destino (Dinâmico) — CRUD de Registros

- **Direção:** Outbound (SmartGrid → módulo destino variável)
- **Criticidade:** CRITICAL (sem módulo destino, save/delete não funciona)
- **Modo:** API REST (síncrono)
- **Auth:** Bearer JWT + scopes do módulo destino
- **Resolução de endpoints:** Via campo `target_endpoints` no `context_framer` tipo OPERACAO (PEND-SGR-04 — RESOLVIDA)

### INT-003.0 — Mecanismo de Resolução de Endpoints via `target_endpoints` (PEND-SGR-04 — RESOLVIDA)

O SmartGrid resolve os endpoints do módulo destino de forma **explícita e declarativa** através do campo `target_endpoints` presente no `context_framer` tipo OPERACAO do MOD-007. Este campo é lido no mount da grade (junto com `visible_fields`, `required_fields`, `defaults`, `domain_restrictions`) e armazenado em `operationConfig` (estado client-side).

**Como `target_endpoints` resolve os 3 fluxos:**

| Fluxo | Campo usado | Método | Path exemplo | Feature |
|---|---|---|---|---|
| Inclusão (save lote) | `target_endpoints.create` | `POST` | `/api/v1/compras/servicos` | F02 (FR-009) |
| Alteração (formulário) | `target_endpoints.update` | `PATCH` | `/api/v1/compras/servicos/{id}` | F03 (FR-006) |
| Exclusão (em massa) | `target_endpoints.delete` | `DELETE` | `/api/v1/compras/servicos/{id}` | F04 (FR-007) |

**Exemplo de `target_endpoints` no context_framer:**

```json
{
  "target_endpoints": {
    "create": { "method": "POST", "path": "/api/v1/compras/servicos" },
    "update": { "method": "PATCH", "path": "/api/v1/compras/servicos/{id}" },
    "delete": { "method": "DELETE", "path": "/api/v1/compras/servicos/{id}" }
  }
}
```

O placeholder `{id}` no path de update e delete é substituído pelo `object_id` do registro no momento da chamada.

> **Nota amendment MOD-007:** O campo `target_endpoints` deve ser adicionado ao schema de `context_framer` no MOD-007 via amendment. Este é um amendment menor — adiciona um campo opcional ao schema existente. Context_framers que não definem `target_endpoints` continuam funcionando normalmente (backward compatible). Ver DATA-011 §6 para schema completo.

### INT-003.1 — Inclusão de Registros (F02 — FR-009)

| Aspecto | Detalhe |
|---|---|
| Endpoint | Resolvido via `target_endpoints.create` — ex: `POST /api/v1/compras/servicos` |
| Resolução | SmartGrid lê `target_endpoints.create.method` e `target_endpoints.create.path` do `operationConfig` (mount da grade) |
| Frequência | 1 chamada por registro salvo no lote |
| Idempotência | `correlationId` no header para dedup pelo módulo destino |
| Fallback | Toast com correlationId e mensagem de falha por registro |

### INT-003.2 — Alteração de Registros (F03 — FR-006)

| Aspecto | Detalhe |
|---|---|
| Endpoint | Resolvido via `target_endpoints.update` — ex: `PATCH /api/v1/compras/servicos/{id}` |
| Resolução | SmartGrid lê `target_endpoints.update.method` e `target_endpoints.update.path` do `operationConfig`, substitui `{id}` pelo `object_id` |
| Frequência | 1 chamada por registro alterado |
| Payload | Apenas campos alterados + `correlation_id` |
| domain_events | Emitido pelo módulo destino com `{ field, old_value, new_value, source: "smartgrid" }` |
| Fallback | Inline error no formulário com correlationId |

### INT-003.3 — Exclusão Lógica de Registros (F04 — FR-007)

| Aspecto | Detalhe |
|---|---|
| Endpoint | Resolvido via `target_endpoints.delete` — ex: `DELETE /api/v1/compras/servicos/{id}` |
| Resolução | SmartGrid lê `target_endpoints.delete.method` e `target_endpoints.delete.path` do `operationConfig`, substitui `{id}` pelo `object_id` |
| Frequência | 1 chamada por registro liberado (bloqueados são ignorados) |
| Idempotência | Exclusão duplicada de registro já excluído = no-op ou 422 |
| Resultado | `deleted_at` + `status=INACTIVE` no módulo destino (BR-009) |
| domain_events | Emitido pelo módulo destino com `{ delete_type: "soft", source: "smartgrid" }` |
| Fallback | Feedback por registro: ✅ excluído ou ❌ falhou |

### INT-003.4 — Leitura de Registro para current_record_state (F03, F04)

| Aspecto | Detalhe |
|---|---|
| Endpoint | Dinâmico — `GET /api/v1/{modulo}/{entity}/{id}` (derivado do path base de `target_endpoints.update` sem o método) |
| Propósito | Obter estado atual do registro para popular `current_record_state` |
| Campos necessários | Campos referenciados em `condition_expr` das rotinas (ex: `status`, `tipo`) |
| Frequência | 1 chamada por registro ao abrir formulário (F03) ou pré-validar exclusão (F04) |

## Detalhamento

### Nova regra: SEC-SEED-PROD-001 — Proibição de credenciais default em produção

**Contexto:** Seeds de bootstrap (ex: `seed-admin.ts`) definem valores default para e-mail, senha e nome do admin via operador `??`. Esses defaults são aceitáveis em desenvolvimento local, mas representam risco de segurança em ambientes de produção e staging.

**Regra:**

> Todo script de seed que crie usuários com credenciais (senha, token, API key) **DEVE** verificar se os valores são defaults hardcoded e **RECUSAR** execução quando `NODE_ENV=production`.

**Implementação obrigatória:**

```ts
// Fail-safe: não permitir credenciais default em produção
if (ADMIN_PASSWORD === '<valor-default>' && process.env.NODE_ENV === 'production') {
  console.error('❌ ADMIN_PASSWORD default não é permitido em produção. Defina a variável de ambiente.');
  process.exit(1);
}
```

**Requisitos:**

1. O guard DEVE ser posicionado imediatamente após a definição das constantes de credenciais e antes de qualquer uso (hash, insert)
2. A mensagem de erro DEVE indicar claramente qual variável de ambiente precisa ser definida
3. O script DEVE encerrar com `process.exit(1)` (código de erro) — nunca silenciosamente ignorar
4. O guard aplica-se a **todos** os seeds que criam credenciais, não apenas ao `seed-admin.ts`
5. Em `NODE_ENV !== 'production'` (dev, test, undefined), os defaults continuam funcionando normalmente

**Cenários Gherkin:**

```gherkin
Funcionalidade: Fail-safe de credenciais em seed de produção

  Cenário: Seed em produção sem variáveis de ambiente definidas
    Dado que NODE_ENV é "production"
    E que ADMIN_PASSWORD não está definida no ambiente
    Quando o script seed-admin.ts é executado
    Então o script DEVE abortar com código de saída 1
    E a mensagem "ADMIN_PASSWORD default não é permitido em produção" DEVE ser exibida

  Cenário: Seed em produção com variáveis de ambiente definidas
    Dado que NODE_ENV é "production"
    E que ADMIN_PASSWORD está definida como "S3nh4-Pr0d-F0rt3!"
    Quando o script seed-admin.ts é executado
    Então o script DEVE executar normalmente

  Cenário: Seed em desenvolvimento sem variáveis de ambiente
    Dado que NODE_ENV não está definida (ou é "development")
    E que ADMIN_PASSWORD não está definida no ambiente
    Quando o script seed-admin.ts é executado
    Então o script DEVE executar normalmente com os defaults
    E o aviso "Troque a senha após o primeiro login!" DEVE ser exibido
```

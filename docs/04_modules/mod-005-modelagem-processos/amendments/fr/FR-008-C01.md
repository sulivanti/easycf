### 2. Lacuna de implementação frontend

O "Done funcional" de FR-008 lista corretamente `PATCH /admin/process-roles/:id edita (nome, descrição, can_approve)`, porém a implementação frontend gerada pelo codegen não inclui:

| Camada | Artefato esperado | Status |

### 3. Contrato de implementação

#### 3.1 Tipo TypeScript

export interface UpdateProcessRoleRequest {
  nome?: string;
  descricao?: string | null;
  can_approve?: boolean;
}

#### 3.2 API client

```typescript
/** @contract FR-008 — PATCH /admin/process-roles/:id */
export async function updateProcessRole(
  id: string,
  data: UpdateProcessRoleRequest,
): Promise<ProcessRoleDTO> {
  return httpClient.patch<ProcessRoleDTO>(`/admin/process-roles/${id}`, data);
}
```

#### 3.3 Hook React Query

```typescript
/** @contract FR-008 — PATCH /admin/process-roles/:id */
export function useUpdateProcessRole() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: ({ id, ...data }: UpdateProcessRoleRequest & { id: string }) =>
      updateProcessRole(id, data),
    onSuccess: () => {
      qc.invalidateQueries({ queryKey: PROCESS_ROLES_KEY });
    },
  });
}
```

#### 3.4 UI — ProcessRolesPage

- Adicionar coluna "Ações" na tabela com botão de editar (ícone lápis) em cada linha
- Dialog de edição preenchido com dados atuais do papel:
  - `codigo`: exibido como texto readonly (não editável — CON-001/BR-006)
  - `nome`: Input editável (required, maxLength=255)
  - `descricao`: Input editável (maxLength=1000)
  - `can_approve`: Checkbox editável
- Toast de sucesso após update + invalidação de cache
- Toast de erro sem fechar dialog em caso de falha

### 4. Critérios de aceite

- **AC-001**: Cada linha da tabela tem botão de editar
- **AC-002**: Dialog abre com dados preenchidos e `codigo` readonly
- **AC-003**: Submit envia PATCH e tabela reflete atualização
- **AC-004**: Erro exibe toast sem fechar dialog

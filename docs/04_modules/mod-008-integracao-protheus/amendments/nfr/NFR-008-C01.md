## Detalhamento

### Problema — 7 `no-explicit-any` + 5 unused directives

**Arquivo:** `apps/api/src/modules/integration-protheus/infrastructure/drizzle-repositories.ts`

**Padrão quebrado (5 métodos):**

```typescript
// eslint-disable-next-line @typescript-eslint/no-explicit-any  ← UNUSED (wrong line)
const [row] = await c
  .insert(integrationServices)
  .values(service as any)  // ← ACTUAL `any` here
  .returning();
```

**Métodos afetados:**

| Linha | Método | Tabela |
|-------|--------|--------|
| ~90 | `IntegrationServiceRepository.create()` | `integrationServices` |
| ~161 | `IntegrationRoutineRepository.create()` | `integrationRoutines` |
| ~226 | `FieldMappingRepository.create()` | `integrationFieldMappings` |
| ~295 | `IntegrationParamRepository.create()` | `integrationParams` |
| ~368 | `CallLogRepository.create()` | `integrationCallLogs` |

### Correção — Tipar com `InferInsertModel`

```typescript
import type { InferInsertModel } from 'drizzle-orm';

// Em cada método create/update, substituir:
.values(service as any)
// Por:
.values(service as InferInsertModel<typeof integrationServices>)
```

Remover todas as 5 linhas `// eslint-disable-next-line` que ficaram órfãs.

**Resultado esperado:** 0 errors, 0 warnings ESLint neste arquivo.

# Emenda: FR-001-C01

- **Documento base:** [FR-001](../../requirements/fr/FR-001.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-30
- **owner:** arquitetura
- **Motivação:** Regressão no logout: o hook `useLogout` do módulo backoffice-admin (criado pelo codegen batch `fed0682`) não limpa `localStorage['auth_tokens']` e usa `navigate()` em vez de `forceLogout()`. Resultado: após clicar "Sair", tokens permanecem no localStorage, router context mantém `auth.user` válido, route guard não redireciona, e chamadas API falham com 401 na `/dashboard`. A correção FR-000-C06 (que criou `forceLogout`) resolveu o cenário de sessão expirada por 401, mas o fluxo de logout manual foi regredido pelo codegen que duplicou o hook sem a limpeza de tokens.
- **rastreia_para:** FR-004, FR-000-C06, FR-022, spec-fix-logout-redirect-regression

## Detalhamento

### DEF-001 — Hook `useLogout` não limpa localStorage antes de redirecionar

**Localização:** `apps/web/src/modules/backoffice-admin/hooks/use-logout.ts:53-56`

O callback `onSettled` executa `queryClient.clear()` e `navigate({ to: '/login' })`, mas **não remove** `auth_tokens` do localStorage. Além disso, usa `navigate()` do TanStack Router em vez de `forceLogout()`.

```typescript
// ANTES (buggy — regressão introduzida por codegen fed0682)
onSettled: () => {
  queryClient.clear();
  navigate({ to: '/login' }); // ← tokens permanecem, context não reseta
},
```

### DEF-002 — Router context estático impede redirect via navigate()

**Localização:** `apps/web/src/main.tsx:17-32`

`getAuthFromStorage()` é executado UMA vez no render do `<App>`. O `navigate()` do router NÃO re-executa essa função — o `context.auth.user` permanece preenchido com dados do JWT antigo. O route guard `_auth.tsx:beforeLoad` vê `context.auth?.user` como válido e permite acesso à `/dashboard`.

### DEF-003 — Duplicação de hook `useLogout`

Existem duas implementações:

| Módulo | Arquivo | Limpa tokens? | Tipo de redirect |
|--------|---------|---------------|-----------------|
| Foundation | `foundation/hooks/use-auth.ts:70-87` | `clearTokens()` | `navigate()` |
| Backoffice Admin | `backoffice-admin/hooks/use-logout.ts:53-56` | **NÃO** | `navigate()` |

O `LogoutConfirmDialog.tsx` importa do backoffice-admin (caminho relativo `../hooks/use-logout`), não do foundation.

### Correção — Usar `forceLogout` do Foundation (padrão FR-000-C06)

**Arquivo:** `apps/web/src/modules/backoffice-admin/hooks/use-logout.ts`

```typescript
// DEPOIS (corrigido — alinhado com FR-000-C06 / FR-022)
import { forceLogout } from '@modules/foundation/utils/force-logout';

// ... (mutationFn permanece inalterado — telemetria preservada)

onSettled: () => {
  forceLogout(queryClient); // limpa localStorage + cache + hard redirect /login
},
```

**Mudanças:**
1. Import `forceLogout` do Foundation (já existe via FR-000-C06)
2. `onSettled` substitui `queryClient.clear() + navigate()` por `forceLogout(queryClient)`
3. Remove import de `useNavigate` (não mais necessário se não usado em outro ponto)
4. **Não altera** `mutationFn` — toda a telemetria (`emitRequested`, `emitSucceeded`, `emitFailed`) permanece intacta

**Justificativa do `forceLogout` sobre `navigate`:**
- `forceLogout` faz `localStorage.removeItem('auth_tokens')` + `queryClient.clear()` + `window.location.href = '/login'`
- O hard redirect (`window.location.href`) força re-execução de `getAuthFromStorage()` no `main.tsx`, gerando `context.auth = { user: null }`
- O route guard `_auth.tsx` corretamente redireciona para `/login` em requests subsequentes

### Gherkin — Critérios de aceite

```gherkin
Funcionalidade: Logout manual redireciona para login

Cenário: Logout limpa tokens e redireciona
  Dado que o usuário está autenticado na /dashboard
  Quando clica em "Sair" no ProfileWidget e confirma
  Então POST /auth/logout deve ser chamado
  E localStorage "auth_tokens" deve ser removido
  E o cache React Query deve ser limpo
  E o browser deve navegar para "/login" via full reload
  E a tela de login deve carregar normalmente

Cenário: Logout com erro de rede ainda limpa tokens
  Dado que o usuário clica em "Sair" e o POST /auth/logout falha (rede)
  Quando onSettled executa
  Então localStorage "auth_tokens" deve ser removido
  E o browser deve navegar para "/login" via full reload

Cenário: Após logout, navegar para /dashboard redireciona para /login
  Dado que o logout foi executado com sucesso
  Quando o usuário digita /dashboard na barra de endereço
  Então getAuthFromStorage() retorna { user: null }
  E o route guard _auth.tsx redireciona para /login

Cenário: Telemetria de logout preservada
  Dado que o usuário clica em "Sair"
  Quando o mutationFn executa
  Então emitRequested deve ser chamado com screenId=UX-SHELL-001
  E emitSucceeded ou emitFailed deve ser chamado conforme resposta
```

## Impacto nos Pilares

- **Pilares afetados:**
  - **FR (MOD-001 FR-004):** O cenário "Logout encerra sessão" do FR-004 já descreve "redirect para /login" — a implementação passa a usar `forceLogout` (hard redirect) em vez de `navigate` (soft redirect), alinhando com o padrão estabelecido por FR-000-C06
  - **UX (MOD-001 UX-001):** Nenhuma mudança na jornada UX — o comportamento visível ao usuário é o mesmo (clique "Sair" → tela de login). Apenas o mecanismo técnico muda

- **Ação requerida:**
  1. **Sem cascata necessária** — a correção é self-contained no hook `use-logout.ts`
  2. **Sem impacto em MOD-000** — `forceLogout` já existe e está MERGED (FR-000-C06)
  3. **Sem impacto em backend** — correção exclusivamente frontend

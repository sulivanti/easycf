# FR-001 — Requisitos Funcionais da Estrutura Organizacional

---

## FR-001 — CRUD de Unidades Organizacionais (N1–N4)

- **Descrição:** O sistema DEVE prover API CRUD completa para unidades organizacionais, incluindo: listagem flat com filtros e cursor-based pagination (`GET /org-units`), criação com nivel derivado (`POST /org-units`), detalhe com breadcrumb/ancestors (`GET /org-units/:id`), atualização de nome/descrição/status (`PATCH /org-units/:id`), e soft delete condicional (`DELETE /org-units/:id`).
- **Prioridade:** Must
- **Perfis impactados:** Administrador organizacional
- **Done funcional:**
  - POST /org-units cria nó com nivel calculado automaticamente
  - GET /org-units lista com cursor pagination, filtros por nivel/status
  - GET /org-units/:id retorna nó + ancestors[] + breadcrumb
  - PATCH /org-units/:id atualiza nome/descrição/status (codigo e parent_id são imutáveis)
  - DELETE /org-units/:id executa soft delete (deleted_at + status=INACTIVE) se sem filhos ativos
- **Efeito colateral?** [x] Sim *(create e link_tenant suportam Idempotency-Key)*
- **Idempotência:** required (POST /org-units aceita `Idempotency-Key`)
- **Timeline/Notifications:** yes — domain events: `org.unit_created`, `org.unit_updated`, `org.unit_deleted`
- **Dependências (IDs):** BR-001, BR-002, BR-003, BR-004, BR-005, BR-008, DATA-001, DATA-003, SEC-001, SEC-EventMatrix, NFR-001

## FR-002 — Tree Query via CTE Recursivo

- **Descrição:** O sistema DEVE prover endpoint `GET /org-units/tree` que retorna a hierarquia completa N1–N5 em estrutura aninhada, usando CTE recursivo. Nós N4 incluem array de tenants vinculados (N5). Nós com `deleted_at` preenchido são excluídos. Latência alvo < 200ms para 100 nós.
- **Prioridade:** Must
- **Perfis impactados:** Administrador organizacional
- **Done funcional:**
  - Retorna árvore completa N1→N2→N3→N4→(tenants N5)
  - Cada nó inclui breadcrumb e path
  - Nós soft-deleted são excluídos da árvore
  - Cache de árvore invalidado apenas em operações de escrita
- **Efeito colateral?** [ ] Não
- **Idempotência:** n/a (leitura)
- **Timeline/Notifications:** no
- **Dependências (IDs):** BR-001, DATA-001, NFR-001

## FR-003 — Vinculação N4 → Tenant (N5)

- **Descrição:** O sistema DEVE prover endpoints para vincular (`POST /org-units/:id/tenants`) e desvincular (`DELETE /org-units/:id/tenants/:tenantId`) tenants existentes (MOD-000-F07) a nós N4. Vinculação rejeita nós que não sejam N4. Tenant inexistente retorna 404. Vínculo duplicado retorna 409.
- **Prioridade:** Must
- **Perfis impactados:** Administrador organizacional
- **Done funcional:**
  - POST /org-units/:id/tenants com { tenant_id } cria vínculo (201)
  - Validação: org_unit.nivel == 4, tenant existe, par não duplicado
  - DELETE /org-units/:id/tenants/:tenantId executa soft unlink (deleted_at)
  - Domain events emitidos: org.tenant_linked, org.tenant_unlinked
- **Efeito colateral?** [x] Sim *(link_tenant suporta Idempotency-Key)*
- **Idempotência:** required (POST /org-units/:id/tenants aceita `Idempotency-Key`)
- **Timeline/Notifications:** yes — domain events: `org.tenant_linked`, `org.tenant_unlinked`
- **Dependências (IDs):** BR-006, BR-007, DATA-001, DATA-003, SEC-001, SEC-EventMatrix

## FR-004 — Restore de Unidade Organizacional (Soft-Deleted)

- **Descrição:** O sistema DEVE prover endpoint `PATCH /org-units/:id/restore` para restaurar um nó previamente soft-deleted. A restauração só é permitida se o nó pai (`parent_id`) estiver ativo (`deleted_at IS NULL`). Nós N1 (raiz) podem ser restaurados sem restrição de pai. Após restore, `deleted_at` é setado para `NULL` e `status` volta para `ACTIVE`.
- **Prioridade:** Must
- **Perfis impactados:** Administrador organizacional
- **Done funcional:**
  - PATCH /org-units/:id/restore restaura nó soft-deleted (200)
  - Validação: nó deve ter `deleted_at IS NOT NULL`
  - Validação: pai deve estar ativo (se não for raiz)
  - `deleted_at` → NULL, `status` → ACTIVE
  - Domain event emitido: `org.unit_restored`
- **Efeito colateral?** [x] Sim
- **Idempotência:** required (restore de nó já ativo retorna 200 sem efeito)
- **Timeline/Notifications:** yes — domain event: `org.unit_restored`
- **Dependências (IDs):** BR-009, DATA-001, DATA-003, SEC-001, SEC-EventMatrix

# Emenda: FR-001-C03

- **Documento base:** [FR-001](../../requirements/fr/FR-001.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)

- **owner:** ECF Core Team
- **Motivação:** 3 endpoints org-units retornam 500 "Response doesn't match the schema" porque os handlers enviam dados brutos do repository (camelCase, Date objects) sem mapear para o schema Zod (snake_case, ISO strings). - **rastreia_para:** spec-fix-org-units-response-schema-mismatch, FR-001, FR-005

---

## Detalhamento

### 1. Problema observado

Após criar uma unidade organizacional (POST → 201 OK), chamadas subsequentes retornam:
```json
{ "statusCode": 500, "error": "Internal Server Error", "message": "Response doesn't match the schema" }
```

### 2. Causa-raiz

3 handlers enviam dados raw do repository/use-case sem mapear para o contrato snake_case do schema Zod:

| Endpoint | Linha | Problema |
|----------|-------|----------|
| GET /org-units (list) | 103 | `data: result.data` — `parentId`, `createdAt` (Date) |
| GET /org-units/:id (detail) | 140 | `...result` spread inclui campos camelCase extras |
| PATCH /org-units/:id (update) | 182 | `result` raw sem mapeamento |

### 3. Correção — DEF-001: GET /org-units (list)

**Arquivo:** `org-units/presentation/routes/org-units.route.ts` ~linha 103

```typescript
// ANTES:
data: result.data,

// DEPOIS:
data: result.data.map((item: Record<string, any>) => ({
  id: item.id,
  codigo: item.codigo,
  nome: item.nome,
  nivel: item.nivel,
  status: item.status,
  parent_id: item.parentId ?? null,
  created_at: item.createdAt instanceof Date
    ? item.createdAt.toISOString()
    : item.createdAt,
})),
```

### 4. Correção — DEF-002: GET /org-units/:id (detail)

**Arquivo:** `org-units/presentation/routes/org-units.route.ts` ~linha 140

```typescript
// ANTES:
return reply.status(200).send({
  ...result,
  parent_id: result.parentId,
  ...
});

// DEPOIS:
return reply.status(200).send({
  id: result.id,
  codigo: result.codigo,
  nome: result.nome,
  descricao: result.descricao ?? null,
  nivel: result.nivel,
  parent_id: result.parentId ?? null,
  status: result.status,
  created_by: result.createdBy ?? null,
  created_at: result.createdAt instanceof Date
    ? result.createdAt.toISOString()
    : result.createdAt,
  updated_at: result.updatedAt instanceof Date
    ? result.updatedAt.toISOString()
    : result.updatedAt,
  deleted_at: result.deletedAt
    ? (result.deletedAt instanceof Date
        ? result.deletedAt.toISOString()
        : result.deletedAt)
    : null,
  ancestors: result.ancestors,
  tenants: result.tenants.map((t: Record<string, any>) => ({
    tenant_id: t.tenantId,
    codigo: t.codigo,
    name: t.name,
  })),
});
```

### 5. Correção — DEF-003: PATCH /org-units/:id (update)

**Arquivo:** `org-units/presentation/routes/org-units.route.ts` ~linha 182

```typescript
// ANTES:
return reply.status(200).send(result);

// DEPOIS:
return reply.status(200).send({
  id: result.id,
  codigo: result.codigo,
  nome: result.nome,
  descricao: result.descricao ?? null,
  nivel: result.nivel,
  status: result.status,
});
```

### 6. Critérios de aceitação

- **AC-001:** GET /api/v1/org-units retorna 200 com `parent_id` (snake_case) e `created_at` (ISO string).
- **AC-002:** GET /api/v1/org-units/:id retorna 200 com todos os campos snake_case e datas ISO.
- **AC-003:** PATCH /api/v1/org-units/:id retorna 200 com 6 campos do schema.
- **AC-004:** Nenhum endpoint retorna 500 "Response doesn't match the schema".

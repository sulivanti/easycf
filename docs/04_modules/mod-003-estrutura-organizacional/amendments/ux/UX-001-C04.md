## Detalhamento

### BUG-001: Handler `onLinkTenant` mapeado para `handleOpenEdit`

**Localização:** `OrgTreePage.tsx:362`

```typescript
// ANTES (incorreto):
onLinkTenant={(id) => handleOpenEdit(id)}

// DEPOIS (correto):
onLinkTenant={(id) => handleLinkTenant(id)}
```

O `OrgTreePage` deve gerenciar um novo estado `linkTenantState` para controlar a abertura do `LinkTenantModal`:

```typescript
interface LinkTenantState {
  open: boolean;
  orgUnitId: string;
  orgUnitNome: string;
  linkedTenantIds: string[];
}
```

O handler `handleLinkTenant` deve:
1. Selecionar o nó N4 (`setSelectedId(id)`)
2. Buscar os tenants já vinculados ao nó (disponível em `detailVM.tenants`)
3. Abrir o `LinkTenantModal` com os dados do nó

### BUG-002: Botão "+ Novo Departamento" na seção de tenants

**Localização:** `DetailPanel.tsx:361-365`

A seção "Departamentos Vinculados" do `DetailPanel` exibe `detail.tenants` (tenants N5 vinculados) mas o botão chama `onCreateChild(detail.id)`, que abre o formulário de criação de org_unit (impossível em N4 — BR-011, nível máximo).

**Correção:**
1. Renomear seção para "Estabelecimentos Vinculados" (tenants N5)
2. Renomear botão para "+ Vincular Estabelecimento"
3. Trocar handler para abrir o `LinkTenantModal` via nova prop `onLinkTenant`
4. Botão visível apenas quando `detail.nivel === 4 && hasWrite && !isEditing`

### Novo componente: LinkTenantModal

Conforme manifesto UX-ORG-001 (componente `link-tenant-modal`, seção components):

**Props:**

```typescript
interface LinkTenantModalProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  orgUnitId: string;
  orgUnitNome: string;
  linkedTenantIds: string[];
  onSuccess: () => void;
}
```

**Comportamento:**

| Elemento | Especificação |
|----------|---------------|
| Dialog title | "Vincular Estabelecimento" |
| Dialog subtitle | "Selecione um estabelecimento para vincular a '{orgUnitNome}'" |
| Campo de busca | SearchBar com debounce 300ms, placeholder "Buscar por nome ou código..." |
| Lista de resultados | Tenants `status=ACTIVE` filtrados por busca, excluindo `linkedTenantIds` |
| Item da lista | `{codigo} — {nome}` com botão "Vincular" (ou clique na linha) |
| Loading | Skeleton na lista durante fetch |
| Empty search | "Nenhum estabelecimento encontrado para o termo buscado." |
| Todos vinculados | "Todos os estabelecimentos já estão vinculados a esta unidade." |
| Botão "Vincular" | `POST /api/v1/org-units/:orgUnitId/tenants` com `{ tenant_id }` |
| Sucesso | Toast "Estabelecimento '{codigo}' vinculado a '{orgUnitNome}'." + fecha modal + invalidate queries |
| Erro 409 | Toast "Este estabelecimento já está vinculado a esta unidade." |
| Erro genérico | Toast com mensagem do servidor |

**Hooks necessários:**

1. `use-available-tenants.ts` — `GET /api/v1/tenants?status=ACTIVE&search=<termo>` com filtro client-side para excluir `linkedTenantIds`
2. `use-link-tenant.ts` — mutation `POST /api/v1/org-units/:id/tenants` com invalidação de queries (`org-tree`, `org-unit-detail`)

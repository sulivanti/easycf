## INT-000 — Integrações e Contratos Externos (Foundation)

---

### INT-000-01 — Provedor de E-mail Transacional

- **Sistema externo:** SendGrid / SMTP configurável via `EMAIL_PROVIDER` env.
- **Modo:** API (REST)
- **Auth:** API Key via header `Authorization: Bearer {SENDGRID_API_KEY}`
- **Uso:** Envio de e-mails de recuperação de senha (F04), confirmação de cadastro (F05) e ativação de conta.
- **Timeout da requisição:** 5000ms
- **Retry (Tentativas):** 3
- **Backoff e Jitter:** Exponencial, 500ms inicial
- **Dead Letter Queue (DLQ):** [x] Sim — via Outbox Pattern (`domain_events` + Worker BullMQ)
- **Idempotência:** `dedupe_key = (user_id, template_type, created_at_date)` — previne reenvio duplicado no mesmo dia
- **Fallback:** Enfileirar na DLQ e alertar monitoramento; NÃO bloquear a requisição principal.
- **Observabilidade:** `correlation_id` enviado como `X-Correlation-ID` ao provider; log de `email_sent`, `email_failed` com contagem de tentativas.

---

### INT-000-02 — Provedor SSO OAuth2 (Google)

- **Sistema externo:** Google Identity Services (OAuth2)
- **Modo:** OAuth2 Authorization Code Flow
- **Auth:** `client_id` + `client_secret` via env (`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`)
- **Endpoints externos:** `https://oauth2.googleapis.com/token`, `https://www.googleapis.com/oauth2/v3/userinfo`
- **Timeout da requisição:** 8000ms
- **Retry (Tentativas):** 2
- **Dead Letter Queue (DLQ):** [ ] Não — fluxo síncrono, falha reverte para tela de login
- **Idempotência:** State CSRF (`state` param UUID por sessão de login)
- **Fallback:** Redirecionar para tela de login com mensagem de erro genérica.

---

### INT-000-03 — Provedor SSO OAuth2 (Microsoft / Azure AD)

- **Sistema externo:** Microsoft Identity Platform (OAuth2 / OIDC)
- **Modo:** OAuth2 Authorization Code Flow
- **Auth:** `client_id` + `client_secret` via env (`MICROSOFT_CLIENT_ID`, `MICROSOFT_CLIENT_SECRET`, `MICROSOFT_TENANT_ID`)
- **Endpoints externos:** `https://login.microsoftonline.com/{tenant}/oauth2/v2.0/token`, `https://graph.microsoft.com/v1.0/me`
- **Timeout da requisição:** 8000ms
- **Retry (Tentativas):** 2
- **Dead Letter Queue (DLQ):** [ ] Não
- **Idempotência:** State CSRF (`state` param UUID por sessão de login)
- **Fallback:** Redirecionar para tela de login com mensagem de erro genérica.

---

### INT-000-04 — Storage (S3-compatible: AWS S3 / GCS / MinIO)

- **Sistema externo:** AWS S3 (ou compatível via `S3_ENDPOINT` env)
- **Modo:** API (AWS SDK / presigned URLs)
- **Auth:** `AWS_ACCESS_KEY_ID` + `AWS_SECRET_ACCESS_KEY` via env
- **Uso:** Geração de presigned URLs para upload direto (F16). Confirm de upload valida existência do objeto no bucket.
- **Timeout:** Presigned URL TTL: 300s (5 min). Request de confirm: 5000ms.
- **Retry (Tentativas):** 3
- **Idempotência:** `Idempotency-Key` em `POST /storage/upload-url`
- **Fallback:** Retornar erro 503 com `problem_type=/problems/storage-unavailable`; nunca salvar arquivo na API.

---

### INT-000-05 — Redis (Filas BullMQ + Idempotência + Rate Limiting)

- **Sistema externo:** Redis (self-hosted ou Redis Cloud)
- **Modo:** Client direto via `ioredis`
- **Auth:** `REDIS_URL` via env (inclui senha se necessário)
- **Uso:** BullMQ (filas de Worker), Rate Limiting, `Idempotency-Key` TTL cache, RBAC cache (TTL 5min).
- **MUST NOT:** Usar como banco primário ou Pub/Sub autônomo.
- **MUST NOT:** Criar camadas de cache de leitura/busca sem ADR documentada.

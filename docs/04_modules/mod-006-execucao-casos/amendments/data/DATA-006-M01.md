### Alteracoes neste pilar

#### A1 — Adicionar colunas `description` e `priority` em `case_instances`

Adicionar ao schema da tabela `case_instances` (DATA-006 §2.1):

| Campo | Tipo | Constraint | Descricao |
|---|---|---|---|
| `description` | text | nullable | Descricao do caso informada na abertura |
| `priority` | varchar(10) | NOT NULL DEFAULT 'NORMAL' CHECK(priority IN ('NORMAL','HIGH','URGENT')) | Prioridade do caso |

**Value Object adicional:**

- `CasePriority`: NORMAL | HIGH | URGENT — sem regra de negocio associada (campo informativo)

**Migracao:**

```sql
ALTER TABLE case_instances
  ADD COLUMN description text,
  ADD COLUMN priority varchar(10) NOT NULL DEFAULT 'NORMAL'
    CHECK (priority IN ('NORMAL', 'HIGH', 'URGENT'));
```

**Nota:** Migracao non-breaking — `description` e nullable e `priority` tem default. Nenhuma tabela existente precisa de backfill.

#### A2 — Indice para filtro por prioridade (opcional)

Se no futuro houver filtro por prioridade na listagem, considerar indice:

```sql
CREATE INDEX idx_case_instances_priority ON case_instances (tenant_id, priority) WHERE priority != 'NORMAL';
```

Por ora, **NAO criar** — apenas documentar como evolucao futura. Filtro por prioridade nao esta na spec UX-006-M01.

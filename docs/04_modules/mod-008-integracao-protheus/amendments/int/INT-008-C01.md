# Emenda: INT-008-C01

- **Documento base:** [INT-008](../../requirements/int/INT-008.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-30
- **owner:** arquitetura
- **Motivação:** Fix de prefixo de registro Fastify — `servicesRoutes` e `routinesRoutes` registrados com prefixo `/api/v1` mas o contrato (INT-008 §5, frontend API client) espera `/api/v1/admin`. Causa erro "Route GET:/api/v1/admin/integration-services not found" em produção.
- **rastreia_para:** spec-fix-integration-services-route-prefix, INT-008 §5, FR-008

---

## Detalhamento

### Problema

No arquivo `apps/api/src/index.ts` (linhas 155-158), os plugins de rota do MOD-008 são registrados assim:

```typescript
await app.register(servicesRoutes, { prefix: '/api/v1' });       // ❌
await app.register(routinesRoutes, { prefix: '/api/v1' });       // ❌
await app.register(integrationEngineRoutes, { prefix: '/api/v1' }); // ✅
```

Os paths definidos nos arquivos de rota são relativos (ex: `/integration-services`), resultando em rotas finais **sem** o segmento `/admin/`:

| Rota resultante | Rota esperada (INT-008 §5) |
|---|---|
| `GET /api/v1/integration-services` | `GET /api/v1/admin/integration-services` |
| `POST /api/v1/integration-services` | `POST /api/v1/admin/integration-services` |
| `PATCH /api/v1/integration-services/:id` | `PATCH /api/v1/admin/integration-services/:id` |
| `POST /api/v1/routines/:id/integration-config` | `POST /api/v1/admin/routines/:id/integration-config` |
| `POST /api/v1/routines/:id/field-mappings` | `POST /api/v1/admin/routines/:id/field-mappings` |
| `PATCH /api/v1/field-mappings/:id` | `PATCH /api/v1/admin/field-mappings/:id` |
| `DELETE /api/v1/field-mappings/:id` | `DELETE /api/v1/admin/field-mappings/:id` |
| `POST /api/v1/routines/:id/params` | `POST /api/v1/admin/routines/:id/params` |
| `PATCH /api/v1/integration-params/:id` | `PATCH /api/v1/admin/integration-params/:id` |

O `engineRoutes` funciona corretamente porque inclui `/admin/` diretamente nos paths das rotas administrativas (`/admin/integration-logs`) e não usa `/admin/` nas rotas públicas (`/integration-engine/execute`).

### Correção

Alterar o prefixo de registro no `apps/api/src/index.ts`:

```typescript
// MOD-008 Integration Protheus — relative paths
await app.register(servicesRoutes, { prefix: '/api/v1/admin' });    // ✅ corrigido
await app.register(routinesRoutes, { prefix: '/api/v1/admin' });    // ✅ corrigido
await app.register(integrationEngineRoutes, { prefix: '/api/v1' }); // sem alteração
```

### Arquivos afetados

| Arquivo | Alteração |
|---|---|
| `apps/api/src/index.ts` | Linhas 156-157: prefixo `/api/v1` → `/api/v1/admin` |

### Validação

- `GET /api/v1/admin/integration-services` deve retornar 200 (com auth) ou 401 (sem auth), nunca 404
- `POST /api/v1/admin/routines/:id/integration-config` deve retornar 200/201, nunca 404
- `POST /api/v1/integration-engine/execute` e `GET /api/v1/admin/integration-logs` devem continuar funcionando

### Novos eventos no catálogo DATA-003

Adicionar à tabela de domain events (§ Catálogo de Eventos), na seção de eventos do entity_type `user`:

| event_type | Descrição | origin_command | entity_type | Emit (scope) | View (ACL) | Notify | sensitivity_level | PII no payload |
|---|---|---|---|---|---|---|---|---|
| `user.password_reset` | Admin iniciou reset de senha | `users:reset_password` | user | `users:user:write` | `canRead(user)` + tenant | user + admin | 1 | — |
| `user.invite_cancelled` | Admin cancelou convite pendente | `users:cancel_invite` | user | `users:user:delete` | `canRead(user)` + tenant | admin | 0 | — |

### Payload dos novos eventos

**`user.password_reset`**

```json
{
  "user_id": "uuid",
  "reset_by": "uuid",
  "timestamp": "ISO8601"
}
```

- **sensitivity_level:** 1 (contém informação de ação administrativa sobre credenciais)
- **dedupe_key:** `{user_id}:password_reset:{timestamp}`

**`user.invite_cancelled`**

```json
{
  "user_id": "uuid",
  "cancelled_by": "uuid",
  "timestamp": "ISO8601"
}
```

- **sensitivity_level:** 0 (sem dados sensíveis)
- **dedupe_key:** `{user_id}:invite_cancelled:{timestamp}`

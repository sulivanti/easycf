## Impacto nos Pilares

- **UX (este pilar):** Atualização da jornada UX-006 com layout visual completo para criação e edição de roles. Novos componentes: BackLink, ScopeChip, ScopeGrid, AddScopeRow, InfoBox, PageHeaderWithBadge, RoleFormPage.
- **FR:** `PUT /api/v1/roles/:id` (FR-007) não aceita `status` no body — necessário amendment FR-000-M05 para adicionar campo opcional `status: 'ACTIVE' | 'INACTIVE'`.
- **INT:** Schema `UpdateRoleRequest` no OpenAPI v1.yaml precisa do campo `status` — amendment INT-000-M04.
- **DATA:** Sem impacto — coluna `roles.status` já existe com CHECK constraint (`ACTIVE` | `INACTIVE`).
- **SEC:** Sem impacto — scopes `users:role:read` e `users:role:write` já existem.
- **Manifests:** O manifest `ux-role-001.roles-list.yaml` descreve a listagem, não o formulário. Impacto informacional.
- **COD:** O `RolesPage.tsx` atual usa form inline — o codegen deverá criar `RoleFormPage.tsx` com rotas dedicadas e todos os componentes do layout visual.

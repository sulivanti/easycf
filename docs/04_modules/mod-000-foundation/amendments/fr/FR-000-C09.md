# Emenda: FR-000-C09

- **Documento base:** [FR-000](../../requirements/fr/FR-000.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-30
- **owner:** arquitetura
- **Motivação:** O `seed-admin.ts` aborta inteiramente quando o admin já existe (`if existing → return`), impedindo que novos scopes adicionados ao catálogo sejam propagados para a role `super-admin`. Isso causava o admin não ver `/filiais` (faltavam `tenants:branch:read/write`) e qualquer outro scope adicionado após o primeiro seed.
- **rastreia_para:** FR-000-C01, DOC-FND-000 §2.2, DATA-000-C03

---

## Detalhamento

### Problema

Quando novos scopes são adicionados ao array `SCOPES` em `seed-admin.ts` (alinhado com DOC-FND-000 §2.2), a re-execução do seed não tem efeito — o script detecta que o admin existe e retorna sem sincronizar permissões. Isso é uma regressão silenciosa: o catálogo cresce mas a role `super-admin` fica congelada no estado do primeiro seed.

### Correção

Nova função `syncSuperAdminPermissions(db)` chamada quando o admin já existe:

1. Busca a role `super-admin` pelo campo `codigo`
2. Se role não existe → cria role + todas as permissões do catálogo
3. Se role existe → calcula diff (`SCOPES` - permissões existentes)
4. Insere apenas os scopes faltantes
5. Loga quais scopes foram sincronizados

### Regras de negócio

- **RN-C09-01:** O seed DEVE sincronizar permissões da role `super-admin` mesmo quando o admin já existe
- **RN-C09-02:** Apenas scopes faltantes são inseridos (operação aditiva, nunca remove)
- **RN-C09-03:** O log DEVE listar os scopes adicionados para rastreabilidade

### Cenários Gherkin

```gherkin
Funcionalidade: Sync de permissões no re-seed

  Cenário: Admin existe e faltam scopes
    Dado que o admin "admin@ecf.local" já existe no banco
    E a role "super-admin" tem 13 scopes atribuídos
    E o catálogo SCOPES contém 63 scopes
    Quando executo "npx tsx db/seed-admin.ts"
    Então 50 novos scopes são inseridos em role_permissions
    E o log contém "Sincronizados 50 novos scopes para super-admin"

  Cenário: Admin existe e todos os scopes já estão sincronizados
    Dado que o admin "admin@ecf.local" já existe no banco
    E a role "super-admin" já tem todos os 63 scopes do catálogo
    Quando executo "npx tsx db/seed-admin.ts"
    Então nenhum insert é feito
    E o log contém "Todas as permissões já estão sincronizadas"

  Cenário: Admin existe mas role super-admin foi removida
    Dado que o admin "admin@ecf.local" já existe no banco
    E a role "super-admin" não existe no banco
    Quando executo "npx tsx db/seed-admin.ts"
    Então a role "super-admin" é criada com 63 permissões
    E o log contém "Role super-admin não encontrada. Criando..."
```

# Emenda: INT-005-C01

- **Documento base:** [INT-005](../../requirements/int/INT-005.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-30
- **owner:** Marcos Sulivan
- **Motivação:** A rota `POST /admin/cycles/:cid/macro-stages` define `schema.params` como JSON Schema literal em vez de Zod schema. O `validatorCompiler` do `fastify-type-provider-zod` chama `.safeParse()` no objeto recebido, que falha com `TypeError` porque o objeto plain não possui esse método. Resultado: 400 Bad Request impedindo criação de macroetapas (e por consequência, o primeiro estágio via duplo clique no canvas vazio).
- **rastreia_para:** spec-fix-macro-stages-params-schema-mismatch, FR-005, UX-005-M01

---

## Detalhamento

### 1. Problema

No arquivo `apps/api/src/modules/process-modeling/presentation/routes/cycles.route.ts`, a rota POST `/cycles/:cid/macro-stages` (linha ~444) define o `schema.params` como:

```typescript
params: {
  type: 'object' as const,
  properties: { cid: { type: 'string' as const, format: 'uuid' } },
  required: ['cid'],
},
```

Todas as outras 11 rotas no mesmo arquivo usam Zod schemas (ex: `params: cycleIdParam`). Esta é a única rota com JSON Schema literal.

### 2. Correção

Adicionar um novo Zod schema `cidParam` no DTO e usá-lo na rota:

**Em `apps/api/src/modules/process-modeling/presentation/dtos/process-modeling.dto.ts`:**

```typescript
// Adicionar junto aos outros param schemas (linha ~19)
export const cidParam = z.object({ cid: z.string().uuid() });
```

**Em `apps/api/src/modules/process-modeling/presentation/routes/cycles.route.ts`:**

```typescript
// ANTES (JSON Schema literal — causa 400):
params: {
  type: 'object' as const,
  properties: { cid: { type: 'string' as const, format: 'uuid' } },
  required: ['cid'],
},

// DEPOIS (Zod schema — funciona com validatorCompiler):
params: cidParam,
```

E atualizar o type generic da rota:

```typescript
// ANTES:
app.post<{ Params: { cid: string }; Body: z.infer<typeof createMacroStageBody> }>(

// DEPOIS:
app.post<{ Params: z.infer<typeof cidParam>; Body: z.infer<typeof createMacroStageBody> }>(
```

### 3. Nota na seção de contratos do INT-005

Adicionar nota ao endpoint `POST /admin/cycles/:cid/macro-stages`:

> **Nota de implementação:** O param `:cid` DEVE usar Zod schema (`cidParam`) para compatibilidade com `fastify-type-provider-zod`. JSON Schema literals causam `TypeError` no `validatorCompiler`.

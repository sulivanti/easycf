## ADR-001 — Abordagem UX-First e Delegação Auth para o MOD-000

- **Data:** 2026-03-08
- **Status:** Ratificado
- **Contexto:** Necessidade arquitetural de derivar layouts complexos num módulo separado, sem replicar estruturas de autenticação no Backoffice e seguindo os diagramas de Screen Manifests.
- **Opções consideradas:**
  1. Integrar front end completo e novo backend na mesma pasta do MOD-001.
  2. Implementar a interface via Server-Side Rendering dentro da fundação.
  3. Abordagem UX-First orientada à Manifestos (Venceu). O MOD-001 trata exclusivamente os Screen Manifests que invocarão, através dos `operationIds`, o motor centralizado provisionado originalmente pelo épico US-MOD-000.
- **Decisão:** Escolhemos a Opção 3.
- **Consequências (Prós/Contras):**
  - **Prós:** Alto reuso; divisão nítida de responsabilidades; UX tem autonomia para prototipar telas com rastreabilidade pré-validada.
  - **Contras:** Multiplicação de acoplamentos normativos — os desenvolvedores de front-end devem consultar a interface de API do MOD-000.

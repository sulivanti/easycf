## NFR-001 — Requisitos Não-Funcionais e Performance (Backoffice Admin)

### 1. Desempenho (Performance)

1. **LCP (Largest Contentful Paint):** < 1.5s no 95º percentil (App Shell estático deve carregar imediatamente, buscando os dados asincronamente).
2. **FID (First Input Delay):** < 100ms.
3. **Respostas da API:** Tempo de resposta do BFF deve ser inferior a 200ms na mediana.

### 2. Confiabilidade e Tolerância a Falhas

1. Módulo deve falhar suavemente; se o dashboard estiver em downtime, apenas o componente do dashboard exibirá `Error Boundary`.

### 3. Escalabilidade

1. Arquitetura SPA (Single Page Application) estática; front-end deve ser cacheado globalmente em CDN.

### 4. Manutenibilidade e Logs

1. Adoção obrigatória do DOC-PADRAO-001 de Logging com levels `info` e `error` propagando para centralizador de logs.

# FR-000 — Funcionalidades de Autenticação e Sessão

> | Versão | Data       | Responsável | Status/Integração |
> |--------|------------|-------------|-------------------|
> | 0.1.0  | 2026-03-05 | arquitetura | Baseline Inicial (scaffold-module via US-MOD-1&2) |

- **Descrição:** Provisiona endpoints para login e controle de sessão de usuários, permitindo iniciar acesso e expurgar acessos remotos suspeitos via Kill-Switch e garantir proteção por expiração.
- **Prioridade:** Must (Acesso central do sistema).
- **Perfis impactados:** Qualquer usuário não-autenticado ou autenticado tentando gerenciar acessos.
- **Done funcional:** A API `POST /api/v1/auth/login` emite tokens e log de auditoria. A API de Kill-Switch revoga de fato as entradas de `auth_sessions` na Base de Dados e o Middleware bloqueia ativamente sem depender do JWT Expirator Mode.
- **Efeito colateral?** [x] Sim [] Não *(Idempotência por Sessão/Token — Gerenciamento de estado de credenciais).*
- **Dependências (IDs):** BR-000 | DATA-000 | SEC-000
- **Gatilhos ADR-XXX (Opcional):**
  - Autenticação e revogação de estado `MUST` consultar o banco (`auth_sessions`) contra os IDs presentes no JWT.

### Endpoints / Contratos UI Mapeados

1. **POST `/api/v1/auth/login`**: Valida Email/Senha, suporta `remember_me` (12h ou 30d). Audita auth.login.success/failure/blocked. Desvia para MFA se necessário. Setar Token via Cookies.
2. **POST `/api/v1/auth/logout`**: Revoga `is_revoked=true` do cookie atrelado atual e expurga os headers client.
3. **GET `/api/v1/auth/sessions`**: Relatório de dispositivos (`device_fp`) com sessão ATIVA aberta para o usuário listado.
4. **DELETE `/api/v1/auth/sessions/:sessionId`**: Revoga (`is_revoked=true`) uma sessão alheia (Ex: Computador logado do Escritório sendo revogado pelo Celular).
5. **DELETE `/api/v1/auth/sessions`**: Kill-Switch Global. `UPDATE auth_sessions set is_revoked=true WHERE user_id = :id`.
6. **POST `/api/v1/auth/refresh`**: Verifica o RefreshToken assinado e fornece um novo Access Token via Cookie, desde que `is_revoked=false`.
7. **Página UI:** Criação da `LoginPage.tsx` (frontend) para consumo dos campos e-mail, senha e toggle de "remember_me", inclusive a geração do footprint do device (device_fp).

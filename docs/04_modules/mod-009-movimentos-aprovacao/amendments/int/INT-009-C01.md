### Causa raiz

Todos os handlers fazem `reply.send(result)` passando diretamente o output do use case (camelCase) sem mapear para o schema Zod (snake_case).

### Correção

Adicionar mapeamento explícito em cada handler, extraindo campos do domain object e renomeando para snake_case conforme o DTO Zod. Padrão de referência: endpoints já corrigidos do MOD-003 (GET /:id, POST /, etc.).

### Mapeamentos principais

- `ControlledMovementProps`: `tenantId`→`tenant_id`, `controlRuleId`→`control_rule_id`, `requesterId`→`requester_id`, `requesterOrigin`→`requester_origin`, `objectType`→`object_type`, `objectId`→`object_id`, `operationType`→`operation_type`, `operationPayload`→`operation_payload`, `caseId`→`case_id`, `currentLevel`→`current_level`, `totalLevels`→`total_levels`, `idempotencyKey`→`idempotency_key`, `createdAt`→`created_at`, `updatedAt`→`updated_at`
- `ApprovalInstanceProps`: `movementId`→`movement_id`, `approverId`→`approver_id`, `decidedAt`→`decided_at`, `timeoutAt`→`timeout_at`, `createdAt`→`created_at`
- `MovementControlRuleProps`: `tenantId`→`tenant_id`, `objectType`→`object_type`, `operationType`→`operation_type`, `originTypes`→`origin_types`, `criteriaType`→`criteria_type`, `valueThreshold`→`value_threshold`, `validFrom`→`valid_from`, `validUntil`→`valid_until`, `createdAt`→`created_at`, `updatedAt`→`updated_at`
- `ApprovalRuleProps`: `tenantId`→`tenant_id`, `controlRuleId`→`control_rule_id`, `approverType`→`approver_type`, `approverValue`→`approver_value`, `requiredScope`→`required_scope`, `allowSelfApprove`→`allow_self_approve`, `timeoutMinutes`→`timeout_minutes`, `escalationRuleId`→`escalation_rule_id`, `createdAt`→`created_at`, `updatedAt`→`updated_at`
- `PaginatedResult`: `pageSize`→`page_size`, calcular `total_pages` = `Math.ceil(total / pageSize)`
- `MovementHistoryEntry`: `eventType`→`event_type`, `actorId`→`actor_id`, `correlationId`→`correlation_id`, `createdAt`→`created_at`

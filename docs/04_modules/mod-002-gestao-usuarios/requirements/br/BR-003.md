# BR-003 — Modos de Criação Mutuamente Exclusivos

- **Regra:** O formulário de criação de usuário possui dois modos mutuamente exclusivos: "Enviar Convite" (padrão) e "Senha Temporária". O modo padrão SEMPRE é "Enviar Convite". A troca de modo DEVE resetar todo o formulário (campos e erros de validação). Campos de senha só são visíveis no modo "Senha Temporária".
- **Exemplo:**
  - Admin abre /usuarios/novo → aba "Enviar Convite" selecionada, campos de senha ocultos
  - Admin preenche nome e e-mail, troca para "Senha Temporária" → todos os campos são limpos, campos de senha aparecem
  - Admin volta para "Enviar Convite" → campos de senha desaparecem, formulário é limpo novamente
- **Exceções:** Nenhuma. Reset é absoluto — não há "preservar campos comuns" entre modos.
- **Impacto:** fluxo, estado

### Payloads por Modo

| Modo | Payload POST /api/v1/users |
|---|---|
| Enviar Convite | `{ fullName, email, roleId, mode: "invite" }` |
| Senha Temporária | `{ fullName, email, roleId, password, forcePasswordReset: true }` |

# Emenda: FR-000-C01

- **Documento base:** [FR-000](../../requirements/fr/FR-000.md)
- **estado_item:** MERGED
- **status_implementacao:** IMPLEMENTADO
- **Natureza:** C (Correção)
- **Data:** 2026-03-25
- **owner:** arquitetura
- **Motivação:** O seed script (`db/seed-admin.ts`) atribui scopes ao role `super-admin` que não correspondem ao catálogo canônico de DOC-FND-000 §2.2. Especificamente: (1) usa `tenants:tenant:read/write` mas o catálogo define `tenants:branch:read/write`, (2) faltam 5 scopes MOD-000 (`system:audit:read`, `system:audit:sensitive`, `users:user:import`, `users:user:export`, `users:user:comment`, `storage:file:upload`, `storage:file:read`). Resultado: sidebar não mostra "Filiais" nem "Auditoria" porque os scopes do JWT não batem com os `requiredScope` do sidebar-config.
- **rastreia_para:** DOC-FND-000 §2.2, DOC-PADRAO-001 §4.4, DOC-PADRAO-001-C01, UX-SHELL-001 sidebar

## Detalhamento

### Correção: Alinhar scopes do seed com catálogo canônico

**Scopes atuais no seed (incorretos):**

```typescript
const SCOPES = [
  'users:user:read',
  'users:user:write',
  'users:user:delete',
  'users:role:read',
  'users:role:write',
  'tenants:tenant:read',    // ❌ Não existe no catálogo
  'tenants:tenant:write',   // ❌ Não existe no catálogo
  'org:unit:read',
  'org:unit:write',
  // ... demais scopes de outros módulos
  // ❌ Faltam: system:audit:*, users:user:import/export/comment, storage:file:*
];
```

**Scopes corrigidos (conforme DOC-FND-000 §2.2 — apenas MOD-000):**

```typescript
// Scopes MOD-000 do catálogo canônico (DOC-FND-000 §2.2)
'users:user:read',
'users:user:write',
'users:user:delete',
'users:user:import',       // ✅ Adicionado
'users:user:export',       // ✅ Adicionado
'users:user:comment',      // ✅ Adicionado
'users:role:read',
'users:role:write',
'tenants:branch:read',     // ✅ Corrigido (era tenants:tenant:read)
'tenants:branch:write',    // ✅ Corrigido (era tenants:tenant:write)
'system:audit:read',       // ✅ Adicionado
'system:audit:sensitive',  // ✅ Adicionado
'storage:file:upload',     // ✅ Adicionado
'storage:file:read',       // ✅ Adicionado
```

**Scopes de outros módulos** (já presentes no seed, manter):
- `org:unit:read/write` (MOD-003)
- `process:cycle:read/write` (MOD-005)
- `case:instance:read/write` (MOD-006)
- `params:routine:read/write` (MOD-008)
- `integration:service:read/write` (MOD-007)
- `movement:rule:read/write`, `movement:approval:read/write` (MOD-009)
- `mcp:agent:read/write` (MOD-010)
- `admin:backoffice:read/write` (MOD-001)

### Regra

O seed script DEVE usar **exatamente** os nomes de scopes do catálogo canônico (DOC-FND-000 §2.2). Nomes inventados ou abreviados (ex: `tenants:tenant:*` no lugar de `tenants:branch:*`) causam mismatch silencioso com os `requiredScope` dos componentes de UI.

### Impacto visível

Após esta correção, os itens "Filiais" e "Auditoria" da sidebar passarão a aparecer para o admin, pois seus `requiredScope` (`tenants:branch:read` e `system:audit:read`) serão satisfeitos pelo JWT.

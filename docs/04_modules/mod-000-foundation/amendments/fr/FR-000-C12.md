- **Motivação:** O `drizzle-repositories.ts:573` (`mapToRole`) chama `Scope.create(p.scope)` em scopes lidos do banco. Se o banco contém dados legados com hífen (ex: `mcp:agent:phase2-enable`), a operação inteira falha com `DomainValidationError`. Nenhuma das 7 correções anteriores (C01–C11) resolveu este ponto porque: (1) não há migration SQL que corrija o dado automaticamente, e (2) o repository não tem fallback para dados inválidos.

## Detalhamento

### Correção 1: Fallback resiliente no `mapToRole`

**Arquivo:** `apps/api/src/modules/foundation/infrastructure/drizzle-repositories.ts`
**Linha:** 573

```typescript
// ANTES (crash na operação inteira):
scopes: perms.map((p) => Scope.create(p.scope)),

// DEPOIS (degradação graceful com warning):
scopes: (() => {
  const valid: Scope[] = [];
  for (const p of perms) {
    try {
      valid.push(Scope.create(p.scope));
    } catch {
      console.warn(
        `[RoleRepository] Scope inválido ignorado: "${p.scope}" (role: ${row.codigo}). Execute migration para corrigir.`,
      );
    }
  }
  return valid;
})(),
```

**Justificativa:** O fallback garante que a aplicação não crasha ao ler roles com dados legados. Loga WARNING para diagnóstico. Temporário — pode ser removido após confirmar que a migration rodou em todos os ambientes.

### Correção 2: Anotação em FR-007 (Roles CRUD)

Adicionar nota no FR-007 (se existente) ou em seção de observações do FR-000:

> **Resiliência de leitura (FR-000-C12):** Ao carregar roles do banco, scopes que falham na validação `Scope.create()` são ignorados com log WARNING. Isso previne que dados legados (scopes com hífens) quebrem operações de leitura.

### Acceptance Criteria

- **AC-C12-001**: App inicia e carrega roles sem erro, mesmo se `role_permissions` contém scopes com hífens.
- **AC-C12-002**: Scopes inválidos são excluídos da role carregada (degradação, não crash).
- **AC-C12-003**: WARNING é logado com o scope inválido e o código da role para diagnóstico.
- **AC-C12-004**: Scopes válidos continuam sendo carregados normalmente.

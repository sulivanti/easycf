## 2. UX-MCP-001 вҖ” Gestao de Agentes e Acoes

### 2.1 Jornada Principal (Happy Path)

```
1. Admin acessa /automacao/agentes
2. GET /admin/mcp-agents carrega lista de agentes (painel "Agentes" ativo por padrao)
3. Admin cria novo agente via drawer (nome, codigo, owner, scopes)
4. POST /admin/mcp-agents retorna 201 com api_key
5. Modal ApiKeyModal exibe key UMA VEZ вҖ” admin deve copiar e confirmar checkbox
6. Agente aparece na tabela com status ACTIVE
7. Admin navega para painel "Catalogo de Acoes" вҖ” cria/edita acoes
8. Admin navega para painel "Permissoes" вҖ” vincula acoes a agentes via checkboxes
```

### 2.2 Estados da Tela

| Estado | Comportamento | Componente |
|---|---|---|
| **loading** | Skeleton de tabela com 5 linhas placeholder | Skeleton rows |
| **loaded** | Tabela populada com agentes/acoes/matriz conforme painel ativo | AgentsTable / ActionsCatalog / PermissionsMatrix |
| **empty** | Ilustracao + "Nenhum agente cadastrado. Crie o primeiro agente MCP." | EmptyState com botao "Criar Agente" |
| **error** | Toast com mensagem RFC 9457 + correlationId copiavel | Toast de erro |
| **saving** | Botao de submit desabilitado + spinner inline | Button loading state |

### 2.5 Alternativas e Erros

| Situacao | Comportamento UX |
|---|---|
| Criar agente com escopo bloqueado | Inline error no campo scopes: "Escopo bloqueado para agentes MCP: {scope} (corresponde ao padrao {pattern})" (422) |
| Criar agente com codigo duplicado | Inline error: "Ja existe um agente com este codigo neste tenant." (409) |
| Revogar agente вҖ” confirmacao | Modal: "Revogar o agente '{nome}'? Esta acao e irreversivel." + campo obrigatorio "Motivo da revogacao" |
| Revogar agente REVOKED | Botao "Revogar" desabilitado (badge "REVOGADO") |
| Rotacionar key вҖ” confirmacao | Modal aviso amarelo: "A chave atual sera invalidada imediatamente. O agente ficara inoperante ate a nova chave ser configurada. Continuar?" |
| ApiKeyModal вҖ” tentar fechar sem checkbox | Botao "Fechar" desabilitado. Tooltip: "Confirme que copiou a chave antes de fechar." |
| ApiKeyModal вҖ” navegacao acidental | `beforeunload` warning: "A chave nao sera exibida novamente. Voce copiou a chave?" |
| Editar agente REVOKED | Formulario desabilitado. Banner: "Agente revogado вҖ” somente leitura." |
| Admin sem scope mcp:agent:revoke | Botao "Revogar" oculto (guard de scope no manifest) |
| Admin sem scope mcp:action:write | Painel "Catalogo de Acoes" somente-leitura (botao "Criar Acao" oculto) |
| Erro de rede generico | Toast: "Erro de conexao. Verifique sua rede e tente novamente." + correlationId |


```gherkin
Funcionalidade: Gestao de Agentes MCP (UX-MCP-001)

Cenario: Criar agente e visualizar API key uma unica vez
  Dado que o admin tem scopes "mcp:agent:write" e "mcp:agent:read"
  E o admin esta na tela /automacao/agentes
  Quando o admin clica em "Criar Agente"
  E preenche codigo="AGENT-01", nome="Agente Comercial", owner, scopes=["process:case:read"]

  Entao modal ApiKeyModal abre com a chave de API
  E o botao "Fechar" esta desabilitado
  Quando o admin clica em "Copiar"
  E marca o checkbox "Copiei e armazenei a chave com seguranca"
  Entao o botao "Fechar" fica habilitado
  Quando o admin clica em "Fechar"
  Entao o modal fecha e a chave e eliminada da memoria
  E o agente aparece na tabela com status ACTIVE

Cenario: Revogar agente com motivo obrigatorio
  Dado que o admin tem scope "mcp:agent:revoke"
  E existe o agente "AGENT-01" com status ACTIVE
  Quando o admin clica em "Revogar" no agente "AGENT-01"
  Entao modal RevokeModal abre com campo "Motivo"
  Quando o admin digita motivo com menos de 10 caracteres
  Entao botao "Revogar definitivamente" permanece desabilitado
  Quando o admin digita motivo com 10+ caracteres e clica "Revogar definitivamente"
  Entao agente muda para status REVOKED com badge vermelho
  E linha desabilitada com opcoes de edicao bloqueadas

Cenario: Rotacionar API key com aviso de invalidacao
  Dado que o admin tem scope "mcp:agent:write"
  E existe o agente "AGENT-01" com status ACTIVE
  Quando o admin clica em "Rotacionar Key"
  Entao modal RotateKeyModal abre com aviso amarelo
  Quando o admin confirma
  Entao ApiKeyModal abre com a nova chave
  E a chave anterior esta imediatamente invalidada

Cenario: Admin sem scope de revogacao nao ve botao
  Dado que o admin tem scope "mcp:agent:write" mas NAO tem "mcp:agent:revoke"
  Quando o admin acessa /automacao/agentes
  Entao botao "Revogar" esta oculto em todos os agentes

  E existem execucoes nas ultimas 24h

  Quando o admin filtra por status="BLOCKED"
  Entao tabela exibe apenas execucoes bloqueadas
```

## Detalhamento

### Migration SQL: Renomear scopes com hífens

**Tipo:** Migration Drizzle (custom SQL)
**Tabela afetada:** `role_permissions`
**Operação:** Renomear TODOS os scopes que contêm hífens, substituindo por underscores.

```sql
-- Migration: fix_scope_hyphens
-- Idempotent: safe to run multiple times (no-op if no hyphens exist)
UPDATE role_permissions
SET scope = REPLACE(scope, '-', '_')
WHERE scope LIKE '%-%';
```

### Características

| Aspecto | Valor |
|---------|-------|
| **Idempotência** | Sim — se não há hífens, nenhuma linha é afetada |
| **Reversibilidade** | Não automaticamente (underscores não podem ser distinguidos dos originais) |
| **Escopo** | Todas as roles, não apenas super-admin |
| **CHECK constraint** | Já está na regex canônica (FR-000-C10) — migration alinha dados com a constraint |

### Relação com RENAMES no seed

O RENAMES em `seed-admin.ts:122-124` pode ser **mantido** como fallback para cenários de restore de backup, mas a migration é a solução primária:

| Mecanismo | Quando roda | Cobertura |
|-----------|-------------|-----------|
| **Migration SQL** (esta) | Deploy / `drizzle-kit migrate` | Todas as roles, automático |
| **RENAMES no seed** | `pnpm db:seed` manual | Todas as roles, manual |
| **CHECK constraint** | INSERT/UPDATE | Previne novos dados inválidos |

### Acceptance Criteria

- **AC-C04-001**: Após executar a migration, `SELECT scope FROM role_permissions WHERE scope LIKE '%-%'` retorna 0 linhas.
- **AC-C04-002**: Migration é idempotente — executar N vezes produz o mesmo resultado.
- **AC-C04-003**: Migration NÃO altera scopes que já usam underscores.
- **AC-C04-004**: Migration é rastreável no histórico de migrations do Drizzle.

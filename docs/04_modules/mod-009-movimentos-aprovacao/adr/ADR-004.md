# ADR-004 — Override Auditado com Justificativa Mínima 20 Caracteres

## Contexto

O MOD-009 permite override de movimentos pendentes por gestores com scope `approval:override`. Override é uma operação excepcional que contorna a cadeia de aprovação normal — o movimento é aprovado diretamente, sem decisão dos aprovadores designados. Para compliance, o override deve ser auditado de forma imutável.

A decisão envolve:

1. Qual o comprimento mínimo da justificativa obrigatória.
2. Onde e como registrar o override para garantir imutabilidade.

Requisitos de contexto:

- Override é operação de alto risco — compliance regulatório exige rastreabilidade (OKR-2: 100% auditado).
- A justificativa deve ser substantiva — "ok", "urgente", "aprovado" não são justificativas aceitáveis para auditoria.
- O registro de override deve ser imutável — não pode ser alterado ou excluído após criação.

## Decisão

**Justificativa de override obrigatória com mínimo de 20 caracteres (max 2000). Registro em tabela append-only `movement_override_log` (imutável — apenas INSERT, sem UPDATE/DELETE).**

O valor de 20 caracteres foi escolhido como ponto de equilíbrio:

- **Abaixo de 20:** Permite justificativas vazias semanticamente ("aprovado pelo chefe", "ok do diretor") que não agregam valor para auditoria.
- **Acima de 20:** Força o gestor a fornecer contexto mínimo ("Urgência operacional aprovada pela diretoria em reunião extraordinária" = 73 chars).
- **Comparativo:** Campos de justificativa em sistemas regulados (BACEN, SOX) tipicamente exigem 20-50 caracteres mínimos.

Fluxo completo (SEC-009 §3.4):

1. Verificar scope `approval:override` no JWT
2. Validar justificativa: min 20 chars, max 2000 chars, não whitespace-only
3. INSERT em `movement_override_log` (append-only): `overridden_by`, `overridden_at`, `justificativa`, `scope_used`
4. UPDATE `controlled_movements.status` → `OVERRIDDEN`
5. INSERT em `movement_history` com `event_type=OVERRIDDEN`
6. Emitir `movement.overridden` (DATA-003 evento 10) via outbox
7. Notificar solicitante + admin + compliance

Frontend implementa dupla confirmação: checkbox "Confirmo que estou aplicando override" + campo justificativa (UX-APROV-001, OverrideModal).

## Alternativas Consideradas

1. **Sem mínimo de caracteres:** Permitir qualquer texto não-vazio. Permite justificativas como "ok" ou "sim" que não servem para auditoria. Em auditoria regulatória, justificativas vazias de conteúdo são equivalentes a ausência de justificativa.

2. **Mínimo de 50 caracteres:** Mais restritivo, forçaria justificativas mais detalhadas. Pode gerar frustração em cenários legítimos de urgência onde o gestor precisa agir rápido. Risco de padding com texto sem sentido ("aaaaaaaaaaaaaaaa...").

3. **Mínimo de 10 caracteres:** Muito permissivo — "urgente ok" (10 chars) não é justificativa aceitável.

4. **Campos estruturados em vez de texto livre:** Dropdown de motivos pré-definidos + campo complementar. Reduz liberdade do gestor; motivos pré-definidos nunca cobrem todos os cenários reais. Pode parecer artificial ("Motivo: Outros | Complemento: ver email").

5. **Registro de override na mesma tabela de history:** Usar apenas `movement_history` para registrar overrides. Menos isolamento — `movement_history` aceita vários tipos de evento, enquanto `movement_override_log` é uma tabela de compliance dedicada com retenção indefinida (SEC-009 §5). Separação permite políticas de acesso distintas (`approval:override` ou `system:audit:read` para override_log vs. `approval:movement:read` para history).

## Consequências

- **Positivas:** Auditoria completa e imutável em tabela dedicada; justificativa substantiva obrigatória; retenção indefinida para compliance (SEC-002); acesso restrito (somente `approval:override` ou `system:audit:read`); dupla confirmação no frontend previne overrides acidentais.
- **Negativas:** Overhead de UX para gestores em cenários de urgência (2 campos: justificativa + checkbox). Justificativas de 20+ chars podem ainda ser semanticamente vazias (ex: "aprovado com urgência ok" = 25 chars mas pouco informativo). Mitigação: relatório de compliance pode flaggar justificativas suspeitas por tamanho próximo ao mínimo.
- **Riscos:** Gestores podem desenvolver "justificativas template" que passam no mínimo mas não agregam valor. Mitigação: auditoria periódica de overrides com análise de justificativas.

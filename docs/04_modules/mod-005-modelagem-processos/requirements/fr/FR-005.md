# FR-005 — Requisitos Funcionais da Modelagem de Processos

---

## FR-001 — CRUD de Ciclos de Processo

- **Goal:** Permitir criar, listar, visualizar, editar e excluir ciclos de processo com máquina de estados (DRAFT → PUBLISHED → DEPRECATED).
- **Done funcional:**
  - POST /admin/cycles cria ciclo em DRAFT com version=1
  - GET /admin/cycles lista com paginação cursor-based
  - GET /admin/cycles/:id retorna detalhe
  - PATCH /admin/cycles/:id edita (apenas DRAFT — BR-001)
  - DELETE /admin/cycles/:id soft-delete (apenas DRAFT — BR-005)
- **Dependências:** BR-001, BR-005, BR-006, BR-010, DATA-005 (process_cycles), INT-005 §1.1
- **Idempotency:** n/a (CRUD padrão)
- **Timeline/Notifications:** yes — emite `process.cycle_created` (DATA-003)

- **estado_item:** DRAFT
- **owner:** Marcos Sulivan
- **data_ultima_revisao:** 2026-03-17

## FR-002 — Publicação de Ciclo

- **Goal:** Promover ciclo DRAFT para PUBLISHED após validação de pré-condições.
- **Done funcional:**
  - POST /admin/cycles/:id/publish valida ao menos 1 estágio com is_initial=true (BR-003)
  - Define published_at=now() e status=PUBLISHED
  - Ciclo torna-se imutável (BR-001)
  - Aceita instâncias no MOD-006
- **Dependências:** BR-001, BR-002, BR-003, DATA-005, INT-005 §1.1
- **Idempotency:** required — publicação duplicada deve retornar 422 se já PUBLISHED (sem side-effect)
- **Timeline/Notifications:** yes — emite `process.cycle_published`, notifica admin (SEC-EventMatrix)

- **estado_item:** DRAFT
- **owner:** Marcos Sulivan
- **data_ultima_revisao:** 2026-03-17

## FR-003 — Fork de Ciclo Publicado

- **Goal:** Criar nova versão DRAFT a partir de ciclo PUBLISHED, copiando toda a estrutura.
- **Done funcional:**
  - POST /admin/cycles/:id/fork cria novo ciclo DRAFT com parent_cycle_id e version+1
  - Copia atomicamente: macroetapas, estágios, gates, role_links, transições (BR-004)
  - Novos UUIDs para todos os elementos; referências internas remapeadas
  - Retorna 201 com novo ciclo
- **Dependências:** BR-004, DATA-005 (7 tabelas), INT-005 §1.1
- **Idempotency:** required — fork duplicado (mesmo request) deve gerar apenas 1 ciclo (Idempotency-Key)
- **Timeline/Notifications:** yes — emite `process.cycle_forked` (DATA-003)

- **estado_item:** DRAFT

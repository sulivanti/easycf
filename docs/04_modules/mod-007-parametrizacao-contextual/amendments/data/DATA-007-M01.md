### Alteracoes neste pilar

#### A1 — Novo campo `incidence_type` em `incidence_rules` (E-005)

Adicionar coluna `incidence_type` a tabela `incidence_rules`:

| Campo | Tipo | Constraint | Obrigatorio | Descricao |
|---|---|---|---|---|
| `incidence_type` | varchar(10) | NOT NULL DEFAULT 'OBR' | SIM | Tipo de incidencia: OBR (obrigatorio), OPC (opcional), AUTO (auto-apply) |

**Constraints adicionais:**

- CHECK: `incidence_type IN ('OBR', 'OPC', 'AUTO')`

**Index adicional:**

- `idx_incidence_type` (incidence_type) — filtro por tipo na matriz UI

#### A2 — Semantica da celula da matriz

A mudanca semantica e a seguinte:

| Aspecto | Antes (v1) | Depois (v2) |
|---|---|---|
| Celula da matriz | Regra existe (ACTIVE badge) ou nao (INACTIVE badge / vazio) | Tipo de incidencia: OBR (roxo), OPC (teal), AUTO (ambar), EMPTY (cinza) |
| Status da regra | `status` (ACTIVE/INACTIVE) define se a regra esta ativa | `status` continua existindo para ativacao/desativacao; `incidence_type` define a natureza da incidencia |
| UNIQUE constraint | `(tenant_id, framer_id, target_object_id)` — inalterado | Inalterado — continua 1 regra por par enquadrador+objeto |

#### A3 — Migration

```sql
-- Migration: add_incidence_type_to_incidence_rules
ALTER TABLE incidence_rules
  ADD COLUMN incidence_type varchar(10) NOT NULL DEFAULT 'OBR';

ALTER TABLE incidence_rules
  ADD CONSTRAINT chk_incidence_type CHECK (incidence_type IN ('OBR', 'OPC', 'AUTO'));

CREATE INDEX idx_incidence_type ON incidence_rules (incidence_type);
```

**Dados existentes:** Todos os registros existentes receberao `incidence_type = 'OBR'` (default). Isso preserva o comportamento anterior — regras existentes sao tratadas como obrigatorias.

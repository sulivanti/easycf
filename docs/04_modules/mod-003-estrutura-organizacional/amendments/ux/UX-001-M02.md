## Detalhamento

### M01 — Eliminacao do FormPanel para edicao (UX-ORG-002 modo editar)

O FormPanel inline 480px (UX-001-M01 §D3) NAO DEVE mais ser usado para **edicao** de dados cadastrais de unidades organizacionais. O modo editar passa a ser inline no DetailPanel.

> **NOTA:** O FormPanel **continua valido** para **criacao** de novos nos ("+ Nova Subdivisao"), onde faz sentido ter um formulario separado com campos de codigo e pai.

**Componentes a remover (modo editar):**
- `OrgNodeForm` / `FormPanel` — nao mais necessario para edicao
- Pattern H (FormPanel substitui arvore) — eliminado para edicao

### M02 — Estado View do DetailPanel (padrao)

O PainelDetalhe em estado View permanece como definido em UX-001-M01 §D2, com as seguintes acoes no header:

```
DetailHeader
  Actions: [Editar Dados (secondary, icone pencil)] [+ Nova Subdivisao (primary, icone plus)]
```

Card "DADOS CADASTRAIS":
- Header: bg `#F8F8F6`, label `#888888`, texto "DADOS CADASTRAIS"
- Body: ReadOnlyFields em grid (conforme UX-001-M01 §D9)
  - Row: CNPJ (1fr) | Razao Social (2fr)
  - Row: Filial (1fr) | Responsavel (1fr)
  - Row: Telefone (1fr) | E-mail (1fr)

### M03 — Estado Edit (Inline Edit) do DetailPanel

Ao clicar "Editar Dados", o DetailPanel entra em modo Edit **in-place**:

```
DetailPanel (mesmo layout, arvore continua visivel)
  DetailHeader
    (info inalterada)
    Actions MUDAM: [Cancelar (secondary)] [Salvar Alteracoes (primary, icone check)]
  Card "DADOS CADASTRAIS — EDITANDO"
    Header: bg #E3F2FD, label #2E86C1, icone pencil
    Body: Inputs editaveis em grid (mesma grid do view)
      Row: Nome da Subdivisao * (input, full width)
      Row: CNPJ (input) | Razao Social (input, 2fr)
      Row: Filial (input) | Responsavel (input)
      Row: Telefone (input) | E-mail (input)
  Card "HIERARQUIA" (sempre read-only, so aparece em modo edit)
    Body: ReadOnlyField[lock] Unidade Pai | ReadOnlyField[lock] Nivel
  Card "DEPARTAMENTOS VINCULADOS" (inalterado)
  MetricRow (inalterado)
```

**Mudancas visuais no modo edit:**

| Botao header direito | "+ Nova Subdivisao" (primary) | "Salvar Alteracoes" (primary, icone check) |
| Card header bg | `#F8F8F6` | `#E3F2FD` |
| Card header label cor | `#888888` | `#2E86C1` |

| Arvore (TreePanel) | visivel | **visivel** (principal vantagem) |

### M04 — Componente InlineEditCard (NOVO)

Criar componente reutilizavel `InlineEditCard` com dois modos:

**Props:**
- `title: string` — texto do header (ex: "DADOS CADASTRAIS")
- `isEditing: boolean` — controla modo view/edit
- `children: ReactNode` — conteudo do card (ReadOnlyFields ou inputs)

**Modo View:**
- Header: bg `#F8F8F6`, label uppercase 10px 700 `#888888` ls:1px
- Body: padding 20px 24px

**Modo Edit:**
- Header: bg `#E3F2FD`, label uppercase 10px 700 `#2E86C1` ls:1px, icone pencil 14x14 `#2E86C1`
- Titulo sufixo: " — EDITANDO"
- Body: padding 20px 24px

**Inputs editaveis (dentro do card em modo edit):**
- Height: min 42px
- Border: 2px solid `#2E86C1`
- Border-radius: 8px
- Background: `#FFFFFF`
- Font: inherit, 14px, weight 500
- Placeholder: 14px 400 `#CCCCCC`
- Placeholders especificos: CNPJ "00.000.000/0000-00", Telefone "(00) 00000-0000", E-mail "contato@empresa.com"

### M05 — Componente HierarchyCard (NOVO)

Criar componente `HierarchyCard` exibido **somente em modo edit**:

```
Card "HIERARQUIA" (r:12, border #E8E8E6, mt:20px)
  Header: bg #F8F8F6, label "HIERARQUIA" uppercase
  Body: grid 2 colunas
    ReadOnlyField[lock] "UNIDADE PAI" — valor do parent_name
    ReadOnlyField[lock] "NIVEL" — valor formatado (ex: "N2 — Diretoria")
```

Usa o componente `ReadOnlyField` existente (UX-001-M01 §D9) com icone de cadeado.

### M06 — Transicoes de Estado e API

**Fluxo: View → Edit → Save/Cancel**

1. Clique "Editar Dados" → `setIsEditing(true)` → campos viram inputs, botoes mudam
2. Clique "Salvar Alteracoes" → `PATCH /api/v1/org-units/:id` com campos editaveis
   - Sucesso (200) → Toast "Unidade '{codigo} — {nome}' atualizada." + `setIsEditing(false)`
   - Erro → tratamento conforme UX-001 §Alternativas/Erros + UX-001-C01
3. Clique "Cancelar" → `setIsEditing(false)` + reverter campos ao estado original (sem chamada API)

**Campos editaveis (PATCH):** nome, cnpj, razao_social, filial, responsavel, telefone, email_contato
**Campos imutaveis (HierarchyCard):** codigo, nivel, parent_id

### M07 — Campo "Nome da Subdivisao" obrigatorio

No modo edit, o campo "Nome da Subdivisao" DEVE:
- Aparecer como primeiro campo, full width
- Ter asterisco vermelho (`#E74C3C`) indicando obrigatoriedade
- Validacao: required, max 200 chars

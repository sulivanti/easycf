# Emenda: INT-006-C01

- **Documento base:** [INT-006](../../requirements/int/INT-006.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-30
- **owner:** Marcos Sulivan
- **Motivação:** 4 endpoints em cases.route.ts enviam `reply.send(result)` sem mapeamento camelCase→snake_case. Use cases retornam objetos com camelCase mas DTOs Zod esperam snake_case. Fastify valida e retorna 500. Adicionalmente, os use cases de transition/resolve/waive retornam estruturas reduzidas que não contêm todos os campos esperados pelo DTO — o handler precisa construir a resposta a partir dos dados disponíveis.
- **rastreia_para:** INT-001-C01 (mesma categoria de correção no MOD-003)

---

## Detalhamento

### Endpoints afetados

| Método | Path | operationId | Linha | DTO Response |
|--------|------|-------------|-------|-------------|
| POST | /cases/:id/transition | `cases_transition` | 251 | `transitionResponse` |
| POST | /cases/:id/gates/:gateId/resolve | `cases_gates_resolve` | 453 | `resolveGateResponse` |
| POST | /cases/:id/gates/:gateId/waive | `cases_gates_waive` | 491 | `waiveGateResponse` |
| PATCH | /cases/:id/assignments/:aid | `cases_assignments_update` | 621 | `updateAssignmentResponse` |

### Causa raiz

1. **transition (linha 251):** Use case retorna `{ transitionId, fromStageId, toStageId, isTerminal, caseCompleted }`. DTO espera `{ id, codigo, current_stage_id, status, stage_history_entry, new_gate_instances }`. Mismatch estrutural — handler precisa recarregar caso pós-transição para obter `codigo` e `status`.

2. **resolve gate (linha 453):** Use case retorna `{ gateInstanceId, status, decision }`. DTO espera `{ id, gate_id, status, decision, resolved_by, resolved_at }`. Campos `gate_id`, `resolved_by`, `resolved_at` não estão no output do use case mas podem ser derivados dos inputs do handler.

3. **waive gate (linha 491):** Use case retorna `{ gateInstanceId, status }`. DTO espera `{ id, gate_id, status, resolved_by, resolved_at }`. Mesma situação do resolve.

4. **update assignment (linha 621):** Use case `assignResponsibleUseCase.update()` — o handler assume existência deste método. Se retorna `CaseAssignmentRow` (camelCase), precisa mapear para snake_case do DTO.

### Correção

Para cada endpoint, construir a resposta mapeando campos disponíveis:

- **transition:** Recarregar caso via `app.caseExecution.caseRepo` para obter `id`, `codigo`, `currentStageId`, `status`. Construir `stage_history_entry` a partir de `fromStageId`/`toStageId`.
- **resolve/waive:** Mapear `gateInstanceId`→`id`, derivar `resolved_by` de `user.id`, `resolved_at` de `new Date().toISOString()`. Para `gate_id`, carregar gate instance via repo.
- **update assignment:** Mapear `caseId`→`case_id`, `stageId`→`stage_id`, `processRoleId`→`process_role_id`, etc.

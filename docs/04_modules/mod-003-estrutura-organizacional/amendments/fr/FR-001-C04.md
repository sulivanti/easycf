## Detalhamento

### Correção: Seed deve criar hierarquia completa N1→N2→N3→N4

**Estado atual (incorreto):**
- Seed cria 1 nó: N1 (GRUPO-A1)
- Vincula tenant ao N1 → **viola BR-006**

**Estado correto:**
- Seed cria 4 nós: N1 → N2 → N3 → N4
- Vincula tenant ao N4 → **conforme BR-006**

**Hierarquia de seed:**

| Nível | Código | Nome | parent_id |
|-------|--------|------|-----------|
| N1 | GRUPO-A1 | Grupo A1 | null |
| N2 | UNIDADE-A1 | Unidade A1 | → N1 |
| N3 | MACRO-A1 | Macroárea A1 | → N2 |
| N4 | SUB-A1 | Subunidade A1 | → N3 |

**Vínculo:** `org_unit_tenant_links(org_unit_id=N4.id, tenant_id=tenant.id)`

**Arquivos impactados:**
- `apps/api/db/seed-admin.ts` — criar 4 nós, vincular ao N4
- `apps/api/db/fix-seed-org-unit.ts` — mesmo ajuste

**Critério de aceite adicional ao Gherkin existente:**

```gherkin
Cenário: Seed bootstrap cria hierarquia conforme BR-006
  Dado que o banco está limpo
  Quando seed-admin.ts é executado
  Então devem existir 4 nós org_units com nivel 1, 2, 3 e 4
  E org_unit_tenant_links deve vincular o tenant ao nó com nivel=4
```

# Emenda: UX-001-C01

- **Documento base:** [UX-001](../../requirements/ux/UX-001.md)
- **estado_item:** MERGED
- **Natureza:** C (Correcao)
- **Data:** 2026-03-29
- **owner:** arquitetura
- **Motivacao:** A rota `/usuarios` (`_auth.usuarios.tsx`) importa `UsersListPage` do modulo foundation (MOD-000) em vez do modulo users (MOD-002). A implementacao foundation e basica (6 colunas, delete simples, sem filtros avancados) e nao atende a UX-001 Jornada 1 que especifica: 7 colunas com dropdown de acoes por status, filtros por role/status, busca com debounce, deactivation modal, e scope-based permissions.
- **rastreia_para:** UX-001, FR-001, 05-users-list-spec

---

## Detalhamento

### Problema

O route wrapper `apps/web/src/routes/_auth.usuarios.tsx` atualmente:

```typescript
import { UsersListPage } from '@modules/foundation/pages/users/UsersListPage';
// ...
<UsersListPage
  onCreateClick={() => navigate({ to: '/usuarios/form', search: {} })}
  onEditClick={(id) => navigate({ to: '/usuarios/form', search: { id } })}
/>
```

Deveria usar a implementacao completa do MOD-002:

```typescript
import { UsersListPage } from '@modules/users/pages/UsersListPage';
```

### Divergencias da Foundation vs MOD-002

| Aspecto | Foundation (atual) | MOD-002 (correto) | UX-001 |
|---------|-------------------|-------------------|---------|
| Colunas tabela | 6 (Codigo, Nome, Email, Status, Criado, Acoes) | 7 (Status, Nome, Email, Perfil, Empresa, Ultimo Acesso, Acoes) | 7 colunas |
| Dropdown acoes | Delete simples | 4 variantes por status (ATIVO/INATIVO/BLOQUEADO/PENDENTE) | Dropdown contextual |
| Filtros | Busca basica | Status + Role + Search com debounce | Filtros avancados |
| Permissoes | Nenhuma | `canCreateUser()`, `canDeactivateUser()` via scopes | Scope guard |
| Deactivation | Delete direto | Modal confirmacao (LGPD-safe, sem PII) | ConfirmationModal |
| Paginacao | Cursor-based | Cursor-based + "Carregar mais" | Load more |

### Correcao Requerida

**Arquivo:** `apps/web/src/routes/_auth.usuarios.tsx`

**De:**
```typescript
import { createRoute, useNavigate } from '@tanstack/react-router';
import { Route as authRoute } from './_auth';
import { UsersListPage } from '@modules/foundation/pages/users/UsersListPage';

export const Route = createRoute({
  path: '/usuarios',
  getParentRoute: () => authRoute,
  component: UsuariosWrapper,
});

function UsuariosWrapper() {
  const navigate = useNavigate();

  return (
    <UsersListPage
      onCreateClick={() => navigate({ to: '/usuarios/form', search: {} })}
      onEditClick={(id) => navigate({ to: '/usuarios/form', search: { id } })}
    />
  );
}
```

**Para:**
```typescript
import { createRoute, useNavigate } from '@tanstack/react-router';
import { Route as authRoute } from './_auth';
import { UsersListPage } from '@modules/users/pages/UsersListPage';
import { useAuthMe } from '@modules/backoffice-admin/hooks/use-auth-me';

export const Route = createRoute({
  path: '/usuarios',
  getParentRoute: () => authRoute,
  component: UsuariosWrapper,
});

function UsuariosWrapper() {
  const navigate = useNavigate();
  const { data: me } = useAuthMe();
  const userScopes = me?.scopes ?? [];

  return (
    <UsersListPage
      userScopes={userScopes}
      onNavigateToCreate={() => navigate({ to: '/usuarios/form', search: {} })}
      onNavigateToInvite={(userId) =>
        navigate({ to: '/usuarios/$userId/convite', params: { userId } })
      }
    />
  );
}
```

### Padrao de Referencia

Esta integracao segue o padrao ja estabelecido nas rotas:
- `_auth.cases.tsx` — usa `useAuthMe()` + passa `userScopes`
- `_auth.processos.ciclos.tsx` — usa `useAuthMe()` + callbacks de navegacao

## Detalhamento

### Correção: `_auth.dashboard.tsx` deve importar DashboardPage real

**Estado atual (incorreto):**

O route file `apps/web/src/routes/_auth.dashboard.tsx` contém um `DashboardPage` inline com ~8 linhas de JSX (título h1 e parágrafo estático).

**Estado esperado (conforme CA-07):**

```typescript
// apps/web/src/routes/_auth.dashboard.tsx
import { createRoute } from '@tanstack/react-router';
import { Route as authRoute } from './_auth';
import { DashboardPage } from '@modules/backoffice-admin/pages/DashboardPage';

export const Route = createRoute({
  path: '/dashboard',
  getParentRoute: () => authRoute,
  component: DashboardPage,
});
```

O `DashboardPage` já existe em `apps/web/src/modules/backoffice-admin/pages/DashboardPage.tsx` e implementa:
- WelcomeWidget com saudação baseada em horário (BR-008)
- ModuleShortcuts filtrados por scopes do usuário
- Skeleton loading com timeout de 3 segundos (BR-009)
- Error state com retry e correlationId (BR-010)
- Dados de `useAuthMe()` como single source of truth

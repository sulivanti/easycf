# Emenda: FR-000-M04

- **Documento base:** [FR-000](../../requirements/fr/FR-000.md)
- **estado_item:** MERGED
- **Natureza:** M (Melhoria)
- **Data:** 2026-03-30
- **owner:** ECF Core
- **Motivação:** Quando a conexão com o servidor é perdida ou a sessão é invalidada externamente, a página permanece inalterada — o usuário precisa fechar todas as abas do navegador. Falta: detecção de `visibilitychange`, sync cross-tab via `storage` event, `refetchOnReconnect` no QueryClient, e forceLogout quando token expirado no keep-alive.
- **rastreia_para:** spec-session-disconnect-detection, FR-023, FR-000-C06, UX-000-C01

## Detalhamento

### Requisito Afetado: FR-023 — Smart Session Keep-Alive

**Adições ao Done Funcional de FR-023:**

- **Detecção de visibilidade (REQ-003):** O `useSessionKeepAlive` DEVE escutar `document.visibilitychange`. Ao retornar de `hidden` para `visible`:
  - Se o token já expirou (`remainingMs <= 0`): chamar `forceLogoutFromClient()` imediatamente
  - Se o token está dentro da janela de refresh (`remainingMs <= refreshWindowSeconds * 1000`): executar refresh imediato (não esperar o próximo ciclo de 30s)
  - Se o token ainda tem margem: não fazer nada (ciclo normal assume)

- **ForceLogout em keep-alive falho (REQ-005):** O catch block do `useSessionKeepAlive` DEVE verificar se o token já está expirado. Se `getTokenExpiresAt()` retorna valor no passado e o refresh falhou, chamar `forceLogoutFromClient()` em vez de silenciar o erro.

### Novo Requisito: FR-024 — Sincronização Cross-Tab de Logout

- **Descrição:** O sistema DEVE sincronizar logout entre múltiplas abas do navegador via `storage` event listener.
- **Prioridade:** Must
- **Perfis impactados:** Todos os usuários autenticados com múltiplas abas
- **Done funcional:**
  - Um hook `useCrossTabSync` escuta `window.storage` events
  - Quando a chave `auth_tokens` é removida (`event.newValue === null`) por outra aba: redirect para `/login`
  - O hook é montado uma única vez no `AppShell`
  - Apenas reage à chave `auth_tokens` — ignora outras mudanças no localStorage
  - Storage events só disparam para mudanças de **outra aba** na mesma origin (segurança nativa do browser)
- **Efeito colateral?** [ ] Não
- **Dependências:** FR-023, FR-000-C06

```gherkin
Funcionalidade: Sincronização Cross-Tab de Logout

Cenário: Logout em uma aba sincroniza com outra
  Dado que o usuário tem 2 abas abertas
  Quando faz logout na aba A (auth_tokens removido do localStorage)
  Então a aba B detecta via storage event
  E a aba B redireciona para /login automaticamente

Cenário: Refresh de token não dispara logout cross-tab
  Dado que o usuário tem 2 abas abertas
  Quando o keep-alive de aba A atualiza auth_tokens (refresh)
  Então a aba B NÃO é redirecionada (event.newValue !== null)
```

### Configuração do QueryClient (REQ-001, REQ-002)

- **`refetchOnReconnect: true`** no `defaultOptions.queries` do `QueryClient` (`main.tsx`): queries são re-executadas automaticamente quando conectividade é restaurada.
- **`refetchOnWindowFocus: true`** seletivamente no hook `useAuthMe`: o perfil do usuário é revalidado quando a aba ganha foco. Queries gerais mantêm `refetchOnWindowFocus: false` (CON-001).

### Alteração em FR-011 — Monitor de Integrações

#### 1. Adicionar `avg_latency_ms` ao endpoint de métricas

O endpoint GET /admin/integration-logs/metrics DEVE incluir o campo `avg_latency_ms` na resposta:

```
metricsResponse = {
  total: number,          // existente
  success: number,        // existente
  failed: number,         // existente
  dlq: number,            // existente
  running: number,        // existente
  queued: number,         // existente
  success_rate: number,   // existente
  avg_latency_ms: number  // NOVO — AVG(duration_ms) das últimas 24h, arredondado para inteiro
}
```

**Implementação:**
- Na query de métricas, adicionar: `AVG(duration_ms)::integer as avg_latency_ms`
- Filtrar apenas logs com `duration_ms IS NOT NULL` (excluir QUEUED/RUNNING sem duração)
- Se não houver logs com duração, retornar `avg_latency_ms: 0`

#### 2. Done funcional atualizado para FR-011

Adicionar ao done funcional de FR-011:
- "Header com métricas do dia: total, taxa de sucesso, DLQ count, **latência média (ms)**"

```gherkin
Cenário: Header exibe latência média
  Dado que existem 150 chamadas nas últimas 24h com durations [200, 300, 400, 500, 600]ms
  Quando admin acessa /integracoes/monitor
  Então header exibe card "Latência Média" com valor "400ms"
  E valor calculado como AVG(duration_ms) arredondado
```

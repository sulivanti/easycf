### Alterações neste pilar

#### 1. Expansão do Mapeamento de Scopes por Ação

Adicionar à tabela "Mapeamento de Scopes por Ação" de BR-001:

| Scope | Ações visíveis (adição) | Telas acessíveis |
|---|---|---|
| `users:user:read` | Editar (navegação para formulário de edição) | /usuarios/:id |
| `users:user:write` | Bloquear, Desbloquear, Reativar, Resetar senha | — (ações dentro de /usuarios) |
| `users:user:delete` | Cancelar convite (modal de confirmação) | — (ação dentro de /usuarios) |

#### 2. Exemplos Atualizados

Adicionar exemplos na seção "Exemplo":

- Usuário com `users:user:read` (sem `:write` e `:delete`): vê "Editar" no dropdown de todos os status, mas NÃO vê "Bloquear", "Desbloquear", "Reativar", "Resetar senha", "Desativar", "Cancelar convite"
- Usuário com `users:user:read` + `users:user:write` (sem `:delete`): vê "Editar" + "Bloquear"/"Desbloquear"/"Reativar"/"Resetar senha" conforme status, mas NÃO vê "Desativar" nem "Cancelar convite"
- Usuário com todos os scopes: vê todas as ações conforme status

#### 3. Regra de Visibilidade por Status + Scope

A visibilidade do dropdown combina scope + status:

| Status | `read` only | `read` + `write` | `read` + `write` + `delete` |
|---|---|---|---|
| **ATIVO** | Editar | Editar, Resetar senha, Bloquear | Editar, Resetar senha, Desativar, Bloquear |
| **INATIVO** | Editar | Editar, Reativar, Bloquear | Editar, Reativar, Bloquear |
| **BLOQUEADO** | Editar | Editar, Desbloquear | Editar, Desbloquear, Desativar |
| **PENDENTE** | Editar | Editar, Bloquear | Editar, Cancelar convite, Bloquear |

> **Nota:** "Reenviar convite" (PENDING) requer `users:user:write` — já documentado em BR-001 original.

#### 4. Novo cenário Gherkin

```gherkin
  Cenário: Ação "Bloquear" oculta sem scope de escrita
    Dado que o usuário possui scope "users:user:read" apenas
    Quando a listagem de usuários ATIVOS é exibida
    Então o dropdown de ações deve incluir apenas "Editar"
    E NÃO deve incluir "Bloquear", "Resetar senha", "Desativar"

  Cenário: Ação "Cancelar convite" oculta sem scope de exclusão
    Dado que o usuário possui scopes "users:user:read" e "users:user:write"
    Mas NÃO possui scope "users:user:delete"
    Quando a listagem de usuários PENDENTES é exibida
    Então o dropdown deve incluir "Editar" e "Bloquear"
    Mas NÃO deve incluir "Cancelar convite"
```

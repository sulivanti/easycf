## 2. UX-PARAM-001 — Configurador de Enquadradores

### 2.1 Jornada Principal (Happy Path)

```
1. Analista acessa /parametrizacao/enquadradores
2. GET /admin/framers carrega lista de enquadradores
3. GET /admin/target-objects carrega objetos-alvo
4. GET /admin/incidence-rules carrega regras de incidência
5. Layout 3-panel renderiza simultaneamente: FramersList (esq), TargetObjectsPanel (centro), IncidenceMatrix (dir)
6. Analista seleciona enquadrador na FramersList (seleção visual com bg azul + border-left)
7. Analista vincula/desvincula objetos-alvo no painel central
8. Matriz tri-state renderiza grid enquadradores × rotinas PUBLISHED com pads OBR/OPC/AUTO
9. Analista clica em célula para ciclar tipo de incidência (EMPTY → OBR → OPC → AUTO)
10. Analista simula motor via botão "Simular Dry-Run" no header (abre DryRunModal centralizado)
```

| Estado | Comportamento | Componente |
|---|---|---|
| **loading** | Skeleton nos 3 painéis: FramersList (4 items skeleton), TargetObjectsPanel (3 items skeleton), IncidenceMatrix (grid 3×4 skeleton cells) | Skeleton list + cards + grid |

## 3. UX-ROTINA-001 — Cadastro e Editor de Rotinas

### 3.1 Jornada Principal (Happy Path)

```
1. Administrador acessa /parametrizacao/rotinas
2. GET /admin/routines carrega lista de rotinas com filtro por status
3. Administrador cria nova rotina DRAFT (codigo, nome)
4. Clica na rotina → split-view editor abre à direita
5. GET /admin/routines/:id carrega detalhes com itens e links
6. Adiciona itens com formulário adaptativo por tipo (7 tipos)
7. Reordena itens via drag-and-drop (DRAFT only)
8. Auto-save com debounce 600ms (sem botão explícito)
9. Publica rotina com modal de confirmação (valida min 1 item)
10. Editor entra em modo readonly com banner
11. Faz fork com motivo obrigatório → nova versão DRAFT
12. Visualiza preview dry-run do motor
13. Visualiza timeline de versões com ancestralidade
```

### 5.2 Semântica das Colunas e Linhas

| Eixo | Antes (v1) | Depois (v2) |
|---|---|---|
| **Linhas** | Enquadradores (código monospace) | Enquadradores (nome abreviado 12px 600 + ActiveDot verde 6×6 se ≥1 regra) |
| **Colunas** | Objetos-alvo (nomes) | Rotinas PUBLISHED (Rot. Validação, Rot. Visibilidade, etc.) — filtradas por FilterBadge |
| **Célula** | Regra existe (badge) ou não (vazio) | Tipo de incidência (OBR/OPC/AUTO/EMPTY) |

> **UNIQUE constraint** impede que o mesmo par (framer_id, target_object_id) tenha mais de uma regra (BR-003). O `incidence_type` é atributo da regra, não cria duplicata.

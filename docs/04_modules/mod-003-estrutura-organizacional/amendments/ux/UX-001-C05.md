## Detalhamento

### Correção: Conectar checkbox "Mostrar inativos" ao endpoint tree

**Dependência:** FR-001-C05 adiciona o query param `include_inactive` ao `GET /org-units/tree`.

#### 1. API Client — `org-units.api.ts`

```typescript
// ANTES:
export async function fetchOrgTree(signal?: AbortSignal): Promise<OrgUnitTreeResponseDTO> {
  return httpClient.get<OrgUnitTreeResponseDTO>('/org-units/tree', { signal });
}

// DEPOIS:
export async function fetchOrgTree(
  includeInactive = false,
  signal?: AbortSignal,
): Promise<OrgUnitTreeResponseDTO> {
  const query = includeInactive ? '?include_inactive=true' : '';
  return httpClient.get<OrgUnitTreeResponseDTO>(`/org-units/tree${query}`, { signal });
}
```

#### 2. Hook — `use-org-tree.ts`

```typescript
// ANTES:
export function useOrgTree() {
  return useQuery({
    queryKey: [...ORG_TREE_KEY],
    queryFn: ({ signal }) => fetchOrgTree(signal),
    select: (data) => data.tree.map(toTreeNodeVM),
  });
}

// DEPOIS:
export function useOrgTree(includeInactive = false) {
  return useQuery({
    queryKey: [...ORG_TREE_KEY, { includeInactive }],
    queryFn: ({ signal }) => fetchOrgTree(includeInactive, signal),
    select: (data) => data.tree.map(toTreeNodeVM),
  });
}
```

**Nota (GUD-001):** `includeInactive` é incluído na `queryKey` para que o React Query trate cada combinação como cache separado, invalidando automaticamente ao trocar o toggle.

#### 3. Página — `OrgTreePage.tsx`

```typescript
// ANTES (linha 98):
const { data: tree, isLoading, isError, error, refetch } = useOrgTree();

// DEPOIS:
const { data: tree, isLoading, isError, error, refetch } = useOrgTree(showInactive);
```

**Nota (CON-002):** A filtragem client-side no `OrgTreeNode` (linha 131: `visibleChildren`) permanece como segunda camada — quando o toggle é desligado, a API já não retorna inativos, mas a filtragem local é mantida como safety net.

### Acceptance Criteria

- **AC-C05-001**: Given o checkbox "Mostrar inativos" está desmarcado, When a árvore carrega, Then `GET /org-units/tree` é chamado sem query param `include_inactive`, e apenas unidades ativas são exibidas (comportamento atual preservado).
- **AC-C05-002**: Given o checkbox "Mostrar inativos" está marcado, When a árvore carrega, Then `GET /org-units/tree?include_inactive=true` é chamado, e unidades inativas aparecem na árvore com badge "Inativo" e opacity 50%.
- **AC-C05-003**: Given o toggle é alternado rapidamente, When o React Query recebe nova queryKey, Then a request anterior é cancelada via signal e nova request é feita — sem duplicação de dados ou race condition.
- **AC-C05-004**: Given unidades inativas são exibidas, When o usuário clica no menu contextual de uma inativa, Then apenas "Restaurar" e "Ver histórico" são disponíveis (comportamento já implementado no OrgTreeNode, sem alteração).

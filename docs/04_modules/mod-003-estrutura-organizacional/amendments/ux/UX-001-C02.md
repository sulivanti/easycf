## Detalhamento

### C01 — Elevacao do estado `isEditing` para OrgTreePage

O estado `isEditing` atualmente e local ao `DetailPanel` (useState). Para que o guard funcione, o `OrgTreePage` precisa saber se ha edicao ativa antes de alterar `selectedId`.

**Alteracoes no `DetailPanel`:**
- Remover `const [isEditing, setIsEditing] = useState(false)` local
- Receber `isEditing: boolean` e `onEditingChange: (editing: boolean) => void` via props
- Substituir `setIsEditing(true)` por `onEditingChange(true)` em `handleStartEdit` e `useEffect(requestEdit)`
- Substituir `setIsEditing(false)` por `onEditingChange(false)` em `handleCancel` e `handleSave.onSuccess`

**Alteracoes no `OrgTreePage`:**
- Adicionar `const [isEditing, setIsEditing] = useState(false)`
- Passar `isEditing` e `onEditingChange={setIsEditing}` como props ao `DetailPanel`

### C02 — Calculo de dirty state

O `DetailPanel` deve calcular e expor se o formulario possui alteracoes nao salvas (dirty).

**Implementacao:**
- Adicionar callback `onDirtyChange: (dirty: boolean) => void` nas props do `DetailPanel`
- No `DetailPanel`, calcular dirty via `useEffect` comparando `form` com `detailToForm(detail)`:

```typescript
const isDirty = useMemo(() => {
  if (!detail || !isEditing) return false;
  const original = detailToForm(detail);
  return Object.keys(original).some(
    (key) => form[key as keyof EditFormState] !== original[key as keyof EditFormState]
  );
}, [form, detail, isEditing]);

useEffect(() => {
  onDirtyChange(isDirty);
}, [isDirty, onDirtyChange]);
```

**No `OrgTreePage`:**
- Adicionar `const [isDirty, setIsDirty] = useState(false)`
- Passar `onDirtyChange={setIsDirty}` ao `DetailPanel`

### C03 — Guard de edicao com ConfirmationModal

Implementar interceptacao de acoes que causariam perda de dados nao salvos.

**Novo state no `OrgTreePage`:**

```typescript
interface EditGuardState {
  open: boolean;
  pendingAction: (() => void) | null;
}

const EDIT_GUARD_INITIAL: EditGuardState = { open: false, pendingAction: null };
const [editGuard, setEditGuard] = useState<EditGuardState>(EDIT_GUARD_INITIAL);
```

**Helper `guardedAction`:**

```typescript
const guardedAction = useCallback((action: () => void) => {
  if (isEditing && isDirty) {
    setEditGuard({ open: true, pendingAction: action });
    return;
  }
  if (isEditing) {
    setIsEditing(false);
  }
  action();
}, [isEditing, isDirty]);
```

> Se editando mas **sem** alteracoes (not dirty): sai do modo edicao e executa a acao diretamente, sem dialogo.

**Handlers envolvidos com guard:**

| Handler | Acao original | Com guard |
|---------|---------------|-----------|
| `handleSelect(id)` | `setSelectedId(id)` | `guardedAction(() => { setIsEditing(false); setSelectedId(id); })` |
| `handleOpenCreate(parentId?)` | `setFormPanel({ open: true, ... })` | `guardedAction(() => { setIsEditing(false); setFormPanel({ open: true, mode: 'create', parentId }); })` |
| `handleOpenEdit(id)` | `setSelectedId(id); setRequestEdit(true)` | `guardedAction(() => { setIsEditing(false); setSelectedId(id); setFormPanel({ open: false }); setRequestEdit(true); })` |

**ConfirmationModal do guard (no JSX do `OrgTreePage`):**

```jsx
<ConfirmationModal
  open={editGuard.open}
  onOpenChange={(open) => !open && setEditGuard(EDIT_GUARD_INITIAL)}
  title={COPY.modal.editGuardTitle}
  description={COPY.modal.editGuardBody}
  variant="default"
  confirmLabel={COPY.modal.editGuardConfirm}
  cancelLabel={COPY.modal.editGuardCancel}
  onConfirm={() => {
    const action = editGuard.pendingAction;
    setEditGuard(EDIT_GUARD_INITIAL);
    setIsEditing(false);
    action?.();
  }}
/>
```

- **"Cancelar Edicao"** (botao confirm) → executa `pendingAction`, fecha guard
- **"Continuar Editando"** (botao cancel) → fecha guard, nao executa acao

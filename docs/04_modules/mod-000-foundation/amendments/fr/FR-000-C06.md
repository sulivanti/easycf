# Emenda: FR-000-C06

- **Documento base:** [FR-000](../../requirements/fr/FR-000.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-25
- **owner:** arquitetura
- **Motivação:** Quando a sessão do usuário expira por timeout (JWT expirado no servidor), o AppShell exibe toasts repetidamente mas impossibilita o retorno à tela de login, criando um loop de redirecionamento. O handler de 401 no AppShell chama `navigate('/login')` sem limpar `localStorage['auth_tokens']`, e a rota `/login` redireciona usuários com token (mesmo expirado) de volta para `/dashboard`. Além disso, o `useAuthMe` retenta em 401 (retry:1 herdado), duplicando toasts.
- **rastreia_para:** FR-001, FR-004, UX-000 (UX-001), spec-fix-session-timeout-redirect-loop

## Detalhamento

### DEF-001 — AppShell não limpa localStorage ao detectar 401

**Localização:** `apps/web/src/modules/backoffice-admin/components/AppShell.tsx:218-229`

O `useEffect` que trata o erro 401 chama `navigate({ to: '/login' })` sem antes remover `auth_tokens` do localStorage. O token expirado permanece, fazendo o route guard do `/login` (`login.tsx:beforeLoad`) redirecionar de volta para `/dashboard`.

```typescript
// ANTES (quebrado)
useEffect(() => {
  if (error instanceof ApiError && error.status === 401) {
    toast.error('Sua sessão expirou. Faça login novamente.', {
      description: error.correlationId ? `ID: ${error.correlationId}` : undefined,
    });
    navigate({ to: '/login' }); // ← NÃO LIMPA localStorage → loop infinito
  } else if (error instanceof ApiError) {
    toast.error('Não foi possível carregar seu perfil. Tente novamente.', {
      description: error.correlationId ? `ID: ${error.correlationId}` : undefined,
    });
  }
}, [error, navigate]);
```

### DEF-002 — Router context é estático

**Localização:** `apps/web/src/main.tsx:17-29`

`getAuthFromStorage()` é executado UMA vez na montagem do `<App>`. Remover tokens do localStorage não atualiza `context.auth.user` automaticamente. Mesmo após `localStorage.removeItem`, `_auth.tsx:beforeLoad` e `login.tsx:beforeLoad` usam o context stale.

### DEF-003 — useAuthMe retenta em 401

**Localização:** `apps/web/src/modules/backoffice-admin/hooks/use-auth-me.ts:60-65`

O `QueryClient` default tem `retry: 1`. Quando `GET /auth/me` retorna 401, React Query retenta automaticamente — outra chamada 401, outro disparo do `useEffect`, outro toast empilhado.

### Correção 1 — Função `forceLogout` (novo arquivo)

**Arquivo:** `apps/web/src/modules/foundation/utils/force-logout.ts`

Função reutilizável que limpa estado de autenticação e força reload completo para resetar o router context.

```typescript
import type { QueryClient } from '@tanstack/react-query';

/**
 * Limpa estado de autenticação e redireciona para /login.
 * Usa window.location.href para forçar reset do router context
 * (que é computado estaticamente em main.tsx via getAuthFromStorage).
 */
export function forceLogout(queryClient: QueryClient): void {
  localStorage.removeItem('auth_tokens');
  queryClient.clear();
  window.location.href = '/login';
}
```

**Justificativa do full reload:** O router context é computado uma única vez no `main.tsx`. `navigate()` opera dentro do Router cujo context já está "contaminado" com o user antigo. Só `window.location.href` (full reload) re-executa `getAuthFromStorage()` e gera um context limpo.

### Correção 2 — AppShell usa `forceLogout` em vez de `navigate`

**Arquivo:** `apps/web/src/modules/backoffice-admin/components/AppShell.tsx`

```typescript
// DEPOIS (corrigido)
import { useQueryClient } from '@tanstack/react-query';
import { forceLogout } from '@modules/foundation/utils/force-logout';

// Dentro de AppShell():
const queryClient = useQueryClient();

useEffect(() => {
  if (error instanceof ApiError && error.status === 401) {
    toast.error('Sua sessão expirou. Faça login novamente.', {
      id: 'session-expired', // ← ID fixo evita duplicatas (Sonner dedup nativo)
    });
    forceLogout(queryClient);
    return; // early return — nada mais após redirect
  }
  if (error instanceof ApiError) {
    toast.error('Não foi possível carregar seu perfil. Tente novamente.', {
      description: error.correlationId ? `ID: ${error.correlationId}` : undefined,
    });
  }
}, [error, queryClient]);
```

**Mudanças:**
1. `navigate('/login')` → `forceLogout(queryClient)` — quebra o loop
2. `id: 'session-expired'` no toast — deduplicação nativa do Sonner
3. `useNavigate()` pode ser removido se não usado em outros pontos do componente

### Correção 3 — useAuthMe desabilita retry em 401

**Arquivo:** `apps/web/src/modules/backoffice-admin/hooks/use-auth-me.ts`

```typescript
// ANTES
return useQuery<AuthMeResponse, ApiError>({
  queryKey: AUTH_ME_QUERY_KEY,
  queryFn: () => fetchAuthMe({ screenId, actionId }),
  staleTime: 30_000,
  enabled,
});

// DEPOIS
return useQuery<AuthMeResponse, ApiError>({
  queryKey: AUTH_ME_QUERY_KEY,
  queryFn: () => fetchAuthMe({ screenId, actionId }),
  staleTime: 30_000,
  enabled,
  retry: (failureCount, error) => {
    if (error.status === 401) return false; // Sessão expirada — não retentar
    return failureCount < 1; // Outros erros: retry 1x (padrão)
  },
});
```

### Alterações neste pilar

#### Novos contratos por operationId

### `users_update_status` — Atualização de Status

- **Método:** `PATCH /api/v1/users/:id/status`
- **Usado em:** FR-001-M01 — Bloquear, Desbloquear, Reativar
- **Request body:**

  ```json
  { "status": "ACTIVE" | "BLOCKED" | "INACTIVE" }
  ```

- **Response 200:**

  ```json
  { "id": "uuid", "status": "enum", "updatedAt": "ISO8601" }
  ```

- **Erros possíveis:** 401, 403, 404, 409 (transição de status inválida), 500
- **Scopes:** `users:user:write`

### `users_reset_password` — Reset de Senha

- **Método:** `POST /api/v1/users/:id/reset-password`
- **Usado em:** FR-001-M01 — Resetar senha
- **Request body:** vazio (ou `{}`)
- **Response 200:**

  ```json
  { "message": "Password reset email sent." }
  ```

- **Erros possíveis:** 401, 403, 404, 500
- **Scopes:** `users:user:write`

### `users_cancel_invite` — Cancelamento de Convite

- **Método:** `DELETE /api/v1/users/:id/invite`
- **Usado em:** FR-001-M01 — Cancelar convite
- **Response 200:**

  ```json
  { "id": "uuid", "status": "INACTIVE", "updatedAt": "ISO8601" }
  ```

- **Erros possíveis:** 401, 403, 404, 409 (não está PENDING), 500
- **Scopes:** `users:user:delete`

## Detalhamento

### 1. GatesTab — CRUD completo (FR-007)

**Hooks existentes:** `useCreateGate()`, `useUpdateGate()`, `useDeleteGate()`

**Adições à UI:**
- Botão "+ Novo gate" no header (oculto quando `readonly`)
- Cada gate na lista ganha ícones editar/excluir (ocultos quando `readonly`)
- Dialog para criar/editar com campos:
  - `nome` (Input, obrigatório)
  - `descricao` (Input, opcional)
  - `gate_type` (Select: APPROVAL, DOCUMENT, CHECKLIST, INFORMATIVE)
  - `required` (Checkbox — desabilitado se INFORMATIVE, BR-007)
  - `ordem` (auto-default: próximo na sequência)
- ConfirmationModal para exclusão

### 2. RolesTab — Link/Unlink (FR-009)

**Hooks existentes:** `useLinkStageRole()`, `useUnlinkStageRole()`

**Adições à UI:**
- Botão "+ Vincular papel" no header (oculto quando `readonly`)
- Cada role na lista ganha botão de desvincular (oculto quando `readonly`)
- Dialog para vincular com campos:
  - Select de papéis do catálogo (filtrar os já vinculados)
  - `required` (Checkbox)
  - `max_assignees` (Input number, opcional)

### 3. TransitionsTab — Create/Delete (FR-010)

**Hooks existentes:** `useCreateTransition()`, `useDeleteTransition()`

**Adições à UI:**
- Botão "+ Nova transição" no header de Saída (oculto quando `readonly`)
- Cada transição ganha botão excluir (oculto quando `readonly`)
- Dialog para criar com campos:
  - `to_stage_id` (Select de estágios, excluindo o atual — BR-008)
  - `nome` (Input, obrigatório)
  - `gate_required` (Checkbox)
  - `evidence_required` (Checkbox)
- ConfirmationModal para exclusão

### 4. Prop `allStages` no StageConfigPanel

- FlowEditorPage computa `allStages = flow.macro_stages.flatMap(ms => ms.stages)` e passa ao StageConfigPanel
- Necessário para o select de estágio destino nas transições (BR-008: filtrar self-transitions)

### 5. Regras de negócio respeitadas

- **BR-001:** Ciclos PUBLISHED são imutáveis → flag `readonly` já existe, ocultar todos os botões de mutação
- **BR-007:** Gate INFORMATIVE nunca bloqueia → forçar `required=false` quando tipo é INFORMATIVE
- **BR-008:** Sem self-transitions → filtrar estágio atual do select de destino
- **BR-012:** Gates têm ordenação → campo `ordem` auto-gerenciado

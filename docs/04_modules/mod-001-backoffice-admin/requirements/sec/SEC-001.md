# SEC-001 — Segurança e Compliance do Backoffice Admin

---

## 1. Autenticação

- **Modelo:** Delegada integralmente ao MOD-000 (Foundation)
- **Fluxos no Shell:**
  - Login nativo: e-mail + senha via UX-AUTH-001 → POST /auth/login (auth_login)
  - Recuperação de senha: UX-AUTH-001 painel forgot-password → POST /auth/forgot-password
  - Reset de senha: UX-AUTH-001 painel reset-password → POST /auth/reset-password
  - Logout: ProfileWidget → POST /auth/logout (auth_logout)
- **Sessões:** Gerenciadas pelo MOD-000 (JWT com sessionId ancorado em banco)
- **Tokens:** Cookies httpOnly (accessToken + refreshToken) — **não acessíveis via JavaScript**
- **MFA:** Quando ativo, login redireciona para /login/mfa (escopo futuro do MOD-001)

## 2. Autorização

- **Modelo:** RBAC por escopos, consumido via `auth_me.scopes[]` (MOD-000)
- **Sidebar:** Filtrada por scopes — itens sem permissão NÃO aparecem (nem desabilitados) (BR-005)
- **Dashboard shortcuts:** Filtrados por `required_scope` contra `auth_me.scopes[]` (FR-005)
- **Proteção de rotas:** JWT expirado/ausente → redirect /login via interceptor HTTP global

### Catálogo de Scopes Consumidos pelo MOD-001

| Scope | Recurso | Usado em |
|---|---|---|
| `users:user:read` | Card "Usuários" | Sidebar, ModuleShortcuts |
| `users:role:read` | Card "Perfis e Permissões" | Sidebar, ModuleShortcuts |
| `tenants:branch:read` | Card "Filiais" | Sidebar, ModuleShortcuts |
| `system:audit:read` | Card "Auditoria" | Sidebar, ModuleShortcuts |

> **Anti-pattern (DOC-FND-000 §2):** O MOD-001 NÃO define novos scopes. Consome scopes definidos pelo MOD-000.

## 3. Prevenção de Enumeração de Usuários

- **Login (BR-001):** Mensagem DEVE ser sempre "E-mail ou senha inválidos." — sem variações
- **Forgot-password (BR-002):** Mensagem DEVE ser sempre positiva — mesma resposta para e-mail existente e inexistente
- **Validação:** Testes E2E devem confirmar que ambas as telas seguem esta regra

## 4. Classificação dos Dados

| Dado | Classificação | Tráfego | Armazenamento |
|---|---|---|---|
| E-mail (input login/forgot) | Interno | HTTPS | Memória (React state) — nunca localStorage |
| Senha (input login/reset) | Confidencial | HTTPS | Memória (React state) — limpar após submit |
| Nome do usuário (auth_me) | Interno | HTTPS | Memória (React state) — via auth_me (BR-008) |
| Tenant name (auth_me) | Interno | HTTPS | Memória (React state) |
| JWT tokens | Confidencial | Cookies httpOnly | Browser cookie store — não acessível via JS |
| Scopes (auth_me) | Interno | HTTPS | Memória (React state) |

## 5. Telemetria e Privacidade

- **tenant_id AUSENTE** em ações pré-autenticação UX-AUTH-001 (BR-006)
- **tenant_id PRESENTE** em ações pós-autenticação UX-SHELL-001 e UX-DASH-001 (BR-006)
- **Nenhuma PII** no UIActionEnvelope (não inclui e-mail, nome ou senha)
- **correlation_id** propagado em todas as requisições HTTP (rastreabilidade sem PII)

## 6. Proteções

- **Rate limit:** Delegado ao MOD-000 (429 + retry_after) — Shell exibe countdown
- **Sanitização:** Inputs de e-mail e senha sanitizados no frontend (XSS prevention)
- **CSRF:** Cookies sameSite=lax protegem contra CSRF
- **Interceptor HTTP 401:** Global — redireciona para /login em qualquer requisição com sessão expirada

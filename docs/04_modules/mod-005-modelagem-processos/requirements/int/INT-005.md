## 1.9 Paginação Cursor-Based

Endpoints com paginação (`admin_cycles_list`, `admin_process_roles_list`) seguem o padrão:

### Request Query Params

| Param | Tipo | Default | Descrição |
|---|---|---|---|
| `cursor` | string | null | Cursor opaco para próxima página |
| `limit` | integer | 50 | Itens por página (max: 100) |
| `status` | string | null | Filtro por status (apenas cycles) |

### Response Envelope

```json
{
  "data": [ /* items */ ],
  "pagination": {
    "next_cursor": "eyJpZCI6Ij...",
    "has_more": true,
    "limit": 50
  }
}
```

> **Nota:** O cursor é opaco e baseado em `(created_at, id)` para estabilidade em inserções concorrentes.

## 2. Contratos de Erro (RFC 9457)

Todos os endpoints retornam Problem Details com `X-Correlation-ID`:

| HTTP | Tipo | Quando | extensions |
|---|---|---|---|
| 201 | — | Recurso criado | — |
| 200 | — | Sucesso (GET/PATCH) | — |
| 400 | `validation_error` | Shape inválido | `invalid_fields[]` |
| 401 | `unauthorized` | JWT ausente/expirado | — |
| 403 | `forbidden` | Scope insuficiente | `required_scope` |
| 404 | `not_found` | Recurso não existe | — |
| 409 | `conflict` | Duplicata (estágio inicial, papel-estágio) | `conflicting_entity` |
| 422 | `business_rule` | Regra de negócio violada | Varia por BR |

### 2.1 Extensões 422 por Regra

| BR | Mensagem | extensions |
|---|---|---|
| BR-001 | "Ciclos publicados são imutáveis..." | `{ cycle_status: "PUBLISHED" }` |
| BR-002 | "Este ciclo já possui um estágio inicial..." | `{ existing_initial_stage_id }` |
| BR-003 | "O ciclo precisa de ao menos um estágio inicial..." | `{ stages_count, initial_count: 0 }` |
| BR-005 (macro) | "Desative os estágios antes de remover..." | `{ active_stages_count }` |
| BR-005 (stage) | "Este estágio possui N instância(s) ativa(s)..." | `{ active_instances }` |
| BR-005 (cycle) | "Somente ciclos em DRAFT podem ser excluídos..." | `{ cycle_status }` |
| BR-008 (cross) | "Os estágios de origem e destino devem pertencer ao mesmo ciclo." | `{ from_cycle_id, to_cycle_id }` |
| BR-008 (auto) | "Um estágio não pode transitar para si mesmo." | `{ stage_id }` |
| BR-010 | "Somente ciclos publicados podem ser depreciados." | `{ cycle_status }` |
| RESTRICT (role) | "Este papel possui vínculos ativos com estágios. Remova os vínculos antes de excluir." | `{ active_links_count }` |

## 4. Integração Inter-Módulo: MOD-006 (Execução de Processos)

### 4.1 MOD-005 → MOD-006: Consulta de Instâncias Ativas

| Propriedade | Valor |
|---|---|
| **external_system** | MOD-006 (Execução de Processos) |
| **purpose** | Verificar se um estágio tem instâncias ativas antes de permitir soft delete |
| **mode** | api (síncrona) |
| **auth** | Service-to-service (internal JWT ou shared secret) |
| **contract** | `GET /internal/instances/count-active?stage_id={uuid}` |
| **response** | `{ "active_count": 5 }` |
| **failure_behavior** | timeout=3000ms, retries=1, backoff=none, dlq=false |
| **observability** | correlationId propagado, log structured, erro 503 se MOD-006 indisponível |

> **Decisão de design:** Se MOD-006 estiver indisponível, a deleção do estágio DEVE ser bloqueada com 503: "Não foi possível verificar instâncias ativas. Tente novamente."

### 4.2 MOD-006 → MOD-005: Consumo de Blueprint

| Propriedade | Valor |
|---|---|
| **external_system** | MOD-006 (Execução de Processos) |
| **purpose** | MOD-006 consome blueprints PUBLISHED para instanciar ciclos |
| **mode** | api (síncrona) |
| **contract** | `GET /api/v1/admin/cycles/:id/flow` (operationId: `admin_cycles_flow`) |
| **auth** | JWT com scope `process:cycle:read` |
| **failure_behavior** | MOD-006 deve cachear blueprint localmente após primeira leitura |

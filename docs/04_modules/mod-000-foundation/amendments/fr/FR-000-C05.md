# Emenda: FR-000-C05

- **Documento base:** [FR-000](../../requirements/fr/FR-000.md)
- **estado_item:** DRAFT
- **Natureza:** C (Correção)
- **Data:** 2026-03-25
- **owner:** arquitetura
- **Motivação:** O `LoginUseCase` gera JWT com `tid: null` e `scopes: []` porque `tokenService.generatePair()` é chamado apenas com `{ userId, sessionId }`, sem resolver o tenant ativo nem as permissões da role. O `TokenPayload` já aceita `tenantId?` e `scopes?` opcionais, mas o login nunca os popula. Resultado: todas as rotas protegidas por scope retornam 403, e o frontend mostra sidebar vazia.
- **rastreia_para:** FR-001, BR-003, SEC-000, spec-fix-auth-flow-session-expired

---

## Detalhamento

### Problema — JWT gerado sem tenant e scopes

**Localização:** `apps/api/src/modules/foundation/application/use-cases/auth/login.use-case.ts:107-110`

```typescript
// ANTES (quebrado)
const tokenPair = await this.tokenService.generatePair({
  userId: user.id,
  sessionId: created.id,
  // ← SEM tenantId, SEM scopes
});
```

**Interface `TokenPayload`** (em `services.ts`):
```typescript
interface TokenPayload {
  userId: string;
  sessionId: string;
  tenantId?: string;     // ← já existe, nunca usado
  scopes?: readonly string[];  // ← já existe, nunca usado
}
```

### Correção — Resolver tenant/scopes antes de gerar JWT

O `LoginUseCase` deve, após autenticar o usuário e antes de gerar tokens:

1. **Buscar primeiro `tenant_users` ativo** do usuário via `TenantUserRepository.findByUserId()`
2. **Buscar scopes da role vinculada** via `RoleRepository.findById()` → `role.scopes`
3. **Incluir `tenantId` e `scopes`** no payload do `generatePair()`

```typescript
// DEPOIS (corrigido)
// 1. Resolver tenant ativo
const tenantUsers = await this.tenantUserRepo.findByUserId(user.id);
const activeTenantUser = tenantUsers.find(tu => tu.status === 'ACTIVE');
if (!activeTenantUser) {
  throw new UnauthorizedError('User has no active tenant association');
}

// 2. Resolver scopes da role
const role = await this.roleRepo.findById(activeTenantUser.roleId);
if (!role) {
  throw new UnauthorizedError('Role not found for tenant association');
}
const scopes = role.scopes.map(s => s.value);

// 3. Gerar JWT com tenant e scopes
const tokenPair = await this.tokenService.generatePair({
  userId: user.id,
  sessionId: created.id,
  tenantId: activeTenantUser.tenantId,
  scopes,
});
```

### Edge cases

1. **Usuário sem tenant ativo:** Retorna erro 401 com mensagem clara (não gera JWT vazio)
2. **Role sem scopes:** JWT gerado com `scopes: []` — válido, usuário sem acesso a módulos
3. **Múltiplos tenants:** Usa o primeiro ACTIVE encontrado. Troca de tenant é feature futura (fora do escopo)

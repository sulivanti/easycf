### Alterações neste pilar

#### Ingestão Queue — Alinhamento com §3.4.1–3.4.5

**Ação derivada:** Alinhar ingestão queue BullMQ (mod-008) com as novas regras normativas de DOC-PADRAO-002 §3.4.

**Requisitos específicos:**

1. **Conexão BullMQ (§3.4.2):** Ingestão queue MUST reutilizar conexão ioredis singleton (opção `connection`). MUST NOT criar conexão Redis por fila.
2. **Nomenclatura de fila (§3.4.2 + §3.4.3):** Fila MUST ser nomeada `mod-008:ingest`. MUST usar `:` como separador.
3. **Job cleanup (§3.4.2):** SHOULD configurar `defaultJobOptions.removeOnComplete` para evitar crescimento infinito.
4. **Database (§3.4.5):** Filas BullMQ MUST usar db1 (`noeviction`). Conexão ioredis para BullMQ MUST especificar `db: 1`.
5. **Health check (§3.4.6):** MUST expor Redis health check no endpoint `/health`.

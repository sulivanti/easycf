# INT-006 — Contrato de Integração: Alteração de Senha

---

## INT-006 — POST /api/v1/auth/change-password (auth_change_password)

- **Sistema externo:** MOD-000 (Foundation API)
- **Modo:** API (HTTP REST)
- **Auth:** JWT (Bearer Token via httpOnly cookie — sessão ativa obrigatória)
- **Contrato:**
  - **Request:** `POST /api/v1/auth/change-password` — `{ current_password: string, new_password: string, confirm_password: string }`
  - **Response 200:** `{ message: "Password changed successfully" }`
  - **Response 401:** Sessão expirada → interceptor redireciona para /login
  - **Response 422 (senha atual incorreta):** RFC 9457 — `{ type: "/problems/invalid-current-password", title: "Unprocessable Entity", status: 422, detail: "Senha atual incorreta.", extensions: { correlationId, field: "current_password" } }`
  - **Response 422 (política de senha):** RFC 9457 — `{ type: "/problems/password-policy-violation", title: "Unprocessable Entity", status: 422, detail: "A nova senha não atende os requisitos mínimos: ...", extensions: { correlationId, field: "new_password" } }`
- **Timeout:** 5000ms
- **Retry:** N/A (retry delegado ao usuário via botão "Salvar")
- **Dead Letter Queue (DLQ):** Não
- **Idempotência:** N/A (gerenciada pelo MOD-000 — bcrypt compare + update)
- **Fallback:** Toast de erro com correlationId (DOC-UX-012 §2.1). Modal NÃO fecha em caso de erro.
- **Observabilidade:** `["correlationId", "logs"]` — X-Correlation-ID no header, UIActionEnvelope (submit_change_password)

- **estado_item:** READY
- **owner:** arquitetura
- **data_ultima_revisao:** 2026-03-23
- **rastreia_para:** US-MOD-001-F01, FR-007, FR-004, DOC-FND-000, PEN-001-003
- **referencias_exemplos:** EX-CI-006
- **evidencias:** Endpoint definido em DOC-FND-000 §1.3: `POST /auth/change-password` — requer sessão ativa, bcrypt compare da senha atual. Domain event `auth.password_changed` emitido pelo MOD-000 (DOC-FND-000 §1.2).

# Emenda: DATA-000-C03

- **Documento base:** [DATA-000](../../requirements/data/DATA-000.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-30
- **owner:** arquitetura
- **Motivação:** O CHECK constraint `role_permissions_scope_check` usa regex `^[a-z][a-z0-9_]*(:[a-z][a-z0-9_]*){1,2}$` que não aceita hífens. O scope `mcp:agent:phase2-enable` (definido no catálogo DOC-FND-000 §2.2 e referenciado em FR-010/MOD-010) contém hífen e viola o constraint, impedindo o INSERT na tabela `role_permissions`.
- **rastreia_para:** FR-000-C09, DOC-FND-000 §2.2

---

## Detalhamento

### Problema

A regex do CHECK constraint `role_permissions_scope_check` na tabela `role_permissions` (DATA-000 §6) permite apenas `[a-z0-9_]` nos segmentos do scope. O scope `mcp:agent:phase2-enable` usa hífen (`-`), que é válido no catálogo canônico mas rejeitado pelo banco.

### Correção

Atualizar a regex para incluir hífen nos caracteres permitidos:

**Antes:**
```
^[a-z][a-z0-9_]*(:[a-z][a-z0-9_]*){1,2}$
```

**Depois:**
```
^[a-z][a-z0-9_-]*(:[a-z][a-z0-9_-]*){1,2}$
```

### Arquivos modificados

| Arquivo | Alteração |
|---------|-----------|
| `apps/api/db/schema/foundation.ts` | Regex do CHECK `role_permissions_scope_check` atualizada: `[a-z0-9_]` → `[a-z0-9_-]` |

### Migração

Requer `ALTER TABLE` no banco para dropar e recriar o constraint:

```sql
ALTER TABLE role_permissions DROP CONSTRAINT role_permissions_scope_check;
ALTER TABLE role_permissions ADD CONSTRAINT role_permissions_scope_check
  CHECK (scope ~ '^[a-z][a-z0-9_-]*(:[a-z][a-z0-9_-]*){1,2}$');
```

Ou via `drizzle-kit push` (já executado em produção 2026-03-30).

# Emenda: INT-000-C01

- **Documento base:** [INT-000](../../requirements/int/INT-000.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-25

- **Motivação:** O OpenAPI `v1.yaml` referencia `LoginResponse` (que inclui campo `user`) para o endpoint `POST /auth/refresh`. Porém o `RefreshTokenUseCase` não retorna dados do user — apenas `{ tokenPair, sessionId }`. O contrato OpenAPI deve refletir a realidade da API criando um schema `RefreshResponse` sem `user`.
- **rastreia_para:** FR-000-C04, INT-000, FR-003

> **Contexto:** Este amendment deriva de [FR-000-C04](../fr/FR-000-C04.md).

### Alterações neste pilar

**Correção propagada:** O endpoint `POST /auth/refresh` no OpenAPI `v1.yaml` referencia `$ref: "#/components/schemas/LoginResponse"` que inclui campo `user`. O use-case real não retorna `user`.

**Arquivo:** `apps/api/openapi/v1.yaml`

**1. Adicionar novo schema `RefreshResponse`:**

```yaml
# Em components/schemas — após LoginMfaResponse
RefreshResponse:
  type: object
  required: [access_token, refresh_token, token_type, expires_in]
  properties:
    access_token: { type: string }
    refresh_token: { type: string }
    token_type: { type: string, enum: [Bearer] }
    expires_in: { type: integer }
```

**2. Trocar referência no endpoint:**

```yaml
# Antes (linha 93)
/auth/refresh:
  post:
    responses:
      "200":
        content:
          application/json:
            schema:
              $ref: "#/components/schemas/LoginResponse"   # ← ERRADO

# Depois
/auth/refresh:
  post:
    responses:
      "200":
        content:
          application/json:
            schema:
              $ref: "#/components/schemas/RefreshResponse"  # ← CORRETO
```

- Valor anterior: `$ref: "#/components/schemas/LoginResponse"` (inclui `user`)
- Valor correto: `$ref: "#/components/schemas/RefreshResponse"` (sem `user`)
- Referência: FR-000-C04 §DEF-002

# FR-001 — Listagem de Usuários com Filtros, Busca e Ações

- **Descrição:** Tela de listagem paginada (cursor-based) de usuários do sistema com filtros por status e perfil, busca textual com debounce de 400ms, e ações contextuais por linha (navegar para convite, desativar). Filtros sincronizados com query params da URL para deep link e preservação ao navegar com "Voltar".
- **Prioridade:** Must
- **Perfis impactados:** Administrador (leitura), Administrador (gestão), Administrador (completo)
- **Done funcional:**
  - [ ] Listagem renderiza tabela com colunas: nome, perfil, status, data de criação
  - [ ] Skeleton exibido durante carregamento inicial
  - [ ] Paginação cursor-based: botão "Carregar mais" com append (nunca substituição)
  - [ ] Busca textual com debounce 400ms — query param `search` sincronizado na URL
  - [ ] Filtro por status (ACTIVE|PENDING|BLOCKED|INACTIVE) — query param `status` sincronizado
  - [ ] Filtro por perfil via select populado por `roles_list` — query param sincronizado
  - [ ] Botão "Limpar filtros" restaura URL sem query params e recarrega
  - [ ] Filtros preservados ao navegar de volta (browser back)
  - [ ] Estado vazio com ilustração + CTA "Criar primeiro usuário" (se scope write)
  - [ ] Estado de erro (GET 500) com botão "Tentar novamente"
  - [ ] Botão "Novo Usuário" visível apenas com scope `users:user:write` (BR-001)
  - [ ] Ação "Reenviar convite" no dropdown apenas para status PENDING (navega para /usuarios/:id/convite)
  - [ ] Ação "Desativar" no dropdown visível apenas com scope `users:user:delete` (BR-001)
  - [ ] Ação "Desativar" indisponível para usuários INACTIVE
  - [ ] Modal de desativação: usa nome, nunca e-mail (BR-002). Botão isLoading durante chamada
  - [ ] Após desativação (200): linha atualizada inline para INACTIVE, modal fechado, toast de sucesso
  - [ ] Erro de desativação (500): toast com correlationId, modal permanece aberto (BR-006)
- **Efeito colateral?** [x] Sim — `users_delete` (desativação via DELETE) *(Idempotência gerida pelo MOD-000-F05 backend)*
- **Idempotência:** n/a (backend MOD-000-F05 garante idempotência do DELETE)

# Emenda: FR-000-C03

- **Documento base:** [FR-000](../../requirements/fr/FR-000.md)
- **estado_item:** DRAFT
- **Natureza:** C (Correção)
- **Data:** 2026-03-25
- **owner:** arquitetura
- **Motivação:** O entry point `apps/api/src/index.ts` contém apenas 3 rotas hardcoded inline (~190 linhas de SQL direto), enquanto 9 módulos exportam ~30 route plugins que nunca são registrados. Isso resulta em 404 para todas as rotas exceto `/info`, `/auth/login` e `/auth/me`. A causa raiz é a ausência de wiring entre módulos e entry point. FR-000-C02 corrigiu o sintoma (adicionou `/auth/me` inline), mas a correção definitiva requer reescrever o entry point para registrar todos os módulos.
- **rastreia_para:** FR-000, FR-000-C02, DOC-ARC-004, DOC-GNP-00-M02

---

## Detalhamento

### Problema — Entry point desconectado dos módulos

O `apps/api/src/index.ts` atual:
- Importa diretamente `drizzle-orm`, `postgres`, `bcrypt` e schemas para queries inline
- Define 3 rotas hardcoded: `GET /info`, `POST /auth/login`, `GET /auth/me`
- Não importa nem registra nenhum dos 9 módulos de rota
- Não registra error handlers
- Não registra decorators de auth/DI

### Correção — Rewrite completo do entry point

O novo entry point:
1. Remove todas as rotas inline e dependências diretas (drizzle, postgres, bcrypt, schemas)
2. Importa route plugins de todos os 9 módulos
3. Importa e registra auth plugin (`verifySession`, `requireScope`)
4. Importa e registra DI plugin (`dipiContainer`)
5. Registra todos os route plugins com prefix correto (DOC-ARC-004 §7)
6. Registra `foundationErrorHandler` como global error handler
7. Adiciona `app.printRoutes()` no startup para validação visual

### Plugins de auth/DI criados

- `apps/api/src/plugins/auth.ts` — Registra `verifySession` e `requireScope` como Fastify decorators
- `apps/api/src/plugins/di.ts` — Registra `dipiContainer` como request decorator e module registries (`caseExecution`, `movementApproval`, `mcpAutomation`)

### O que foi removido do entry point

- Linhas 6-18: imports diretos de drizzle/postgres/bcrypt/schemas

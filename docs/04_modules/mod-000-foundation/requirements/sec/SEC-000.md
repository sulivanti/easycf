## SEC-000 — Segurança e Compliance (Foundation)

- **Autenticação:** JWT customizado com `sessionId` embutido + validação em banco (Kill-Switch). Cookies `httpOnly=true`, `sameSite=lax`. TTL: 12h (sessão normal) ou 30d (`remember_me`).
- **Autorização:** RBAC hierárquico por escopos `modulo:recurso:acao`. DENY explícito prevalece sobre ALLOW. Cache Redis TTL 5min por `(userId, tenantId)`. Validação referencial contra Catálogo de Permissões (F12).
- **Classificação dos Dados:** `Restrito (PII/Financeiro)` — senhas, e-mails, CPFs, tokens de sessão.
- **Política de Retenção:** JWT access token: 12h–30d. Refresh token: acompanha TTL de sessão. Tokens de reset-password: 1h (single-use). Sessions: soft-delete. `domain_events`: retenção de 5 anos.
- **Mascaramento:** Senhas com bcrypt (rounds ≥ 12). `mfa_secret` criptografado em repouso. Campos PII (`email`, `phone`, `cpf`) com `sensitivity_level=2` na `domain_events` — exposição somente para auditor/admin.
- **LGPD:** Minimização de dados no `UIActionEnvelope` (sem PII). `actorId=undefined` em auditoria de falha de login (user enumeration prevention). Soft-delete com TTL de purging.
- **Auditoria:** Trilha completa em `domain_events` para: `auth.login.success/failure/blocked`, `auth.logout.success`, `auth.session.revoked/revoked_all`, `password.reset.requested/completed`, `user.created/updated/deleted`, `role.assigned/revoked`, `branch.created/blocked`.
- **Proteções:**
  - Rate Limiting: login 10 req/15min/IP, forgot-password 3/15min/e-mail (Redis).
  - WAF: headers de segurança via Helmet.js.
  - Secrets: variáveis de ambiente validadas em boot (fail-fast) conforme DOC-PADRAO-004.
  - CORS: configurado por `ALLOWED_ORIGINS` via env.
  - `Idempotency-Key` em todas rotas de mutação com efeito colateral.

### SEC-000-01 — User Enumeration Prevention (MUST)

- Falha de login DEVE sempre retornar HTTP 401 com corpo `{"message": "E-mail ou senha incorretos."}` independente do motivo (e-mail inválido, senha inválida, conta bloqueada — exceto bloqueio de conta, que retorna 403 com type específico).
- `actorId` na auditoria de falha DEVE ser `undefined`; `entityId` DEVE ser `UUID_NULO`.

### SEC-000-02 — Matriz de Autorização de Eventos (SEC-EventMatrix)

| Evento | Emit (permissão do comando) | View (regra) | Notify |
|---|---|---|---|
| `auth.login.success` | público (login) | próprio usuário + admin | Não |
| `auth.session.revoked` | `auth:session:revoke` | próprio usuário + admin | Sim → usuário |
| `user.created` | `users:create` | `canRead(user)` + tenant | Sim → admin |
| `user.deleted` | `users:delete` | auditor/admin | Sim → admin |
| `role.assigned` | `roles:assign` | `canRead(user)` + tenant | Sim → usuário afetado |
| `password.reset.requested` | público (forgot-password) | próprio usuário + admin | Sim → usuário (e-mail) |
| `storage.upload.confirmed` | `storage:upload` | próprio usuário + admin | Não |

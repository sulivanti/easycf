## NFR-001 — Performance de Renderização da Grade

- **Categoria:** Performance (UI)
- **Descrição:** A grade de inclusão em massa (UX-SGR-001) DEVE renderizar de forma fluida até o limite máximo de linhas configurado, sem jank visual ou bloqueio da thread principal.
- **SLO:**

| Métrica | Target | Método de Medição |
|---|---|---|
| First Contentful Paint (mount com config do motor) | <= 1500ms (p95) | Lighthouse / telemetria `capture_duration_ms` |
| Tempo de renderização incremental (adicionar 1 linha) | <= 50ms | Performance API / React Profiler |
| Tempo de renderização do grid completo (N linhas) | <= 300ms para N <= 100 linhas | React Profiler |
| Frame rate durante scroll | >= 30fps | Chrome DevTools Performance |

- **Técnicas recomendadas:**
  - Virtualização de linhas (ex: `react-virtual`, `tanstack-virtual`) para renderizar apenas as linhas visíveis no viewport
  - Memoização de cells e rows (`React.memo`, `useMemo`)
  - Debounce de input para evitar re-renders em cascata

- **estado_item:** DRAFT
- **rastreia_para:** US-MOD-011-F02, FR-002, FR-003

## NFR-002 — Limite Máximo de Linhas por Sessão

- **Categoria:** Capacidade / Limites
- **Descrição:** A grade DEVE impor um limite máximo de linhas por sessão para evitar degradação de performance no client e excesso de chamadas ao motor.
- **SLO:**

| Métrica | Target | Referência |
|---|---|---|
| Limite padrão de linhas | **MAX_GRID_ROWS=200** | FR-005, PEND-SGR-03 |
| Virtualização de linhas | **Obrigatória** (react-virtual / tanstack-virtual) | NFR-001, PEND-SGR-03 |
| Comportamento ao atingir limite | Botão "Adicionar" desabilitado | FR-005 |
| Importação que excede limite | Rejeição com mensagem explicativa | FR-005 |
| Duplicação que excederia limite | Rejeição com mensagem explicativa | BR-007 |

> **PEND-SGR-03 (RESOLVIDA):** Limite padrão definido como 200 linhas (Opção B). Virtualização de linhas via react-virtual/tanstack-virtual é obrigatória para garantir renderização fluida. Se testes de performance indicarem degradação em hardware modesto, reduzir para 100. Configurável por tenant como evolução futura (Opção C).

## NFR-004 — Performance de "Validar Tudo" (Throttling)

- **Categoria:** Performance (Integração)
- **Descrição:** A ação "Validar Tudo" executa N chamadas ao motor (1 por linha — BR-002). Para evitar sobrecarga no servidor e degradação da UI, as chamadas DEVEM ser executadas com throttling (concorrência limitada e configurável).
- **SLO:**

| Métrica | Target | Referência |
|---|---|---|
| Concorrência de chamadas simultâneas | **Configurável via env var `SMARTGRID_CONCURRENCY` (default=10)** | BR-002, BR-003, PEND-SGR-05 |
| Valores recomendados por ambiente | dev=2, staging=5, prod=10 | PEND-SGR-05 |
| Tempo total de "Validar Tudo" para 200 linhas (p95) | <= 10s (200 linhas / 10 concorrentes = 20 batches x 500ms p95) | FR-003, PEND-SGR-03 |
| Feedback visual progressivo | Status de cada linha atualizado conforme resposta chega | FR-003 |
| Progress indicator | Barra de progresso ou contador "X de N validadas" | UX-SGR-001 |

> **PEND-SGR-05 (RESOLVIDA):** Concorrência configurável via environment variable `SMARTGRID_CONCURRENCY` com default=10 (Opção C). Permite tuning sem rebuild. Cálculo: 200 linhas / 10 concorrentes = 20 batches x 500ms p95 = ~10s total. Valores recomendados: dev=2, staging=5, prod=10.

- **Técnicas recomendadas:**
  - Pool de promises com concorrência limitada (ex: `p-limit`, `Promise.allSettled` com batches)
  - Concorrência lida de `SMARTGRID_CONCURRENCY` env var no startup (default=10)
  - Atualização progressiva do status visual conforme cada resposta chega (não esperar todas)
  - Cancelamento via `AbortController` se usuário navegar para fora

- **estado_item:** DRAFT
- **rastreia_para:** US-MOD-011-F02, FR-003, BR-002, BR-003

## NFR-005 — Performance de Export/Import JSON

- **Categoria:** Performance (Client-side)
- **Descrição:** As operações de Export e Import JSON DEVEM ser rápidas e não bloquear a thread principal, mesmo para grades com o número máximo de linhas.
- **SLO:**

| Métrica | Target | Referência |
|---|---|---|
| Tempo de export JSON (100 linhas, 20 colunas) | <= 200ms | FR-004 |
| Tempo de import JSON (100 linhas, 20 colunas) | <= 500ms (inclui parse + populate grid) | FR-004 |
| Tamanho máximo do JSON exportado | Limitado pelo número de linhas (NFR-002) | FR-004, FR-005 |
| Validação do JSON importado | Schema validation antes de popular a grade | FR-004 |

- **Técnicas recomendadas:**
  - `JSON.stringify` / `JSON.parse` nativo (rápido para payloads < 10MB)
  - Para grades muito grandes, considerar `Web Worker` para parse off-thread
  - Download via `Blob` + `URL.createObjectURL` (evita data URI)

- **estado_item:** DRAFT
- **rastreia_para:** US-MOD-011-F02, FR-004

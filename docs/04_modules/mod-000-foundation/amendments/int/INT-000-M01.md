### Nova seção: INT-004 — Contratos REST internos da Users API

**Situação atual (INT-000):** Documenta apenas integrações SSO externas (INT-001 Google, INT-002 Microsoft, INT-003 Apple). Não registra contratos REST internos entre módulos.

**Problema:** MOD-002 (frontend) consome a Users API do MOD-000 (backend). Sem schemas formais em INT-000, o codegen não tem referência para gerar tipos TypeScript, validações Zod, ou mocks de teste.

### INT-004 — Users API (REST — consumida por MOD-002)

- **Sistema:** MOD-000 Foundation (interno)
- **Modo:** REST API (OpenAPI v3.1)
- **Base path:** `/api/v1`

#### Schemas atualizados (conforme FR-000-M01)

**`UserListItem`** — Resposta de `GET /api/v1/users`

```yaml
UserListItem:
  type: object
  required: [id, codigo, email, full_name, status, created_at]
  properties:
    id: { type: string, format: uuid }
    codigo: { type: string }
    email: { type: string, format: email }
    full_name: { type: string }
    status: { type: string, enum: [ACTIVE, PENDING, BLOCKED, INACTIVE] }
    role_id: { type: string, format: uuid, nullable: true }
    role_name: { type: string, nullable: true }
    created_at: { type: string, format: date-time }
```

**`UserDetail`** — Resposta de `GET /api/v1/users/:id`

```yaml
UserDetail:
  type: object
  required: [id, codigo, email, full_name, status, created_at, updated_at]
  properties:
    id: { type: string, format: uuid }
    codigo: { type: string }
    email: { type: string, format: email }
    full_name: { type: string }
    cpf_cnpj: { type: string, nullable: true }
    avatar_url: { type: string, nullable: true }
    status: { type: string, enum: [ACTIVE, PENDING, BLOCKED, INACTIVE] }
    invite_token_expired: { type: boolean, nullable: true }
    created_at: { type: string, format: date-time }
    updated_at: { type: string, format: date-time }
```

**`CreateUserRequest`** — Body de `POST /api/v1/users`

```yaml
CreateUserRequest:
  type: object
  required: [email, full_name, mode]
  properties:
    email: { type: string, format: email, maxLength: 255 }
    full_name: { type: string, minLength: 1, maxLength: 255 }
    cpf_cnpj: { type: string, maxLength: 20 }
    mode:
      type: string
      enum: [invite, password]
    password:
      type: string
      minLength: 8
      maxLength: 128
      description: "Required when mode='password'. Ignored when mode='invite'."
    force_pwd_reset:
      type: boolean
      default: true
    role_id:
      type: string
      format: uuid
      description: "Optional. Creates tenant_users binding atomically."
```

#### Validação condicional

| Condição | Resultado |
|----------|-----------|
| `mode='invite'` + `password` presente | `422` — "Password must not be sent when mode is 'invite'" |
| `mode='password'` + `password` ausente | `422` — "Password is required when mode is 'password'" |
| `role_id` + role inexistente | `422` — "Role not found" |
| `role_id` + sem auth | `403` — "Role assignment requires authentication" |

#### Timeout e retry

| Endpoint | Timeout | Retry | Idempotente? |
|----------|---------|-------|-------------|
| `GET /users` | 5000ms | 1x | Sim (GET) |
| `GET /users/:id` | 5000ms | 1x | Sim (GET) |
| `POST /users` | 5000ms | 0 | Não (criação) |
| `POST /users/:id/invite/resend` | 5000ms | 0 | Não (side-effect: email) |

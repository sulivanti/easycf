## Impacto nos Pilares

- **DATA:** Sem impacto direto — os campos de dados existentes (DTOs) ja suportam as informacoes exibidas. Porem, para exibir nomes de usuarios em vez de UUIDs, os endpoints de listagem (GET /admin/access-shares, GET /access-delegations) DEVEM retornar objetos expandidos com `grantee_name`, `delegatee_name`, `delegator_name` em vez de apenas IDs. Isso pode requerer FR-001-M01 para enriquecer DTOs de resposta.
- **FR:** FR-001-M01 potencialmente necessario — endpoints de listagem precisam retornar nomes de usuarios (expand/populate) e suportar query parameter `q` para autocomplete de usuarios (GET /admin/users?q=). Verificar se endpoint de listagem de usuarios ja existe em MOD-002.
- **UX (manifests):** ux-idn-001.org-scope.yaml e ux-idn-002.shares-delegations.yaml precisam de atualizacao para refletir novos componentes (ScopePill, TreeSelector, TabBar, Banner) e layout reestruturado.
- **BR:** Sem impacto — regras de negocio existentes (BR-001) permanecem inalteradas.
- **SEC:** Sem impacto — scopes existentes cobrem todas as operacoes. Nenhum novo scope necessario.

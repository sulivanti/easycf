## UX-001 — Shell de Autenticação (UX-AUTH-001)

- **Screen ID:** UX-AUTH-001
- **Manifest:** `docs/05_manifests/screens/ux-auth-001.login.yaml`
- **Entidade(s):** `session`, `user`
- **Contexto:** Tela pública de login com 3 painéis: login, forgot-password, reset-password
- **Ações disponíveis (UX-010):** `[submit_login, submit_forgot_password, submit_reset_password, navigate_to_forgot, navigate_to_login]`

### Mapeamento Ações → Endpoints → Domain Events

| action_id | kind | scope | Endpoint | Domain Event (MOD-000) | Scope requerido |
|---|---|---|---|---|---|
| `submit_login` | command | single | `POST /api/v1/auth/login` | `auth.login_success` / `auth.login_failed` | público |
| `submit_forgot_password` | command | single | `POST /api/v1/auth/forgot-password` | `auth.password_reset_requested` | público |
| `submit_reset_password` | command | single | `POST /api/v1/auth/reset-password` | `auth.password_reset_completed` | público (via token) |
| `navigate_to_forgot` | view | single | — (client-only) | — | público |
| `navigate_to_login` | view | single | — (client-only) | — | público |

### Jornada (Happy Path) — Login

1. Usuário acessa `/login` → painel `login` ativo
2. Preenche e-mail e senha → clica "Entrar"
3. Botão entra em isLoading (BR-003)
4. POST /auth/login (auth_login) via X-Correlation-ID
5. UIActionEnvelope: submit_login (requested → succeeded)
6. Resposta 200 → redirect /dashboard + Shell monta

### Jornada (Happy Path) — Recuperação de Senha

1. No painel login, clica "Esqueci minha senha"
2. Transição client-only para painel forgot-password (BR-007)
3. UIActionEnvelope: navigate_to_forgot (ui_only)
4. Preenche e-mail → clica "Enviar link"
5. POST /auth/forgot-password (auth_forgot_password)
6. Toast genérico positivo (BR-002)
7. Link "Voltar ao login" → retorna ao painel login

### Jornada (Happy Path) — Reset de Senha

1. Usuário acessa `/login?token=<uuid>` → painel reset-password ativo automaticamente
2. Preenche nova_senha + confirmar_senha → clica "Redefinir"
3. POST /auth/reset-password (auth_reset_password)
4. Resposta 200 → Toast sucesso + retorno ao painel login

### Alternativas e Erros

| HTTP Status | Contexto | Comportamento UI | Referência |
|---|---|---|---|
| 401 | Login inválido | Toast "E-mail ou senha inválidos." + correlationId | BR-001, BR-004 |
| 403 | Conta bloqueada | Toast "Sua conta está bloqueada." + correlationId | FR-001 |
| 429 | Rate limit | Toast "Muitas tentativas. Tente em X s." + botão desabilitado | FR-001 |
| 200 + mfa_required | MFA ativo | Toast "Autenticação multifator requerida. Contate o administrador." + NÃO redirecionar (fallback PENDENTE-004 Opção B) | FR-001 |
| 400/422 | Token reset expirado | Toast "Link inválido ou expirado." + link forgot | FR-003 |

### Estados

- **Loading:** Botão com isLoading (spinner + desabilitado) durante qualquer submit
- **Empty:** N/A (tela fixa com campos)
- **Error:** Toast com correlationId (DOC-UX-012 §2.1) — nunca detalhes técnicos

## UX-002 — Application Shell (UX-SHELL-001)

- **Screen ID:** UX-SHELL-001
- **Manifest:** `docs/05_manifests/screens/ux-shell-001.app-shell.yaml`
- **Entidade(s):** `user`, `tenant`
- **Contexto:** Layout wrapper das rotas autenticadas (Sidebar + Header + Breadcrumb + ContentArea)
- **Ações disponíveis (UX-010):** `[load_current_user, submit_logout, submit_change_password, navigate_sidebar, navigate_breadcrumb]`

### Mapeamento Ações → Endpoints → Domain Events

| action_id | kind | scope | Endpoint | Domain Event (MOD-000) | Scope requerido |
|---|---|---|---|---|---|
| `load_current_user` | query | single | `GET /api/v1/auth/me` | — (leitura) | autenticado |
| `submit_logout` | command | single | `POST /api/v1/auth/logout` | `auth.logout` | autenticado |
| `submit_change_password` | command | single | `POST /api/v1/auth/change-password` | `auth.password_changed` | autenticado |
| `navigate_sidebar` | view | single | — (client-only) | — | autenticado |
| `navigate_breadcrumb` | view | single | — (client-only) | — | autenticado |

### Jornada (Happy Path)

1. Após login, Shell monta e dispara GET /auth/me (auth_me)
2. UIActionEnvelope: load_current_user (requested)
3. Skeleton exibido no Header (avatar, nome, tenant) e Sidebar (itens de menu)
4. auth_me retorna → Skeleton substitudo por dados reais
5. Sidebar populada por `auth_me.scopes[]` (BR-005)
6. Header/ProfileWidget exibe nome, e-mail, tenant

### Componentes

| Componente | Dados | Fonte | Ações |
|---|---|---|---|
| Sidebar | Itens de menu filtrados por scopes | auth_me.scopes[] | navigate_sidebar (client_only) |
| Header/Breadcrumb | Path atual | React Router | navigate_breadcrumb (client_only) |
| ProfileWidget | Nome, e-mail, tenant | auth_me | "Alterar Senha" (submit_change_password — FR-007), "Sair" (submit_logout) |

### Alternativas e Erros

| Cenário | Comportamento UI | Referência |
|---|---|---|
| auth_me retorna 401 | Interceptor redireciona para /login + Toast "Sessão expirada" | FR-004 |
| auth_me retorna 5xx | Toast erro + correlationId (Shell parcialmente carregado) | BR-004 |
| Logout com erro de rede | Limpar estado local + redirect /login | INT-002 |
| Scopes vazios (scopes=[]) | Sidebar exibe "Nenhum módulo configurado para seu perfil." com ícone Info | FR-004, PENDENTE-002 |

### Estados

- **Loading:** Skeleton no Header (avatar, nome, tenant) + Sidebar (itens de menu) + ContentArea
- **Empty:** Quando `auth_me.scopes=[]`, Sidebar exibe item placeholder com ícone `Info` e texto "Nenhum módulo configurado para seu perfil." (PENDENTE-002 §Opção B). Header/ProfileWidget permanece funcional (logout, alterar senha).
- **Error:** Toast com correlationId — Shell permanece visível com estado parcial

## UX-003 — Dashboard Executivo (UX-DASH-001)

- **Screen ID:** UX-DASH-001
- **Manifest:** `docs/05_manifests/screens/ux-dash-001.main.yaml`
- **Entidade(s):** `user`, `tenant`
- **Contexto:** Tela inicial pós-login dentro do Shell
- **Ações disponíveis (UX-010):** `[load_dashboard_profile, dashboard_skeleton_timeout, dashboard_retry]`

### Mapeamento Ações → Endpoints → Domain Events

| action_id | kind | scope | Endpoint | Domain Event (MOD-000) | Scope requerido |
|---|---|---|---|---|---|
| `load_dashboard_profile` | query | single | `GET /api/v1/auth/me` | — (leitura) | autenticado |
| `dashboard_skeleton_timeout` | view | single | — (client_only) | — | autenticado |
| `dashboard_retry` | query | single | `GET /api/v1/auth/me` | — (leitura) | autenticado |

### Jornada (Happy Path)

1. Dashboard monta dentro do Shell (UX-SHELL-001)
2. Dispara GET /auth/me (auth_me) — pode reutilizar cache do Shell se disponível
3. UIActionEnvelope: load_dashboard_profile (requested)
4. Skeleton exibido: WelcomeWidget (placeholder texto) + ModuleShortcuts (4 cards skeleton)
5. auth_me retorna → Skeleton substituído por dados reais
6. WelcomeWidget: "Bom dia/Boa tarde/Boa noite, {nome}!" + subtexto "{tenant}"
7. ModuleShortcuts: cards filtrados por scopes

### Componentes

| Componente | Dados | Fonte | Lógica |
|---|---|---|---|
| WelcomeWidget | Saudação + nome + tenant | auth_me | Horário local: 5h–12h=dia, 12h–18h=tarde, 18h–5h=noite |
| ModuleShortcuts | Cards filtrados | auth_me.scopes[] | Catálogo: users:user:read, users:role:read, tenants:branch:read, system:audit:read |

### Catálogo de Atalhos (seed MOD-001)

| Card | Label | Rota | Scope Necessário | Ícone |
|---|---|---|---|---|
| shortcut-users | Usuários | /usuarios | `users:user:read` | Users |
| shortcut-roles | Perfis e Permissões | /perfis | `users:role:read` | Shield |
| shortcut-branches | Filiais | /filiais | `tenants:branch:read` | Building |
| shortcut-audit | Auditoria | /auditoria | `system:audit:read` | FileSearch |

### Jornada — Timeout e Retry (BR-009, BR-010)

1. Dashboard monta e dispara `load_dashboard_profile` (requested)
2. Skeleton exibido: WelcomeWidget (placeholder) + 4 cards skeleton
3. Se auth_me NÃO retorna em 3s:
   - UIActionEnvelope: `dashboard_skeleton_timeout` (client_only) — nenhuma requisição HTTP
   - Skeleton substituído por estado de erro parcial (nunca tela branca — NFR-001 §5)
   - Toast "Erro ao carregar dados. Tente novamente." com correlationId
   - Botão "Tentar novamente" visível
4. Usuário clica "Tentar novamente":
   - UIActionEnvelope: `dashboard_retry` (requested → succeeded/failed) com **novo** correlation_id UUID v4
   - Novo GET /auth/me disparado
   - Se succeeded → renderiza dados normalmente
   - Se failed (5xx) → Toast com correlationId + botão retry, SEM desconectar (BR-010)
   - Se failed (401) → redirect /login via interceptor

### Alternativas e Erros

| Cenário | Comportamento UI | UIActionEnvelope | Referência |
|---|---|---|---|
| auth_me timeout > 3s | Skeleton → estado de erro + Toast + botão retry | `dashboard_skeleton_timeout` (client_only) | BR-009, NFR-001 |
| auth_me 5xx | Toast erro + correlationId + botão retry (NÃO desconectar) | `load_dashboard_profile` (failed) | BR-010, FR-005 |
| auth_me 401 | Redirect /login via interceptor | — (interceptor atua antes) | FR-004, SEC-001 |
| Retry após timeout/5xx | Novo GET /auth/me com novo correlation_id | `dashboard_retry` (requested → succeeded/failed) | BR-009, BR-010, DATA-003 |
| Sem scopes (scopes=[]) | WelcomeWidget normal + mensagem "Nenhum módulo disponível" | `load_dashboard_profile` (succeeded) | FR-005 |

### Estados

- **Loading:** Skeleton animado: WelcomeWidget (texto placeholder) + 4 cards skeleton
- **Empty:** "Nenhum módulo disponível para seu perfil. Contate o administrador." (scopes=[])
- **Error:** Skeleton → estado de erro parcial (nunca tela branca) + Toast + retry (BR-009, NFR-001 §5)
- **Retry:** Re-exibe skeleton durante novo GET /auth/me; transiciona para Loading/Error/Success (BR-009, BR-010)

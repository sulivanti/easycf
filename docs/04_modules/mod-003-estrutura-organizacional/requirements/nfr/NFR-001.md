## 2. Performance

| Métrica | Alvo | Observações |
|---|---|---|
| `GET /org-units/tree` latência p95 | < 200ms | Com até 100 nós, cache de árvore invalidado em alterações |
| `GET /org-units` listagem flat p95 | < 150ms | Cursor-based pagination, índices parciais |
| `GET /org-units/:id` detalhe p95 | < 100ms | Inclui ancestors/breadcrumb |
| `POST /org-units` criação p95 | < 300ms | Inclui validação CTE anti-loop + domain event |
| `PATCH /org-units/:id` atualização p95 | < 250ms | Inclui domain event |
| `DELETE /org-units/:id` soft delete p95 | < 250ms | Inclui validação filhos ativos + domain event |
| `PATCH /org-units/:id/restore` p95 | < 250ms | Inclui validação pai ativo + domain event |
| `POST /org-units/:id/tenants` link p95 | < 200ms | Inclui validação nível N4 + domain event |
| CTE recursivo max depth | 4 níveis | N1→N2→N3→N4 (N5 via JOIN tenants) |

- **Paginação:** Cursor-based para listagem flat (`GET /org-units`). Tree query retorna tudo em uma chamada.
- **Cache:** Cache de árvore invalidado apenas em operações de escrita (create/update/delete/restore/link/unlink).
- **Topology:** `sync` — todas as operações são síncronas (sem filas/workers).

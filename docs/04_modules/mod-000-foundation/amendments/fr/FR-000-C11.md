## Detalhamento

### Requisito Afetado: FR-007 — Gestão de Roles/RBAC (Seed Sync)

**3 alterações complementares que fecham o pipeline de validação:**

### 1. Validação de scopes no seed ANTES de inserir (REQ-001)

O `seed-admin.ts` DEVE importar `Scope` e validar cada item do array `SCOPES` via `Scope.create()` antes de qualquer operação no banco. Se algum scope falhar, o seed DEVE abortar com mensagem de erro explícita e `process.exit(1)`.

O array `SCOPES` DEVE ser exportado como constante nomeada para permitir importação nos testes.

```typescript
// Após definição do array SCOPES, antes de qualquer DB operation:
import { Scope } from '../src/modules/foundation/domain/value-objects/scope.vo.js';

const invalidScopes: string[] = [];
for (const s of SCOPES) {
  try { Scope.create(s); } catch { invalidScopes.push(s); }
}
if (invalidScopes.length > 0) {
  console.error(`❌ ${invalidScopes.length} scope(s) inválido(s) no seed:`);
  invalidScopes.forEach((s) => console.error(`   - "${s}"`));
  process.exit(1);
}
```

### 2. Remover normalização band-aid do Scope.create() (REQ-002)

A linha `replace(/-/g, '_')` no `scope.vo.ts` DEVE ser removida. Scopes com hífens devem ser **rejeitados** (DomainValidationError), não normalizados silenciosamente. A tabela RENAMES no seed já resolve migração de dados legados.

```typescript
// ANTES (band-aid):
const trimmed = raw.trim().replace(/-/g, '_');

// DEPOIS (validação estrita):
const trimmed = raw.trim();
```

### 3. Teste de consistência seed ↔ VO (REQ-003, REQ-004)

Dois novos testes unitários:

```gherkin
Cenário: Todos os scopes do seed passam validação do VO (REQ-003)
  Dado que o array SCOPES contém 50+ scopes do catálogo canônico
  Quando cada scope é validado via Scope.create()
  Então nenhum lança DomainValidationError

  E a mensagem indica o formato canônico esperado
```

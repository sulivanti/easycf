# Emenda: FR-001-C01

- **Documento base:** [FR-005 §FR-001](../../requirements/fr/FR-005.md)
- **estado_item:** MERGED
- **Natureza:** C (Correção)
- **Data:** 2026-03-28
- **owner:** Marcos Sulivan
- **Motivação:** Handler de `POST /admin/cycles/:id/deprecate` envia `nome: null` no response body, mas o Zod schema `cycleResponse` declara `nome: z.string()` (não-nullable). Adicionalmente, os handlers de deprecate, publish, create, update e fork usam `new Date().toISOString()` para `created_at`/`updated_at` em vez dos timestamps reais do resultado do use case. Todos os handlers de ações de ciclo devem usar os dados reais retornados pelo use case.
- **rastreia_para:** spec-fix-cycle-response-schema-mismatch, FR-001, FR-002, FR-003, FR-004

## Detalhamento

### BUG-002: Deprecate envia `nome: null`

**Antes (errado)** — `cycles.route.ts`, handler `POST /cycles/:id/deprecate`:
```typescript
return reply.send({
  // ...
  nome: null,       // ← ERRO: cycleResponse.nome é z.string(), não nullable
  descricao: null,  // ← OK (nullable no schema)
  // ...
});
```

**Depois (correto):**
```typescript
return reply.send({
  id: result.id,
  tenant_id: request.session.tenantId,
  codigo: result.codigo,
  nome: result.nome,               // ← string real
  descricao: result.descricao,     // ← valor real
  version: result.version,
  status: result.status,
  parent_cycle_id: result.parentCycleId ?? null,
  published_at: result.publishedAt?.toISOString() ?? null,
  created_by: result.createdBy ?? request.session.userId,
  created_at: result.createdAt.toISOString(),
  updated_at: result.updatedAt.toISOString(),
});
```

### Melhoria associada: Handlers de publish, create, update, fork

Os handlers de `POST /cycles/:id/publish`, `POST /cycles`, `PATCH /cycles/:id` e `POST /cycles/:id/fork` usam `new Date().toISOString()` para timestamps em vez dos valores reais do use case. Devem ser corrigidos para usar `result.createdAt.toISOString()` e `result.updatedAt.toISOString()` quando disponíveis, e `result.descricao` em vez de `null` hardcoded.

## Detalhamento

### Melhoria: Criar componente ComingSoonPage e rotas placeholder

#### 1. Componente shared `ComingSoonPage`

Criar em `apps/web/src/shared/ui/ComingSoonPage.tsx`:

- Ícone `Construction` (Lucide)
- Título: "Módulo em construção"
- Subtexto: "Esta funcionalidade está sendo desenvolvida e estará disponível em breve."
- Botão "Voltar ao Dashboard" → navega para `/dashboard`
- Seguir Design System (DOC-UX-013) — tokens de cor, tipografia, espaçamento
- Layout centralizado verticalmente no content area

#### 2. Route files placeholder

Criar os seguintes route files importando `ComingSoonPage`:

| Route File | Path | Módulo alvo |
|-----------|------|-------------|
| `_auth.usuarios.tsx` | `/usuarios` | MOD-002 (Gestão de Usuários) |
| `_auth.perfis.tsx` | `/perfis` | MOD-002 (Gestão de Usuários) |
| `_auth.filiais.tsx` | `/filiais` | MOD-003 (Estrutura Organizacional) |
| `_auth.auditoria.tsx` | `/auditoria` | MOD-000 (Foundation — Auditoria) |

Cada route file segue o padrão:

```typescript
import { createRoute } from '@tanstack/react-router';
import { Route as authRoute } from './_auth';
import { ComingSoonPage } from '@shared/ui/ComingSoonPage';

export const Route = createRoute({
  path: '/{rota}',
  getParentRoute: () => authRoute,
  component: ComingSoonPage,
});
```

#### 3. Ciclo de vida

Quando o codegen de MOD-002, MOD-003, etc. for executado, os route files placeholder serão **substituídos** pelos route files reais que importam as Page Components do módulo correspondente.

## 2. UX-PROC-001 — Editor Visual de Fluxo

### 2.1 Jornada Principal (Happy Path)

```
1. Analista acessa /processos/ciclos/:id/editor
2. GET /admin/cycles/:id/flow carrega grafo completo
3. Canvas renderiza nós (estágios) em swimlanes (macroetapas)
4. Analista cria/edita/conecta estágios graficamente
5. Analista publica ciclo quando pronto
6. Canvas entra em modo readonly
```

### 2.2 Estados da Tela

| Estado | Comportamento | Componente |
|---|---|---|
| **loading** | Skeleton de nós e arestas placeholder no canvas | Skeleton nodes + edges |
| **loaded** | Nós e arestas renderizados nas posições salvas (`canvas_x`, `canvas_y`) | React Flow canvas |
| **empty** | Canvas vazio com prompt: "Dê duplo clique para criar o primeiro estágio" | EmptyState com ícone de fluxo |
| **error** | Toast de erro + canvas com última versão em cache (se disponível) | Toast RFC 9457 + correlationId |
| **readonly** | Banner "Ciclo publicado — use 'Nova versão' para editar" + nenhuma ação de escrita | Banner + botão "Nova versão" |

### 2.3 Ações (mapeadas para DOC-UX-010)

| action_id (UX-010) | label_pt | Contexto MOD-005 | endpoint / operationId | domain_event |
|---|---|---|---|---|
| `view` | Carregar editor | Carrega grafo do ciclo | GET /admin/cycles/:id/flow → `admin_cycles_flow` | — |
| `create` | Criar estágio | Duplo clique no canvas | POST /admin/macro-stages/:mid/stages → `admin_stages_create` | `process.stage_created` |
| `create` | Criar transição | Drag de aresta entre nós | POST /admin/stage-transitions → `admin_transitions_create` | `process.transition_created` |
| `update` | Mover estágio | Drag de nó no canvas | PATCH /admin/stages/:id → `admin_stages_update` (canvas_x/y) | — |
| `delete` | Deletar estágio | Delete/Backspace no nó selecionado | DELETE /admin/stages/:id → `admin_stages_delete` | — |
| `delete` | Deletar transição | Delete/Backspace na aresta selecionada | DELETE /admin/stage-transitions/:id → `admin_transitions_delete` | — |
| `publish` | Publicar ciclo | Botão "Publicar" no header | POST /admin/cycles/:id/publish → `admin_cycles_publish` | `process.cycle_published` |
| `clone` | Fork (nova versão) | Botão "Nova versão" em PUBLISHED | POST /admin/cycles/:id/fork → `admin_cycles_fork` | `process.cycle_forked` |

### 2.4 Alternativas e Erros

| Situação | Comportamento UX |
|---|---|
| Publicar sem estágio inicial | Toast erro: "O ciclo precisa de ao menos um estágio inicial antes de ser publicado." + nós piscam no canvas |
| Deleção de estágio com instâncias | Modal erro: "Este estágio possui N instância(s) ativa(s) em andamento." + badge ⚠ no nó |
| Criar segundo estágio inicial | Toast erro: "Este ciclo já possui um estágio inicial. Desmarque o atual antes de definir outro." |
| MOD-006 indisponível na deleção | Toast erro: "Não foi possível verificar instâncias ativas. Tente novamente." (503) |
| Ciclo PUBLISHED + tentativa de edição | Bloqueio na UI: canvas readonly, sem ações de escrita |

### 2.5 Componentes Específicos

| Componente | Descrição | Condição de exibição |
|---|---|---|
| **Swimlane** | Faixa horizontal por macroetapa, cor distinta (até 8 predefinidas) | Sempre (1 por macroetapa) |
| **StageNode** | Nó do estágio com nome, badges (gates count, papéis count, ⚑ initial, ⊠ terminal) | Sempre |
| **TransitionEdge** | Aresta com label = nome da transição; laranja se `gate_required=true` | Sempre |
| **MiniMap** | Mini-mapa no canto inferior direito | `nós > 15` |
| **PublishModal** | "Ao publicar, o ciclo se tornará imutável. Continuar?" | Ação `publish` |
| **DeleteConfirmModal** | "Deseja remover o estágio '{nome}'?" | Ação `delete` em nó |
| **ReadonlyBanner** | "Ciclo publicado — use 'Nova versão' para editar" | `cycle.status === PUBLISHED` |

## 3. UX-PROC-002 — Configurador de Estágio (Painel Lateral)

### 3.1 Jornada Principal (Happy Path)

```
1. Analista clica ⚙ em um nó no canvas (UX-PROC-001)
2. Painel lateral desliza da direita com título "{codigo} — {nome}"
3. Aba "Informações" ativa por padrão
4. Analista navega entre 4 abas: Info, Gates, Papéis, Transições
5. Edições são salvas via auto-save (debounce 800ms) ou botão "Salvar"
6. Mudanças refletem imediatamente no canvas (sincronização bidirecional)
```

### 3.2 Estados da Tela

| Estado | Comportamento | Componente |
|---|---|---|
| **loading** | Spinner na aba ativa enquanto carrega dados do estágio | Spinner inline |
| **loaded** | 4 abas com dados populados | Tabs com conteúdo |
| **saving** | Indicador "Salvando..." no header do painel (debounce 800ms) | AutoSaveIndicator |
| **error** | Toast de erro + campo reverte ao valor anterior | Toast RFC 9457 |
| **readonly** | Banner "Ciclo publicado — somente leitura." + sem controles de edição | ReadonlyBanner |

## 4. Sincronização Bidirecional Canvas ↔ Painel

| Ação no Painel | Reflexo no Canvas |
|---|---|
| Salvar nome do estágio | Nó atualiza label em tempo real |
| Adicionar/remover gate | Badge de gates no nó incrementa/decrementa |
| Vincular/desvincular papel | Badge de papéis no nó incrementa/decrementa |
| Adicionar transição de saída | Aresta aparece no canvas |
| Remover transição | Aresta desaparece do canvas |
| Toggle is_initial | Badge ⚑ aparece/desaparece no nó |
| Toggle is_terminal | Badge ⊠ aparece/desaparece no nó |

| Ação no Canvas | Reflexo no Painel |
|---|---|
| Selecionar outro nó | Painel recarrega dados do novo estágio |
| Deletar nó selecionado | Painel fecha automaticamente |
| Criar transição via drag | Se painel aberto para o from_stage, aba Transições atualiza |

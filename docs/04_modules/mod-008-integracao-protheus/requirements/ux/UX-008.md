## 2. UX-INTEG-001 — Editor de Rotinas de Integração

- **Screen ID:** UX-INTEG-001
- **Entidade(s) principal(is):** `integration_routine`, `integration_field_mapping`, `integration_param`
- **Contexto:** Split-view (lista + editor com 3 abas)
- **Rota:** `/integracoes/rotinas`

### 2.1 Jornada Principal (Happy Path)

```
1. Arquiteto de Integração acessa /integracoes/rotinas
2. GET /admin/routines?type=INTEGRATION carrega lista filtrada
3. Lista exibe rotinas INTEGRATION com coluna "Serviço destino"
4. Arquiteto clica em rotina → split-view abre à direita
5. Aba "Config. HTTP" ativa por padrão
6. Arquiteto configura serviço, método, endpoint, retry, trigger events
7. Salva config → aba "Mapeamentos": configura campos source→target com drag-and-drop
8. Aba "Parâmetros": configura params técnicos (sensíveis mascarados)
9. Botão "Testar agora (HML)": executa chamada de teste em ambiente HML
10. Resultado inline (status, response, duration) exibido na aba
11. Quando pronto: publica rotina → editor entra em modo readonly
```

| Estado | Comportamento | Componente |
|---|---|---|

## 4. Observacoes de Performance (MUST)

- **Auto-refresh:** Polling leve (30s para logs RUNNING/QUEUED, 60s para metricas e DLQ count) — evitar polling agressivo.
- **Paginacao cursor-based:** Obrigatoria para tabela de logs (alto volume). Max 100 registros por pagina.
- **Split-view:** Detalhe carregado sob demanda (GET /logs/:id) — nao pre-carregado.
- **Optimistic update:** Badge DLQ decrementa imediatamente ao confirmar reprocessamento — revert se POST falhar.

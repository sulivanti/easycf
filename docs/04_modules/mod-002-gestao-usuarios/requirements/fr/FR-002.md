# FR-002 — Formulário de Cadastro de Usuário (Dois Modos)

- **Descrição:** Formulário de criação de usuário com dois modos mutuamente exclusivos via abas: "Enviar Convite" (padrão) e "Senha Temporária". Validação inline por campo, select de perfil carregado via API, indicador de força de senha client-side no modo senha, e idempotência via `Idempotency-Key` gerado no mount. Redirect automático 1.5s após sucesso.
- **Prioridade:** Must
- **Perfis impactados:** Administrador (gestão), Administrador (completo)
- **Done funcional:**
  - [ ] Acesso a /usuarios/novo requer scope `users:user:write` — redirect sem ele (BR-001)
  - [ ] Select "Perfil de acesso" carregado via `roles_list` com skeleton durante loading
  - [ ] Falha ao carregar roles: Toast + correlationId, campo com retry inline (BR-006)
  - [ ] Modo padrão: "Enviar Convite" — campos de senha ocultos (BR-003)
  - [ ] Troca de modo reseta todo o formulário (campos + erros) (BR-003)
  - [ ] Modo convite: POST com `{ fullName, email, roleId, mode: "invite" }` (BR-003)
  - [ ] Modo senha: POST com `{ fullName, email, roleId, password, forcePasswordReset: true }` (BR-003)
  - [ ] Indicador de força de senha client-side (Fraca/Média/Forte) — sem chamada à API
  - [ ] Validação de confirmação de senha: "As senhas não coincidem."
  - [ ] Campos obrigatórios em branco: botão submit disabled + mensagem inline
  - [ ] E-mail inválido: validação onBlur com mensagem inline
  - [ ] Erro 409 (e-mail duplicado): inline no campo e-mail, sem toast (BR-006)
  - [ ] Erro 422 (invalid_fields): inline por campo + foco no primeiro erro (BR-006)
  - [ ] Erro 500: toast + correlationId, formulário preservado para retry (BR-006)
  - [ ] `Idempotency-Key`: UUID v4 no header, double-click = 1 requisição (BR-005)
  - [ ] Toast sucesso convite: "Usuário criado com sucesso. Convite enviado." — sem e-mail (BR-002)
  - [ ] Toast sucesso senha: "Usuário criado com sucesso." — sem e-mail (BR-002)
  - [ ] Redirect para /usuarios após 1.5s de sucesso
  - [ ] Cancelar: descarta formulário, nenhuma chamada API, redirect para /usuarios

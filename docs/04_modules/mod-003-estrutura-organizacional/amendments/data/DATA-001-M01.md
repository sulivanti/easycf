## Detalhamento

### Novos Campos na Tabela `org_units`

| Campo | Tipo DB | Nulidade | Default | Constraints | Descricao |
|---|---|---|---|---|---|
| `cnpj` | varchar(18) | NULL | -- | -- | CNPJ formatado (xx.xxx.xxx/xxxx-xx). Opcional — nem toda unidade tem CNPJ proprio. |
| `razao_social` | varchar(300) | NULL | -- | -- | Razao social completa da entidade juridica |
| `filial` | varchar(100) | NULL | -- | -- | Localizacao/filial (ex: "Sao Paulo - SP") |
| `responsavel` | varchar(200) | NULL | -- | -- | Nome do responsavel pela unidade |
| `telefone` | varchar(20) | NULL | -- | -- | Telefone de contato formatado |
| `email_contato` | varchar(254) | NULL | -- | -- | E-mail de contato da unidade (RFC 5321) |

### Justificativas

1. **Todos os campos sao NULL** — unidades N1/N2 podem nao ter CNPJ proprio; campos sao preenchidos progressivamente
2. **Sem UNIQUE em CNPJ** — multiplas unidades podem compartilhar o CNPJ da matriz; unicidade nao e requisito de negocio aqui
3. **varchar(18) para CNPJ** — formato "xx.xxx.xxx/xxxx-xx" tem exatamente 18 caracteres
4. **varchar(254) para email** — limite RFC 5321 padrao

# ADR-004 — Vinculação SSO com Confirmação de Senha Nativa (Identity Linking)

- **Contexto:** O Foundation provê login via SSO OAuth2 (Google, Microsoft, Apple). Quando o e-mail retornado pelo provider já existe no sistema com conta nativa (senha), surge a questão de como vincular: automaticamente (risco de account takeover), com confirmação (passo extra), ou sem vínculo (contas separadas). OWASP recomenda que identity linking exija prova de posse da conta destino. Analisada em PENDENTE-001.

- **Decisão:** Adotar **Opção B — Confirmação via senha nativa**. O fluxo SSO, ao detectar e-mail já cadastrado com conta nativa, DEVE solicitar a senha nativa do usuário antes de vincular o provider SSO à conta existente. O fluxo é: callback SSO → detecta e-mail existente → redirect para tela de confirmação → usuário informa senha nativa → bcrypt compare → vincula `sso_provider` + `sso_provider_id` à conta existente → emite JWT de sessão.

- **Alternativas:**
  - **Opção A — Vinculação automática:** Menor fricção, mas vulnerável a account takeover se o provider SSO for comprometido ou se o atacante controlar um domínio de e-mail. Viola princípio OWASP de prova de posse.
  - **Opção C — Contas separadas com merge admin:** Elimina risco automático, mas gera duplicação de dados, complexidade operacional para o admin, e confusão para o usuário que tem dois cadastros com o mesmo e-mail.

- **Consequências:**
  - UX: Passo adicional apenas na **primeira** vinculação SSO (logins subsequentes são diretos via `sso_provider_id`)
  - Segurança: Account takeover mitigado — atacante precisaria saber a senha nativa para vincular
  - Implementação: Frontend precisa de tela intermediária (`/sso-link-confirm`) para captura de senha nativa
  - Endpoint novo ou fluxo no callback: `POST /auth/sso/link-confirm` aceita `{temp_sso_token, password}`, valida bcrypt, vincula provider e cria sessão
  - Evento adicional: `auth.sso_linked` com `user_id`, `provider`, `correlation_id` (sensitivity_level=1)
  - FR-016 atualizado para referenciar este fluxo em vez de "ver PENDENTE-001"

- **Status:** accepted

## 2. UX-PROC-001 — Editor Visual de Fluxo

### 2.1 Jornada Principal (Happy Path)

```
1. Analista acessa /processos/ciclos/:id/editor
2. GET /admin/cycles/:id/flow carrega grafo completo
3. Canvas renderiza nós (estágios) em swimlanes (macroetapas)
4. Analista cria/edita/conecta estágios graficamente
5. Analista publica ciclo quando pronto
6. Canvas entra em modo readonly
```

#### Fluxo Alternativo — Primeiro Estágio (Canvas Vazio)

```
1. Analista acessa /processos/ciclos/:id/editor (ciclo DRAFT recém-criado)
2. GET /admin/cycles/:id/flow retorna grafo vazio (macro_stages: [])
3. Canvas exibe estado empty: "Dê duplo clique para criar o primeiro estágio"
4. Analista dá duplo clique no canvas
5. Sistema auto-cria macroetapa padrão ("Etapa Geral") via POST /admin/cycles/:cid/macro-stages
6. Sistema cria estágio "Novo estágio" com is_initial=true na posição do clique via POST /admin/macro-stages/:mid/stages
7. Canvas refetch → nó aparece na posição do clique
8. Painel de configuração (UX-PROC-002) abre automaticamente para o estágio criado
```

### 2.2 Estados da Tela

| Estado | Comportamento | Componente |
|---|---|---|
| **loading** | Skeleton de nós e arestas placeholder no canvas | Skeleton nodes + edges |
| **loaded** | Nós e arestas renderizados nas posições salvas (`canvas_x`, `canvas_y`) | React Flow canvas |
| **empty** | Canvas vazio com prompt: "Dê duplo clique para criar o primeiro estágio" + ícone de cursor duplo clique para reforçar affordance. Handler `onDoubleClick` ativo quando `readonly=false` e scope `process:cycle:write` presente. | EmptyState com ícone de fluxo + cursor hint |
| **error** | Toast de erro + canvas com última versão em cache (se disponível) | Toast RFC 9457 + correlationId |
| **readonly** | Banner "Ciclo publicado — use 'Nova versão' para editar" + nenhuma ação de escrita | Banner + botão "Nova versão" |

### 2.4 Alternativas e Erros

| Situação | Comportamento UX |
|---|---|
| Publicar sem estágio inicial | Toast erro: "O ciclo precisa de ao menos um estágio inicial antes de ser publicado." + nós piscam no canvas |
| Deleção de estágio com instâncias | Modal erro: "Este estágio possui N instância(s) ativa(s) em andamento." + badge ⚠ no nó |
| Criar segundo estágio inicial | Toast erro: "Este ciclo já possui um estágio inicial. Desmarque o atual antes de definir outro." |
| MOD-006 indisponível na deleção | Toast erro: "Não foi possível verificar instâncias ativas. Tente novamente." (503) |
| Ciclo PUBLISHED + tentativa de edição | Bloqueio na UI: canvas readonly, sem ações de escrita |
| Deprecar ciclo com sucesso | Modal de confirmacao: "Ao deprecar, novas instancias serao bloqueadas. Continuar?" + Toast sucesso: "Ciclo depreciado." |
| Deprecar ciclo DRAFT | Botão "Deprecar" oculto (somente PUBLISHED) |
| Duplo clique em canvas vazio sem macroetapas | Auto-cria macroetapa "Etapa Geral" + estágio "Novo estágio" com `is_initial: true` na posição do clique. Abre painel UX-PROC-002 automaticamente. |
| Duplo clique em área vazia do canvas com estágios existentes | Cria novo estágio "Novo estágio" com `is_initial: false` na primeira macroetapa existente (menor `ordem`). |
| Duplo clique sobre nó existente (StageNode) | Handler do canvas NÃO dispara — o handler do StageNode prevalece (abre painel de configuração). |
| Duplo clique durante criação em andamento (`isPending`) | Noop — clique ignorado para evitar duplicação de macroetapa/estágio. |
| Erro na criação da macroetapa (rede, 500) | Toast de erro com `correlation_id`. Canvas permanece no estado vazio. Estágio NÃO é criado. |
| Usuário sem scope `process:cycle:write` | Duplo clique no canvas não tem efeito (noop). |

### 2.7 Hooks Frontend (Implementação)

| Hook | Responsabilidade | Arquivo |
|---|---|---|
| `useCreateStageFromCanvas` | Criação condicional: macroetapa padrão (se necessário) + estágio com valores padrão + invalidação do flow + abertura do painel UX-PROC-002 | `apps/web/src/modules/process-modeling/hooks/use-create-stage-from-canvas.ts` |

**Contrato:**

```typescript
interface UseCreateStageFromCanvasOptions {
  cycleId: string;
  flow: FlowResponseDTO | undefined;
  readonly: boolean;
  userScopes: readonly string[];
}

interface UseCreateStageFromCanvasResult {
  handleCanvasDoubleClick: (event: React.MouseEvent, position: { x: number; y: number }) => void;
  isPending: boolean;
  lastCreatedStageId: string | null;
}
```

**Lógica interna:**

1. Guard: `readonly || !canWriteCycle(userScopes) || isPending` → return (noop)
2. Converter coordenadas do mouse via `reactFlowInstance.screenToFlowPosition()`
3. Se `flow.macro_stages.length === 0` → `POST /admin/cycles/:cid/macro-stages` com `{ codigo: "ETAPA-GERAL", nome: "Etapa Geral", ordem: 1 }`
4. Senão → usar `flow.macro_stages[0].id` (menor `ordem`)
5. Calcular `totalStages` somando stages de todas as macroetapas
6. `POST /admin/macro-stages/:mid/stages` com `{ codigo: "EST-{NNN}", nome: "Novo estágio", ordem: totalStages + 1, is_initial: totalStages === 0, canvas_x, canvas_y }`
7. `invalidateQueries(['process-modeling', 'flow', cycleId])`
8. `setLastCreatedStageId(response.id)` → FlowEditorPage usa para abrir StageConfigPanel

**Integração no FlowEditorPage.tsx:**

- Prop `onDoubleClick={handleCanvasDoubleClick}` no `<ReactFlow>`
- Renderização condicional de `<GhostNode>` quando `isPending`
- Efeito para abrir StageConfigPanel quando `lastCreatedStageId` muda (após refetch)

**Prevenção de conflito StageNode ↔ Canvas:**

O handler `onDoubleClick` do canvas NÃO DEVE disparar quando o target é um nó existente. Usar `onPaneDoubleClick` (se disponível) ou filtrar `event.target` no `onDoubleClick`.

## 3. UX-PROC-002 — Configurador de Estágio (Painel Lateral)

```
1. Analista clica ⚙ em um nó no canvas (UX-PROC-001)
```

## 4. Sincronização Bidirecional Canvas ↔ Painel

| Ação no Painel | Reflexo no Canvas |
|---|---|
| Salvar nome do estágio | Nó atualiza label em tempo real |
| Adicionar/remover gate | Badge de gates no nó incrementa/decrementa |
| Vincular/desvincular papel | Badge de papéis no nó incrementa/decrementa |
| Adicionar transição de saída | Aresta aparece no canvas |
| Remover transição | Aresta desaparece do canvas |
| Toggle is_initial | Badge ⚑ aparece/desaparece no nó |
| Toggle is_terminal | Badge ⊠ aparece/desaparece no nó |

| Ação no Canvas | Reflexo no Painel |
|---|---|
| Selecionar outro nó | Painel recarrega dados do novo estágio |
| Deletar nó selecionado | Painel fecha automaticamente |
| Criar transição via drag | Se painel aberto para o from_stage, aba Transições atualiza |

## Detalhamento

### Correção: `_auth.tsx` deve importar AppShell

**Estado atual (incorreto):**

O route file `apps/web/src/routes/_auth.tsx` contém um `AuthLayout` inline com ~25 linhas de JSX (sidebar placeholder, header com texto "Breadcrumbs", sem ProfileWidget).

**Estado esperado (conforme CA-07):**

```typescript
// apps/web/src/routes/_auth.tsx
import { createRoute, Outlet, redirect } from '@tanstack/react-router';
import { Route as rootRoute } from './__root';
import { AppShell } from '@modules/backoffice-admin/components/AppShell';

export const Route = createRoute({
  id: '/_auth',
  getParentRoute: () => rootRoute,
  beforeLoad: async ({ context }) => {
    if (!context.auth?.user) {
      throw redirect({ to: '/login' });
    }
  },
  component: AuthLayout,
});

function AuthLayout() {
  return (
    <AppShell>
      <Outlet />
    </AppShell>
  );
}
```

> **Nota:** O `AppShell` aceita `children: ReactNode` (Props, linha 23-25). TanStack Router layout routes usam `<Outlet />` para renderizar child routes. O wrapper `AuthLayout` faz a ponte entre o Router e o AppShell.

O `AppShell` já existe em `apps/web/src/modules/backoffice-admin/components/AppShell.tsx` e implementa:
- Sidebar filtrada por scopes (BR-005)
- Header com breadcrumbs dinâmicos
- ProfileWidget com dropdown (logout, change password)
- Skeleton durante loading de `useAuthMe()`
- Empty state quando scopes=[]
- Redirect para /login em 401

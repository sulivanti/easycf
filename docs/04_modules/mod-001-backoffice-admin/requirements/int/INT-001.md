# INT-001 — Integrações e Contratos do Backoffice Admin

## INT-001 — POST /api/v1/auth/login (auth_login)

- **Sistema externo:** MOD-000 (Foundation API)
- **Modo:** API (HTTP REST)
- **Auth:** Nenhuma (endpoint público — pré-autenticação)
- **Contrato:**
  - **Request:** `POST /api/v1/auth/login` — `{ email: string, password: string, remember_me?: boolean, device_fp?: string }`
  - **Response 200:** `{ access_token, refresh_token, token_type, expires_in, user, mfa_required?: boolean, temp_token?: string }`
  - **Response 401:** RFC 9457 — `{ type: "/problems/unauthorized", title: "Unauthorized", status: 401, extensions: { correlationId } }`
  - **Response 403:** RFC 9457 — Conta bloqueada
  - **Response 429:** RFC 9457 — Rate limit com `retry_after`
- **Timeout:** 5000ms
- **Retry:** N/A (retry delegado ao usuário via botão)
- **Dead Letter Queue (DLQ):** [x] Não
- **Idempotência:** N/A (gerenciada pelo MOD-000)
- **Fallback:** Toast de erro com correlationId (DOC-UX-012 §2.1)
- **Observabilidade:** `["correlationId", "logs"]` — X-Correlation-ID no header, UIActionEnvelope (submit_login)

- **estado_item:** DRAFT
- **owner:** arquitetura
- **data_ultima_revisao:** 2026-03-17
- **rastreia_para:** US-MOD-001-F01, FR-001, BR-001, US-MOD-000-F01

## INT-002 — POST /api/v1/auth/logout (auth_logout)

- **Sistema externo:** MOD-000 (Foundation API)

- **Auth:** JWT (Bearer Token via httpOnly cookie)
- **Contrato:**
  - **Request:** `POST /api/v1/auth/logout` — body vazio
  - **Response 200:** `{ message: "Logged out" }`
  - **Response erro de rede:** Redirecionar para /login mesmo assim, limpando estado local
- **Timeout:** 5000ms
- **Retry:** N/A (em caso de erro de rede, redirect direto)

- **Fallback:** Limpar estado local e redirecionar para /login independente do resultado
- **Observabilidade:** `["correlationId", "logs"]` — X-Correlation-ID, UIActionEnvelope (submit_logout)

## INT-005 — GET /api/v1/auth/me (auth_me)

- **Sistema externo:** MOD-000 (Foundation API)
- **Modo:** API (HTTP REST)
- **Auth:** JWT (Bearer Token via httpOnly cookie)
- **Contrato:**
  - **Request:** `GET /api/v1/auth/me`
  - **Response 200:** `{ id, name, email, scopes: string[], tenant: { id, name }, ... }`
  - **Response 401:** Sessão expirada → interceptor redireciona para /login
  - **Response 5xx:** Toast com correlationId + botão retry (NÃO desconectar)
- **Timeout:** 3000ms (skeleton timeout do Dashboard — BR-009, NFR-001)
- **Retry:** Manual via botao "Tentar novamente" no estado de erro (BR-009, BR-010). Cada retry gera novo `correlation_id` UUID v4 e novo UIActionEnvelope (`dashboard_retry`).
- **Fallback:**
  - Timeout > 3s → skeleton substituido por estado de erro parcial (nunca tela branca) + Toast com correlationId + botao retry (BR-009)
  - Erro 5xx → Toast com correlationId + botao retry, SEM desconectar o usuario (BR-010)
  - Erro 401 → redirect /login via interceptor HTTP global (unico cenario de desconexao)
- **Dead Letter Queue (DLQ):** Não
- **Idempotência:** N/A (leitura)
- **Observabilidade:** `["correlationId", "logs"]` — X-Correlation-ID, UIActionEnvelope (load_current_user, load_dashboard_profile, dashboard_retry)

- **estado_item:** DRAFT
- **owner:** arquitetura
- **data_ultima_revisao:** 2026-03-17
- **rastreia_para:** US-MOD-001-F01, US-MOD-001-F03, FR-004, FR-005, BR-005, BR-008, BR-009, BR-010, NFR-001, US-MOD-000-F08

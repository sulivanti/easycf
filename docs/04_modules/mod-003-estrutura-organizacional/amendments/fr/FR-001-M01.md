## Detalhamento

### FR-001 §1 — Campos Cadastrais nos Endpoints Existentes

#### POST /org-units (Create)

Request body DEVE aceitar campos opcionais adicionais:

```json
{
  "codigo": "UN-SP-01",
  "nome": "A1 Engenharia",
  "descricao": "Unidade de engenharia",
  "parent_id": "uuid-pai",
  "cnpj": "12.345.678/0001-90",
  "razao_social": "A1 Engenharia e Construcoes Civis Ltda",
  "responsavel": "Marcos Silva"
}
```

Campos `cnpj`, `razao_social`, `filial`, `responsavel`, `telefone`, `email_contato` sao todos opcionais no create.

#### PATCH /org-units/:id (Update)

Request body DEVE aceitar atualizacao dos novos campos:

```json
{
  "nome": "A1 Engenharia Atualizada",
  "cnpj": "12.345.678/0001-90",
  "razao_social": "...",
  "filial": "Sao Paulo - SP",
  "responsavel": "Marcos Silva",
  "telefone": "(11) 3456-7890",
  "email_contato": "contato@a1engenharia.com.br"
}
```

Todos os campos novos sao opcionais no update (partial patch). Campo omitido = nao altera.

#### GET /org-units/:id (Detail)

Response DEVE incluir os novos campos:

```json
{
  "id": "uuid",
  "codigo": "UN-SP-01",
  "nome": "A1 Engenharia",
  "nivel": 2,
  "status": "ACTIVE",
  "cnpj": "12.345.678/0001-90",
  "razao_social": "A1 Engenharia e Construcoes Civis Ltda",
  "filial": "Sao Paulo - SP",
  "responsavel": "Marcos Silva",
  "telefone": "(11) 3456-7890",
  "email_contato": "contato@a1engenharia.com.br",
  "ancestors": [...],
  "tenants": [...]
}
```

Campos null DEVEM ser retornados como `null` (nao omitidos).

#### GET /org-units/tree (Tree)

Nos da arvore NAO precisam dos campos cadastrais (performance). Manter response atual.

#### GET /org-units (List)

Items da lista NAO precisam dos campos cadastrais. Manter response atual.

### FR-001 §2 — Validacao dos Novos Campos

| Campo | Validacao |
|---|---|
| cnpj | varchar max 18, formato livre (validacao de formato e responsabilidade do frontend) |
| razao_social | varchar max 300 |
| filial | varchar max 100 |
| responsavel | varchar max 200 |
| telefone | varchar max 20 |
| email_contato | varchar max 254, validacao basica de formato (contem @) |

Nenhum dos campos novos e obrigatorio. Nenhum participa de regras de unicidade.

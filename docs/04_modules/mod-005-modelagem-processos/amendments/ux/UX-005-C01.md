## Detalhamento

### 1. Correção em §2.2 — Estados da Tela (estado `empty`)

**Antes (atual):**

> | **empty** | Canvas vazio com prompt: "Dê duplo clique para criar o primeiro estágio" + ícone de cursor duplo clique para reforçar affordance. Handler `onDoubleClick` ativo quando `readonly=false` e scope `process:cycle:write` presente. | EmptyState com ícone de fluxo + cursor hint |

**Depois (corrigido):**

O estado `empty` passa a ter 3 variantes visuais conforme o motivo:

| Variante | Condição | Mensagem | Cursor | Ação |
|----------|----------|----------|--------|------|
| **empty_editable** | `readonly=false` && `canWriteCycle=true` | "Dê duplo clique para criar o primeiro estágio." | `pointer` | `onDoubleClick` ativo |
| **empty_readonly** | `readonly=true` (PUBLISHED/DEPRECATED) | "Ciclo publicado — somente leitura." | `default` | Sem handler de double-click |
| **empty_no_permission** | `readonly=false` && `canWriteCycle=false` | "Sem permissão de edição neste ciclo." | `default` | Sem handler de double-click |

### 2. Correção em §2.4 — Alternativas e Erros

**Adicionar à tabela existente:**

| Situação | Comportamento UX |
|----------|------------------|
| Usuário sem scope `process:cycle:write` em canvas vazio | Exibir mensagem contextual "Sem permissão de edição neste ciclo." com cursor `default`. **Não** exibir instrução de double-click. |
| Erro na criação do estágio (rede, 500) | Toast de erro: "Erro ao criar estágio. Tente novamente." com `correlation_id` quando disponível. Canvas permanece no estado vazio. |
| Ciclo PUBLISHED/DEPRECATED em canvas vazio | Exibir mensagem readonly correspondente. **Não** exibir instrução de double-click. |

**Alterar entrada existente:**

- "Usuário sem scope `process:cycle:write`" → de "Duplo clique no canvas não tem efeito (noop)" para "Canvas vazio exibe 'Sem permissão de edição neste ciclo.' com cursor default. Duplo clique desabilitado."

### 4. Correção em §2.7 — Hooks Frontend

**Expandir contrato de UseCreateStageFromCanvasResult:**

```typescript
interface UseCreateStageFromCanvasResult {
  handleCanvasDoubleClick: (event: React.MouseEvent) => void;
  isPending: boolean;
  lastCreatedStageId: string | null;
  error: string | null;         // NOVO: mensagem de erro da última tentativa de criação
  canCreate: boolean;           // NOVO: !readonly && canWriteCycle(userScopes) && !!flow
  blockReason: string | null;   // NOVO: razão pelo qual canCreate=false (COPY key)
}
```

**Expandir lógica interna (ponto 1 — Guards):**

Cada guard, além do `return`, deve:
- Emitir `console.warn('[useCreateStageFromCanvas] Blocked: {motivo}')` para debugging
- Popular `blockReason` com a COPY key correspondente

**Adicionar onError na mutation:**

```typescript
onError: (err) => {
  setError(err instanceof Error ? err.message : COPY.error_create_stage);
}
```

**Adicionar await no invalidateQueries (ponto 7):**

```typescript
onSuccess: async (stage) => {
  await qc.invalidateQueries({ queryKey: [...FLOW_KEY, cycleId] });
  setLastCreatedStageId(stage.id);
}
```

O `await` garante que o re-fetch complete antes de sinalizar sucesso, evitando flash do estado vazio.

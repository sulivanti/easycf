## Detalhamento

### 1. Nova jornada: Criação de estágio via duplo clique no canvas

Adicionar à seção **2.1 Jornada Principal** (UX-005 §2.1) o fluxo alternativo de primeiro estágio:

```
Fluxo Alternativo — Primeiro Estágio (Canvas Vazio):
1. Analista acessa /processos/ciclos/:id/editor (ciclo DRAFT recém-criado)
2. GET /admin/cycles/:id/flow retorna grafo vazio (macro_stages: [])
3. Canvas exibe estado empty: "Dê duplo clique para criar o primeiro estágio"
4. Analista dá duplo clique no canvas
5. Sistema auto-cria macroetapa padrão ("Etapa Geral") via POST /admin/cycles/:cid/macro-stages
6. Sistema cria estágio "Novo estágio" com is_initial=true na posição do clique via POST /admin/macro-stages/:mid/stages
7. Canvas refetch → nó aparece na posição do clique
8. Painel de configuração (UX-PROC-002) abre automaticamente para o estágio criado
```

### 4. Nova entrada na tabela §2.4 (Alternativas e Erros)

| Situação | Comportamento UX |
|---|---|
| Duplo clique em canvas vazio sem macroetapas | Auto-cria macroetapa "Etapa Geral" + estágio "Novo estágio" com `is_initial: true` na posição do clique. Abre painel UX-PROC-002 automaticamente. |
| Duplo clique em área vazia do canvas com estágios existentes | Cria novo estágio "Novo estágio" com `is_initial: false` na primeira macroetapa existente (menor `ordem`). |
| Duplo clique sobre nó existente (StageNode) | Handler do canvas NÃO dispara — o handler do StageNode prevalece (abre painel de configuração). |
| Duplo clique durante criação em andamento (`isPending`) | Noop — clique ignorado para evitar duplicação de macroetapa/estágio. |
| Erro na criação da macroetapa (rede, 500) | Toast de erro com `correlation_id`. Canvas permanece no estado vazio. Estágio NÃO é criado. |
| Usuário sem scope `process:cycle:write` | Duplo clique no canvas não tem efeito (noop). |

### 6. Novo hook frontend

| Hook | Responsabilidade | Arquivo |
|---|---|---|
| `useCreateStageFromCanvas` | Encapsula lógica de criação condicional: macroetapa padrão (se necessário) + estágio com valores padrão + invalidação do flow + abertura do painel | `apps/web/src/modules/process-modeling/hooks/use-create-stage-from-canvas.ts` |

**Contrato do hook:**

```typescript
interface UseCreateStageFromCanvasOptions {
  cycleId: string;
  flow: FlowResponseDTO | undefined;
  readonly: boolean;
  userScopes: readonly string[];
}

interface UseCreateStageFromCanvasResult {
  handleCanvasDoubleClick: (event: React.MouseEvent, position: { x: number; y: number }) => void;
  isPending: boolean;
  lastCreatedStageId: string | null;
}
```

**Lógica interna:**

1. Guard: `readonly || !canWriteCycle(userScopes) || isPending` → return (noop)
2. Converter coordenadas do mouse via `reactFlowInstance.screenToFlowPosition()`
3. Se `flow.macro_stages.length === 0` → `POST /admin/cycles/:cid/macro-stages` com `{ codigo: "ETAPA-GERAL", nome: "Etapa Geral", ordem: 1 }`
4. Senão → usar `flow.macro_stages[0].id` (menor `ordem`)
5. Calcular `totalStages` somando stages de todas as macroetapas
6. `POST /admin/macro-stages/:mid/stages` com `{ codigo: "EST-{NNN}", nome: "Novo estágio", ordem: totalStages + 1, is_initial: totalStages === 0, canvas_x, canvas_y }`
7. `invalidateQueries(['process-modeling', 'flow', cycleId])`
8. `setLastCreatedStageId(response.id)` → FlowEditorPage usa para abrir StageConfigPanel

### 7. Integração no FlowEditorPage.tsx

Adicionar ao `<ReactFlow>`:
- Prop `onDoubleClick={handleCanvasDoubleClick}` (do hook `useCreateStageFromCanvas`)
- Renderização condicional de `<GhostNode>` quando `isPending` (overlay na posição do clique)
- Efeito para abrir StageConfigPanel quando `lastCreatedStageId` muda (após refetch)

### 8. Regra de prevenção de conflito StageNode ↔ Canvas

O handler `onDoubleClick` do `<ReactFlow>` NÃO DEVE disparar quando o target do evento é um nó existente. Verificar via `event.target` ou usando o callback `onNodeDoubleClick` separado do React Flow (que já existe no StageNode).

O React Flow dispara `onPaneClick` / `onPaneContextMenu` apenas para cliques no canvas vazio (não sobre nós), portanto usar `onPaneDoubleClick` (se disponível na versão utilizada) ou filtrar `event.target` no `onDoubleClick`.

| Métrica | Alvo | Endpoint(s) | Fonte |
|---|---|---|---|

| Latência P95 — Fork atômico | < 2000ms | POST /admin/cycles/:id/fork | Estimativa — fork copia 7 tabelas em transação |
| Disponibilidade | 99.5% | Todos os endpoints do MOD-005 | Padrão plataforma |

### 1.1 SLO do Endpoint /flow (Crítico)

O endpoint `/flow` é consumido pelo editor visual (UX-PROC-001) em cada carregamento de tela. O SLA < 200ms é um OKR do módulo (OKR-4).

**Estratégia de otimização:**
- Query única com JOINs (DATA-005 §5.1) + agrupamento no service
- Indexes otimizados: `idx_macro_stages_cycle_ordem`, `idx_stages_macro_ordem`, `idx_gates_stage_ordem`
- Se ultrapassar 200ms com grafos grandes (>50 nós): considerar CTE ou queries separadas paralelas
- Monitorar com métricas de latência P95/P99

## 5. Limites e Capacidade

| Recurso | Limite | Justificativa |
|---|---|---|
| Nós por ciclo (estágios) | até **50** sem degradação visual | OKR-4 — editor visual sem falha até 50 nós |
| Macroetapas por ciclo | até **8** | Limite de cores distintas no canvas (UX-PROC-001) |
| Gates por estágio | até **10** | Limite prático de avaliação sequencial |
| Papéis por estágio | até **10** | Limite prático de atribuição |
| Transições por estágio (saída) | até **10** | Limite visual de arestas no canvas |
| Mini-mapa | obrigatório a partir de **15 nós** | US-MOD-005 épico, OKR-4 |
| Fork — tempo máximo | **5s** (timeout) | Cópia atômica de 7 tabelas; se falhar, rollback completo |
| Paginação — cycles list | cursor-based, **50 por página** | Padrão DOC-GNP-00 |
| Paginação — process-roles list | cursor-based, **50 por página** | Padrão DOC-GNP-00 |

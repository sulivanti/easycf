## Detalhamento

### SEC-002 — Novas Linhas na Matriz de Autorização

Adicionar à tabela "Matriz de Autorização — Domain Events MOD-003":

| action | event_type | operation_id | emit_perm | view | notify | sensitivity |
|---|---|---|---|---|---|---|
| Criar departamento | `org.dept_created` | `departments_create` | `org:dept:write` | RBAC `org:dept:read` | creator + admin | 0 |
| Atualizar departamento | `org.dept_updated` | `departments_update` | `org:dept:write` | RBAC `org:dept:read` | — | 0 |
| Desativar departamento (soft delete) | `org.dept_deleted` | `departments_delete` | `org:dept:delete` | RBAC `org:dept:read` | admin | 0 |
| Restaurar departamento | `org.dept_restored` | `departments_restore` | `org:dept:write` | RBAC `org:dept:read` | admin | 0 |

### Payload Policy por Evento (Departamentos)

| event_type | Campos obrigatórios no payload | Campos opcionais | Payload policy |
|---|---|---|---|
| `org.dept_created` | `id`, `tenant_id`, `codigo`, `nome`, `cor`, `status`, `created_by` | `descricao` | snapshot mínimo |
| `org.dept_updated` | `id`, `codigo`, campos alterados (diff) | — | diff only |
| `org.dept_deleted` | `id`, `codigo`, `tenant_id` | — | metadata |
| `org.dept_restored` | `id`, `codigo`, `tenant_id` | — | metadata |

### Masking Policy

- **Sem campos mascaráveis:** Dados de departamentos são organizacionais (sensitivity_level=0), sem PII.

### Regras de Filtragem (Departamentos)

- **MUST:** Departamentos são por-tenant — filtragem de domain events com `entity_type=department` DEVE filtrar por `tenant_id` do JWT (diferente de org_units que é cross-tenant — ADR-003)
- **MUST:** View restrita a usuários com scope `org:dept:read` E pertencentes ao mesmo tenant
- **MUST:** Notificações DEVEM ser entregues apenas a destinatários com scope `org:dept:read` no perfil ativo
